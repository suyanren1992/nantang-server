## 做什么

上线六个社区空间模块 + 新手任务系统 + 邀请码管理。按 P0→P1→P2 分批执行。

## 关键设计决策

- 食堂预定 vs 现场差价（10 NT vs 15 NT）——鼓励预定减少浪费
- 客栈日期冲突检测——提交和确认时双重校验
- 茶馆 7 天自动归档——渲染时过滤，无定时任务
- 桌游室报名费归发起人，平台不抽成；未到场者自动退款
- 新手任务替代「注册送 100 NT」——75 NT 通过 5 个任务赚取
- CV 抵押：新人 CV 先冻结，每周大扫除后解冻 50%，两周全解。两周内离开→冻结 CV 归公共池
- 取消注册赠送 100NT——新成员初始 NT 为 0
- `ponytail:` 单文件超过 3000 行后按 [模块化策略](../执行方案.md#移动端代码模块化策略) 拆分。Step 5 P0+P1 预计新增 ~1300 行，加上已有 Step 0-4 的 ~1700 行，刚好触及拆分阈值。在本阶段 P1 完成后评估是否拆分。
- `ponytail:` NPC 权限（高琳姐→客栈+食堂，文峰→茶馆）按 `world_id` 硬匹配——人数 >10 时需要角色配置表替代硬编码。

## 第二批（P1）

### 🏨 客栈

- 房型：梅兰竹菊（单人间 ×4）+ A/B（四人间 ×2），共 12 床位
- 入住申请 + 摊主确认 + 日期冲突检测
- 退宿 + 欠费处理
- 数据：`data.inn_rooms[]` + `data.inn_bookings[]`

**房型**：

| 类型 | 编号 | 床位数 | 名称 |
|------|------|--------|------|
| 单人间 | 4间 | 1人/间 | 梅、兰、竹、菊 |
| 四人间 | 2间 | 4人/间 | A、B |

**日期冲突检测**：
```js
function isRoomAvailable(roomId, checkIn, checkOut) {
  return !(data.inn_bookings || []).some(function(b) {
    if (b.room !== roomId) return false;
    if (b.status === 'cancelled' || b.status === 'rejected') return false;
    return checkIn < b.check_out && checkOut > b.check_in;
  });
}
```

**欠费处理**（退宿时余额不足）：
- 扣除全部现有余额
- 记录欠费，用户标记 `in_debt`
- 下次获得 NT 时自动抵扣
- 不限制操作（仍可接委托、标记物品），但欠费提醒在个人页面显示

### 🎲 桌游室

- 发起组局（游戏名/时间/地点/人数/报名费）
- 报名 + 支付
- 发起人确认到场 → 未到场退款
- 报名费归发起人，平台不抽成
- 数据：`data.game_sessions[]`

### 🍵 茶馆

- 发布八卦（5 NT/条，防灌水）
- 7 天自动归档（渲染时 filter，无定时任务）
- 数据：`data.teahouse_posts[]`

### 🌱 新手任务

5 任务清单：
```
□ 签署社区公约                    +5 NT
□ 认识 3 位在地成员              +15 NT
□ 社区空间打卡（5个空间）         +20 NT
□ 参加 1 次饭局                  +10 NT
□ 参加 1 次大扫除                +15 NT
全部完成额外奖励：                +10 NT
─────────────────────────
总计可获得：                     75 NT
```
75 NT ≈ 2.5 天的住宿费。

数据：`data.newbie_quests[name]`

### 🔒 CV 抵押

- 新人 CV 冻结池（`nt_users[name].frozenCV`）
- 每周大扫除后解冻 50%
- 两周全解
- 提前离开→冻结 CV 归公共池

### 🎫 邀请码

- 管理员生成（一次性/多次 + 备注）
- 使用追踪 + 过期管理
- 数据：`data.invite_codes[]`

| 触发条件 | 提醒内容 |
|---------|---------|
| 10:00 | 午餐预定还有 30 分钟截止 |
