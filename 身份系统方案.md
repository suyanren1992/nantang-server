# 身份系统方案

> 2026-07-18 · 用户ID + 角色升级 + 个人资料

## 一、身份流转

```
注册 → 云村民(visitor)
         ├── 认领「打包任务」并完成 → 共建者(builder)
         ├── 输入NPC邀请码 → 南塘NPC(npc)
         └── 输入冒险者邀请码 → 冒险者(adventurer)

管理员(admin) — 仅砚仁，不开放注册
```

## 二、用户数据结构

```javascript
{
  name: "小云",
  role: "visitor",          // visitor | builder | npc | adventurer | admin
  password: "hash...",
  avatar_seed: 0,
  created: "2026-07-18",
  uid: "YT-0003-NT",       // 自动生成唯一ID
  inviteCode: "",           // 升级时填入
  bio: "",                  // 个人简介（新用户空白）
  wallet: "",               // 钱包地址（新用户空白）
  location: "",             // 坐标（新用户空白）
  tasksCompleted: 0,        // 完成任务数
  ntEarned: 0,              // 累计NT
  season_history: []        // 历期身份记录
}
```

## 三、新增函数

```javascript
// 角色升级
function upgradeRole(name, newRole, inviteCode) {
  var users = getUsers();
  if (!users[name]) return {ok: false, error: '用户不存在'};
  if (newRole === 'admin') return {ok: false, error: '不可升级为管理员'};
  users[name].role = newRole;
  users[name].inviteCode = inviteCode || '';
  users[name].season_history.push({role: newRole, changed_at: today()});
  saveUsers(users);
  return {ok: true};
}
```
