
**核心原则**：

- 复用已有的 `avatarURL()`/`avatarImg()`（L467-468），不手写 URL
- 不把 DOM 操作放进 `app-data.js`——保持数据层纯净
- AppData 只新增一个纯数据方法：`myPendingCount()`

## 四、实现：扩展 refreshUserUI()

**位置**：[nantang-mobile.html](nantang-mobile.html)（扩展现有 L1291-1309 的 `refreshUserUI` 函数）

**策略**：在现有函数末尾追加缺失的 DOM 更新，不重写已有逻辑。

```javascript
function refreshUserUI(){
  // ══ 已有逻辑（保持不变） ══
  var u = CURRENT_USER;
  var url = avatarURL(u, 40);  // 复用 L467 的工具函数，替换手写 URL
  var sa = document.querySelector('#spcCard .spc-avatar img'); if(sa) sa.src = url;
  var sn = document.querySelector('#spcCard .spc-name'); if(sn) sn.textContent = u;
  var ma = document.querySelector('.my-topbar .my-avatar img'); if(ma) ma.src = url;
  var mn = document.querySelector('.my-topbar .my-name'); if(mn) mn.textContent = u;

  // 动态角色（复用已有逻辑，不再手写 roleMap）
  var role = '云村民';
  var users = getUsers();
  if (users[u]) {
    var r = users[u].role;
    role = r === 'admin' ? '管理员' : r === 'builder' ? '共建者' : r === 'adventurer' ? '冒险者' : '云村民';
  }

  // 动态天数（修复 "120天" 硬编码）
  var days = 1;
  var appUser = window.AppData ? AppData.me() : null;
  if (appUser && appUser.created) {
    days = Math.max(1, Math.floor((Date.now() - new Date(appUser.created).getTime()) / 86400000));
  }

  var sr = document.querySelector('#spcCard .spc-subtitle span:first-child');
  if (sr) sr.textContent = '🧱 ' + role + ' · 🏅 · ' + days + '天';
  var mr = document.querySelector('.my-id-row2');
  if (mr) mr.textContent = '🧱 ' + role + ' · 来了 ' + days + ' 天';

  // 营期
  var campRow = document.querySelector('.my-id-row3');
  if (campRow && appUser && appUser.camp) campRow.textContent = '🏕️ ' + appUser.camp;

  // NT 余额（顶栏 + 个人NT卡）— 已有逻辑保持不变
  if (window.NT) {
    var bal = NT.getUser(u);
    var b = bal ? bal.ntBalance : 0;
    var ntVal = document.querySelector('.my-nt-val');
    if (ntVal) ntVal.innerHTML = '<img src=豆子.png style=width:18px;height:18px;vertical-align:middle;margin-right:2px>' + b;
    var ntCard = document.getElementById('myNtBalance');
    if (ntCard) ntCard.textContent = b;
    // 个人 NT 进度条
    var ntBar = document.querySelector('#myNtBalance + div > div');
    // ponytail: ntGoal 硬编码 1250，等 AppData 支持 ntGoal 字段后改为动态
    if (ntBar) ntBar.style.width = Math.min(100, Math.round(b / 1250 * 100)) + '%';
  }

  // ══ 新增逻辑 ══

  // 村口气泡角标
  var pendingCount = window.AppData ? AppData.myPendingCount() : 0;
  var badge = document.querySelector('#spcCard .spc-badge');
  if (badge) {
    badge.textContent = pendingCount || '';
    badge.style.display = pendingCount > 0 ? '' : 'none';
  }

  // 村口委托数
  var marketCount = window.AppData ? AppData.taskMarket().length : Object.keys(TASKS).length;
  var vpCount = document.getElementById('vpQuestCount');
  if (vpCount) vpCount.textContent = marketCount + ' 个委托';

  // 任务大厅底栏
  var qma = document.querySelector('.quest-my-avatar-img img');
  if (qma) qma.src = avatarURL(u, 32);
  var qmn = document.querySelector('.quest-my-btn span');
  if (qmn) qmn.textContent = u;

  // 资料卡 seed
  _profileSeed = u;

  // 提醒芯片（委托角标）
  if (typeof updateMyChips === 'function') updateMyChips('tasks');

  // 通知地图 iframe
  var mapFrame = document.querySelector('#overlayMap iframe');
  if (mapFrame && mapFrame.contentWindow) {
    try { mapFrame.contentWindow.postMessage({ type: 'userUpdate', name: u }, '*'); } catch(e) {}
  }
}
```

**改动要点**：
- L1292：`avatarURL(u, 40)` 替换手写的 `'https://api.dicebear.com/7.x/avataaars/svg?seed='+...`（复用 L467 工具函数）
- L1299-1300：`120天` → 动态 `days` 变量
- 追加：营期行、大厅底栏、村口角标、村口委托数、NT 进度条、资料卡 seed
- 不新增 `setText/setHtml` 辅助函数——每个选择器只出现一次，不需要抽象

## 五、AppData 新增：仅一个方法

**位置**：[js/app-data.js](js/app-data.js)，在 `taskMarket()` 之后追加：

```javascript
  // ══ 待处理计数（当前用户的待提交/待审核/退回修改任务数） ══
  myPendingCount: function() {
    var me = this._currentUser;
    var all = this._data.tasks;
    return Object.values(all).filter(function(t) {
      var isMy = (t.claimants || []).some(function(c) { return c.name === me; }) || t.publisher === me;
      if (!isMy) return false;
      return t.status === '待提交' || t.status === '待审核' || t.status === '退回修改';
    }).length;
  },
```

> 注意：当前 `updateMyChips()` 直接读全局 `TASKS` 对象，而 `AppData.myPendingCount()` 读 `AppData._data.tasks`。两者在种子数据加载后一致，但运行时如果通过 `TASKS[name] = ...` 直接写入，AppData 不会同步。
> **ponytail**: 后续将 `TASKS` 的写操作统一走 `AppData.addTask/updateTask`，天花板：当 TASKS 写入点超过 5 个时。
