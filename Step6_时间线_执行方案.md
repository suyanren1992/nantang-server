# Step 6：S2 统一时间线

> 来源文档：`05_在地记录与NT激励.md`（主要蓝本）、`06_云村民互动层.md`（点赞/留言/打赏）
> 前置依赖：Step 3（打扫）、Step 4（NT/CV/XP）、Step 5（社区活动）
> 解锁：Step 7（服务器上线）

---

## 做什么

把所有系统中产生的事件汇入统一的时间线。支持按空间/按人/按时间筛选。每条记录带心情、标签、照片、点赞、留言。

## 关键设计决策

- 时间线是**展示层**——展示上面所有阶段产生的数据，本身不产生新数据
- 删掉原文档的「日限 30 NT」——熟人社区不存在灌水刷 NT 的土壤
- 物品操作自动生成记录：入库/出库时自动发一条 journal entry（可选关闭）。**CV/XP 奖励已在 Step 2/4 的物品操作中发放，journal entry 的 cvReward/xpReward 字段是展示用（记录当时发放了多少），不重复发放。**
- 打赏本质是 `recordTransaction({type:'tip', from, to, amount})`
- 点赞不产生 NT 流转，纯社交信号
- 照片存 IndexedDB（已在 Step 2 建好存储层）

## 记录类型与 CV/XP 奖励

| 类型 | 说明 | CV | XP | 有照片加成 |
|------|------|:--:|:--:|:--:|
| 🍳 厨房记录 | 做饭相关 | +3 | +3 | +1 |
| 🌿 田间记录 | 种植/收获 | +3 | +3 | +1 |
| 🔧 修缮记录 | 修理/维护 | +5 | +5 | +1 |
| 🎨 创作记录 | 画画/手工 | +3 | +3 | +2 |
| 🧹 打扫记录 | 打扫（非任务） | +2 | +2 | +1 |
| 📋 日常记录 | 其他日常 | +2 | +2 | 0 |
| 📦 入库记录 | 物品入库 | +2 | +2 | 0 |
| 🗑 出库记录 | 物品消耗 | +1 | +1 | 0 |

## 数据模型

```js
  linkedItems: ["inv-001"],

    { user: "阿楠", content: "好香！", time: "14:35" }
  ]

```

## 统一时间线汇入来源

- Step 1：用户注册事件
- Step 2：物品放入/消耗/过期事件
- Step 3：打扫开始/提交/复议通过事件
- Step 5：委托发布/接取/完成/确认 + 食堂预定/客栈入住/桌游报名
- Step 6：日常记录/创作记录/修缮记录（用户手动发布）

## 具体任务

| # | 任务 | 说明 |
|---|------|------|
| 6.1 | `nt_journal` 数据模型 | entries[] 结构 + IndexedDB 照片存储 |
| 6.2 | 发布记录 UI | 拍照(可选)+一句话+类型下拉+空间标签+心情 emoji+标签 |
| 6.3 | 时间线渲染 | 时间倒序、按天分组、每条显示头像+类型+内容+照片+互动 |
| 6.4 | 空间过滤 | 时间线→只看厨房/画室等特定空间 |
| 6.5 | 按人过滤 | 时间线→只看某人 |
| 6.6 | 物品操作自动生成记录 | 入库/出库时自动创建 journal entry |
| 6.7 | 点赞系统 | toggle 逻辑，不涉及 NT，被点赞记录排序权重提升 |
| 6.8 | 留言系统 | 物品留言 + 空间留言 + 记录留言，通知被留言者 |
| 6.9 | 打赏系统 | `recordTransaction({type:'tip', from, to, amount})`，可选留言 |
| 6.10 | 名片弹窗 | 时间线/留言中点人头像→弹出迷你资料卡（L0-L1 免费信息） |

## 验证

发一条「包了饺子」→出现在时间线→+3 CV +3 XP→云村民可点赞/留言/打赏→可按「厨房」过滤只看厨房记录。
