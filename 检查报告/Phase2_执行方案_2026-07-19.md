# Phase 2 执行方案 · 地图数据全部可配置化

> 日期：2026-07-19 | 基于全量硬编码审计
> 目标：地图中 60+ 项硬编码数据全部迁入 AppData，管理员可编辑，所有用户实时可见
> 前置：Phase 1（共享/私有分层）已完成

---

## 现状

Phase 1 搭了壳（`map_locations` 字段、`getBuildings()` 桥接、`getPlots()`），但**只 seed 了住宿/人员/田地**。地图的建筑结构、房间物品、清洁标准、NT 定价、信息页模板——全部还是硬编码。

地图的数据分成三类：

| 类 | 说明 | 当前 | 目标 |
|----|------|------|------|
| 配置数据 | NT 定价、脏污速率、过期天数 | 硬编码常量 | AppData `map_locations.config`，管理员编辑 |
| 状态数据 | 房间物品、田地作物、在场人员、停车 | 硬编码在 HARDCODED_BUILDINGS | AppData `map_locations.state`，通过操作 UI 更新 |
| 结构数据 | 建筑 ID/名/图标/楼层/房间布局 | 硬编码在 HARDCODED_BUILDINGS | AppData `map_locations.buildings`，seed 一次 |

---

## 方案概览

```
map_locations (AppData._data, shared)
├── buildings[]        ← 结构数据（seed 一次，地图渲染用）
├── state              ← 状态数据（读写频繁）
│   ├── room_items[]        房间物品清单
│   ├── people_on_site[]    在场人员
│   ├── parking{}          停车信息
│   ├── stage{}            戏台状态
│   └── field_plots[]      田地作物
├── config             ← 配置数据（管理员编辑）
│   ├── cleaning_pricing   清洁 NT 定价
│   ├── dirtiness_rates    脏污每日增长率
│   ├── dirtiness_thresholds 绿/黄/红/急 阈值
│   ├── item_expiry_days   物品默认过期天数
│   └── nt_rewards         stock/cleaning NT 奖励
└── accommodations{}   ← 住宿（已有）
```

#### 票 D2：地图从 `map_locations.state` 读取，不再读硬编码

**文件**：`../实景地图移动端代码/app.js`
**做什么**：

所有渲染函数改为从 `Game.getData().map_locations` 读取，fallback 到 HARDCODED_BUILDINGS（兜底）：

1. `renderInfoPage()` 中的硬编码字符串 → 读 `map_locations.state`：
   - 在场人员列表 → `state.people_on_site`
   - 住宿摘要 → `map_locations.accommodations`
   - 田地摘要 → `map_locations.plots`
   - 厨房物品 → `state.room_items`（过滤 roomId 含 'kitchen'）
   - 停车 → `state.parking`
   - 戏台 → `state.stage`

2. `renderFieldPanel()` 中的提醒 → `map_locations.plots`（已在 B4 部分完成）

3. 各房间渲染（`buildRoomDetail`）中的物品 → `state.room_items` 过滤当前 roomId

4. `_submitMyCleaning()` 等函数中的硬编码日期 `'7/17'` → 改用 `Clock.today()` 或 `(typeof Clock!=='undefined'?Clock.today():new Date().toISOString().slice(5,10))`

5. `renderKitchenPanel()` 中的提醒 → 从 `state.room_items` 过滤 warn 项

6. `_getSpaceType()` 中的 switch → 改为从 building 属性读取 `dirtinessType`

**验证**：
```
1. 地图信息页显示的数据与 AppData._data.map_locations.state 一致
2. 修改 AppData._data.map_locations.state.parking.vehicle → 刷新地图 → 显示变化
```

### Batch E：管理员编辑器（2 张票，依赖 D 完成）

#### 票 E1：管理员配置面板

**文件**：`nantang-mobile.html`（新增 overlay）
**做什么**：

1. 新增 `#overlayAdminConfig` overlay，仅 admin 可见入口（在社区页或设置页加按钮）
2. 面板包含三个 Tab：

├── 清洁 NT：脏🔴 [20]  警告🟡 [15]  干净🟢 [5]
├── 物品入库 NT：[2]  出库 NT：[1]  打扫奖励：[10]

├── 每日增长率：卫生间[15]% 厨房[10]% 走廊[8]% 画室[8]% 卧室[5]% 储物[3]% 户外[2]%

```
admin 登录 → 打开管理面板 → 修改清洁定价 → 地图清洁面板显示新价格
```

#### 票 E2：房间物品编辑器

**文件**：`nantang-mobile.html` + `../实景地图移动端代码/app.js`
**做什么**：

1. 在地图房间详情页（`buildRoomDetail` 渲染时），如果是 admin，每个物品旁边显示 ✏️ 🗑 按钮

```

```

## 执行顺序

```
Batch D (可并行)
├── D1: 补全 map_locations 数据结构    (app-data.js)
└── D2: 地图从 map_locations 读取      (地图 app.js)

         ↓ D 完成后，地图所有数据可配置

Batch E (顺序，E2 依赖 E1)
├── E1: 管理员配置面板                (nantang-mobile.html)
└── E2: 房间物品编辑器                (nantang-mobile.html + 地图 app.js)

         ↓ E 完成后，管理员可编辑所有数据

Batch F (可并行)
├── F1: 工作台地图数据面板            (nantang-mobile.html)
└── F2: 快速操作入口                  (nantang-mobile.html)
```
