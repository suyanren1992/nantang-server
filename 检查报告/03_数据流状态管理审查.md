# 全局数据流审查报告

## 发现汇总

| 优先级 | Bug ID | 描述 |
|--------|--------|------|
| **高** | 1.1 | '待提交'/'已完成' 是死状态，任务卡住无法流转 |
| **高** | 1.3 | 撤销请求(confirmWithdraw)写入后无人读取，请求丢失 |
| **高** | 7.1 | editDraft 移除草稿后取消发布导致数据丢失 |
| **高** | 5.1 | confirm 取消时不发回响应，iframe 永久等待 |
| **高** | 跨系统 | confirmSettle 不更新 LEDGER/data.finance，结算无经济影响 |
| **中** | 4.1 | HTML 硬编码 砚仁，切换用户不刷新 UI |
| **中** | 3.1 | renderItemsInTab filter 参数被完全忽略 |
| **中** | 6.1 | _fromQuestHall 状态泄漏，关闭页面时可能错开 questHall |
| **中** | 7.2 | saveDraft/doPublish 的 scope 映射路径不一致 |
| **中** | 7.3 | 草稿在任务大厅中不可见（过滤器遗漏 draft 状态） |
| **低** | 1.2 | 重复结算无防护 |
| **低** | 2.3 | 非账本 tab 下仍可展开账本 |
| **低** | 3.2 | itemRelist 不重置价格 |
| **低** | 5.2 | postMessage 无 origin 校验 |

## 1. TASKS 状态转换

```
doPublish → 进行中 → doSubmit → 待审核 ⇄ confirmReject → 退回修改
                       ↓ approve
                      待结算 → confirmSettle → 已结算 (终端)
```

死状态: '待提交'(无代码产生), '已完成'(无代码产生)

## 2. LEDGER

完全只读——confirmSettle 结算任务不写入 LEDGER，余额不变化。

## 3. MOCK_ITEMS

状态流转正确。但 renderItemsInTab(filter) 的 filter 参数被忽略。

## 4. CURRENT_USER

逻辑代码中一致使用。但 HTML 多处硬编码 '砚仁'（头像、名字、角色）。

## 5. postMessage

confirm 取消时不回发消息。无 origin 校验。

## 6. 全局状态

_fromQuestHall 可能状态泄漏。_itemFold 永不重置。

## 7. DRAFTS

editDraft 移出草稿后取消发布→数据丢失。草稿在任务大厅不可见。
