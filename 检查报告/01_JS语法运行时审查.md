## P0 问题

### 1. app.js 自动初始化在 DOM 就绪前执行 (L900)
```js
if (!_mapContainer) { currentIdx = 4; ... render(); }
```
`render()` 调用 `$()` 大量读取 DOM。在 iframe 中 CSSOM 可能未完全应用时，offsetWidth 返回 0。

**修复**: 包裹在 DOMContentLoaded 中或加 requestAnimationFrame 守卫。

### 2. app.js $() 链式调用无空值守卫 (L57-88)
`$('crumbPath').innerHTML` 等 25 处调用，若元素不存在 → TypeError。

**修复**: `var el=$(x); if(el) el.innerHTML=...`

### 3. doSubmit() submitNote 无空值检查 (L731)
`document.getElementById('submitNote').value.trim()` —— 若 DOM 被意外清除 → TypeError。

**修复**: 先检查再读取。

## P1 问题

### 4. today() 返回 MM-DD (L846)
与 TASKS deadline 的 YYYY-MM-DD 格式不一致，submittedAt 和 deadline 比较会出错。

**修复**: 改为 `new Date().toISOString().slice(0,10)`

### 5. AVATAR_STYLE 重复声明
nantang-mobile.html 和 mobile-bundle.js 各定义一次。

**修复**: 删除一处。

### 6. _post 重复声明
index.html 和 app.js 各定义一次。

**修复**: 用条件判断去重。

### 7-11. 多处 DOM 写入无空值守卫
renderItemsInTab / saveItemForm / expandCard / showMy 等

## 最快止血方案
1. app.js L900 移入 DOMContentLoaded
2. today() 改 YYYY-MM-DD
3. doSubmit 加 submitNote 空值守卫
4. 删除重复 AVATAR_STYLE
