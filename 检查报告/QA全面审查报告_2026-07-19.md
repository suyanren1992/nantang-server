## 🔴 致命 — 直接导致数据丢失或安全漏洞

### CR-1 · `pendingTransactions` 从未持久化

- **文件**：`js/app-data.js` 第 178、185 行
- **验证**：✅ 三轮独立确认。`init()` 未初始化，`_saveShared` 未包含，`_savePrivate` 未包含。
- **现象**：管理员审核的充值/提现申请在页面刷新后无提示全部丢失。用户 NT 已扣除但管理员看不到申请。
- **修复**：
```javascript
// init() 中
if (!this._data.pendingTransactions) this._data.pendingTransactions = [];
// _saveShared() data 对象中加
pendingTransactions: this._data.pendingTransactions
```

### CR-2 · `_saveKey` 静默吞噬 QuotaExceededError — 所有写入失败无提示

- **文件**：`js/app-data.js` 第 36 行
- **验证**：✅ 新增发现（验证-致命代理）。`try { localStorage.setItem(...) } catch(e) {}` — 存储满时所有保存静默失败。
- **影响**：用户继续操作，页面刷新后全部未保存状态丢失。影响所有写入路径。
- **修复**：catch 块中至少 `console.error`；理想情况 toast 提示用户。

### CR-3 · `_syncItemToAppData` NT 先记账、AppData 后保存 — 保存失败时账目漂移

- **文件**：`../实景地图移动端代码/app.js` 第 987、999、1001 行
- **验证**：✅ 新增发现（验证-致命代理）。`NT.earn()` 在 `AppData._save()` 之前执行。
- **影响**：若 `_save()` 失败（存储满），用户 NT 已到账但物品变更丢失。会计不一致。
- **修复**：先 `_save()`，后 `NT.earn()`；或加保存返回值检查。

## 🔴 安全 — 历史报告遗漏、本次确认存在

### SEC-1 · `renderProfile()` 存储型 XSS

- **来源**：用户数据架构审查 §5.1
- **文件**：`nantang-mobile.html` `renderProfile()` 函数
- **现象**：用户名、bio、wallet 等用户可控字段未经转义直接拼入 `innerHTML`。
- **影响**：恶意用户注册 `<script>alert(1)</script>` 作为名字 → 管理员查看其资料时脚本执行。
- **修复**：对所有用户可控字段调用 `esc()` 转义后再拼入 HTML。

### SEC-2 · `NT.transfer()` 无权限检查

- **来源**：用户数据架构审查 §2.4
- **文件**：`js/nt-core.js` `transfer()` 函数
- **现象**：任何人可在 Console 调 `NT.transfer('任意用户', '自己', 99999, 'hack')` 无权限守卫。
- **修复**：加调用者身份校验或要求支付密码。
