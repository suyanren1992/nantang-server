### 票 A1：`_ntTaskId` 桥接加固

**文件**：`nantang-mobile.html`  
**行数**：~30 行  
**问题**：`NT.createTask()` 成功后 taskId 没有写回 AppData 的保证机制。若 taskId 丢失，后续 claim/submit/verify 全链路静默跳过 NT 操作，AppData 状态更新了但金融账目未动，导致账目漂移。

**做什么**：

1. 在 `doPublish()`（约 line 1651）中，`NT.createTask()` 返回值检查从 `if(!ntR)` 改为：
```javascript
if (!ntR) { showToast('NT 余额不足（需 '+nt+' NT）','error'); return; }
if (!ntR.taskId) { showToast('NT 系统异常，请重试','error'); return; }  // ← 新增
data._ntTaskId = ntR.taskId;
```

2. 在 `publishDraft()`（约 line 1668）中，同样加固：
```javascript
var ntR = NT.createTask(CURRENT_USER, name, ntD, 'other');
if (!ntR) { showToast('NT 余额不足（需 '+ntD+' NT）','error'); return; }
if (!ntR.taskId) { showToast('NT 系统异常，请重试','error'); return; }  // ← 新增
t._ntTaskId = ntR.taskId;
```

3. 在 `claimTask()`（约 line 997）、`doSubmit()`（约 line 981）、`reviewTask()`（约 line 1021）、`confirmReject()`（约 line 1036）中，每个 `if(t._ntTaskId && window.NT)` 之后的 NT 调用，加返回值检查：
```javascript
var result = NT.xxx(...);
if (!result) {
  // NT 操作失败，回滚 AppData 状态
  showToast('操作失败，请刷新后重试', 'error');
  return;
}
```

**验证**：
```
1. 发布任务 → Console 检查 AppData._data.tasks[name]._ntTaskId 非空且以 T 开头
2. Console: NT.getTask(taskId) 返回非 null
3. 认领/提交/审核 每个步骤后验证 NT 状态与 AppData 状态一致
```

### 票 A2：任务名唯一性约束

**文件**：`nantang-mobile.html` + `js/app-data.js`  
**行数**：~10 行  
**问题**：`AppData.addTask()` 用 `this._data.tasks[task.name] = task` 静默覆盖同名。俩人发同名任务 → 第二个人任务消失，但 NT 已冻结。

**做什么**：

1. `AppData.addTask()`（app-data.js 约 line 106）加冲突检测：
```javascript
addTask: function(task) {
  if (this._data.tasks[task.name] && this._data.tasks[task.name].status !== 'draft') {
    return { ok: false, error: '任务名「' + task.name + '」已被使用' };
  }
  this._data.tasks[task.name] = task;
  this._saveShared();
  return { ok: true };
},
```

2. `doPublish()`（nantang-mobile.html 约 line 1660）检查返回值：
```javascript
var result = AppData.addTask(data);
if (result && !result.ok) { showToast(result.error, 'error'); return; }
```

3. `saveDraft()` 同理——但草稿允许覆盖同名草稿，所以冲突检测跳 `status === 'draft'` 的情况（已在 addTask 中实现）。

**验证**：
```
1. 用户A发布任务「买菜」
2. 用户B发布任务「买菜」 → 提示「任务名已被使用」
3. 用户A的任务不受影响
```

### 票 A3：`doPublish` 去冗余双写

**文件**：`nantang-mobile.html`  
**行数**：~5 行  
**问题**：`doPublish()` 做了 `TASKS[name] = data` 然后 `AppData.addTask(data)`。前半段直接改对象不触发存盘，如果中间崩溃任务丢失。后半段 `addTask` 里又写了一次同一个对象——冗余。

**做什么**：

在 `doPublish()`（约 line 1659-1660）中：
```javascript
// 之前:
TASKS[name] = data;  // ← 删除这行
AppData.addTask(data);

// 之后:
var addResult = AppData.addTask(data);
if (!addResult.ok) { showToast(addResult.error, 'error'); return; }
```

`AppData.addTask` 内部已做 `this._data.tasks[task.name] = task; this._saveShared()`，不需要外部再写一遍 TASKS。

**验证**：
```
发布任务 → F5 刷新 → 任务仍在任务大厅 → NT.verify() 通过
```

### 票 A4：MGMT_DATA 持久化

**文件**：`../实景地图移动端代码/app.js`  
**行数**：~40 行  
**问题**：`MGMT_DATA`（清洁历史、住宿历史、田地日志、厨房操作记录）存内存，页面刷新即丢失。

**做什么**：

1. 在 `app.js` 顶部（MGMT_DATA 声明后，约 line 438-470）加持久化逻辑：
```javascript
var MGMT_DATA = {
  // ... 现有字段保持不变 ...
  _load: function() {
    try { var s = JSON.parse(localStorage.getItem('nt_mgmt_data')); if (s) { Object.assign(this, s); } } catch(e) {}
  },
  _save: function() {
    var data = { cleanLog: this.cleanLog, stayLog: this.stayLog, fieldLog: this.fieldLog, kitchenLog: this.kitchenLog, myRoom: this.myRoom, myCheckIn: this.myCheckIn, myCheckOut: this.myCheckOut };
    try { localStorage.setItem('nt_mgmt_data', JSON.stringify(data)); } catch(e) {}
  }
};
MGMT_DATA._load();
```

2. 每次修改 MGMT_DATA 的地方加 `MGMT_DATA._save()` 调用：
   - 清洁操作后
   - 入住/退房后
   - 田地操作后
   - 厨房操作后

**验证**：
```
1. 在地图做清洁操作 → F5 刷新 → 清洁历史仍存在
2. 在地图登记入住 → F5 刷新 → 住宿信息仍存在
```

### 票 B1：`Game.getData()` 返回真实 map_locations

**文件**：`nantang-mobile.html`（Game 桥接对象，约 line 693-696） + `js/app-data.js`  
**行数**：~25 行  
**问题**：`Game.getData()` 返回 `{ map_locations: { buildings: [] } }`——永远空数组。地图 `getBuildings()` 读不到数据只能 fallback 到硬编码。

**做什么**：

1. 在 `app-data.js` 的 `init()` 和 `_saveShared()` 中加入 `map_locations` 字段：
```javascript
// init():
if (!this._data.map_locations) this._data.map_locations = { buildings: [], plots: [] };

// _saveShared(): 加到 data 对象中
var data = { tasks:..., camps:..., map_locations: this._data.map_locations, ... };
```

2. 在 `AppData` 上添加写入方法：
```javascript
setMapLocations: function(locations) { this._data.map_locations = locations; this._saveShared(); },
```

3. 修改 `nantang-mobile.html` 的 `Game.getData()`：
```javascript
getData: function() {
  var ad = (window.AppData && AppData._data) ? AppData._data : {};
  return {
    spaces: ad.spaces || {},
    inventory: ad.inventory || {},
    map_locations: ad.map_locations || { buildings: [], plots: [] }
  };
},
```

4. 在 `_seedIfEmpty()` 中写入种子数据（从 `app.js` 的 `HARDCODED_BUILDINGS` 迁移过来）。

**验证**：
```
打开地图页 → Console: Game.getData().map_locations.buildings.length > 0
```

### 票 B2：地图硬编码数据迁移到 AppData

**文件**：`../实景地图移动端代码/app.js`  
**行数**：~100 行  
**问题**：`HARDCODED_BUILDINGS`（建筑/房间结构、清洁标准、田地作物、在场人员）硬编码在地图脚本中，无法通过工作台修改。

**做什么**：

1. 修改 `getBuildings()`（app.js 约 line 26-32），从 `Game.getData().map_locations` 读取，fallback 保留但只做兜底：
```javascript
function getBuildings() {
  var locs = Game.getData().map_locations;
  if (locs && locs.buildings && locs.buildings.length) return locs.buildings;
  return HARDCODED_BUILDINGS;  // 兜底
}
```

2. 在 `nantang-mobile.html` 的 `_seedIfEmpty()` 或初始化脚本中，将 `HARDCODED_BUILDINGS` 复制一份写入 `AppData.setMapLocations()`。

3. 地图中的清洁标准（cleanStandards）、田地数据（plots）、在场人员（peopleOnSite）同样从 `map_locations` 中读取而非硬编码。

**验证**：
```
1. 清除 localStorage → 刷新 → 地图正常显示（从 AppData 种子读取）
2. 修改 AppData._data.map_locations → 地图跟随变化
```
