# 前置票 · 根因修复（已应用，无需执行）

### 票 0-1 · `_saveShared`/`_savePrivate` 加同步模式

**修复**（已直接写入 `js/app-data.js`）：
```javascript
_saveShared: function(immediate) {
  clearTimeout(this._timerS);
  var data = { tasks:..., camps:..., users:..., canteenMenu:..., spaces:...,
    inventory:..., map_locations:..., member_locations:...,
    campRmb: this._data.campRmb,
    pendingTransactions: this._data.pendingTransactions };
  if (immediate) { this._saveKey('nt_app_v2_shared', data); return; }
  var self = this;
  this._timerS = setTimeout(function() { self._saveKey('nt_app_v2_shared', data); }, 200);
},
_savePrivate: function(immediate) { /* 同理 */ },
_save: function(immediate) { this._saveShared(immediate); this._savePrivate(immediate); },
```

**同时解决了**：
- `init()` 中 `campRmb` 初始化缺失（已在 data 对象中包含）
- `init()` 中 `pendingTransactions` 初始化（调用方已设置，此处持久化）
- `_saveShared` data 对象补齐了 campRmb 和 pendingTransactions 字段

**调用方**：
- 正常操作 → `AppData._save()`（防抖）
- `logout()` → `AppData._save(true)`（同步，票 2-4 MI-12）
- `beforeunload` → `AppData._save(true)`（同步，票 3-7）
- `_syncItemToAppData` → `AppData._save(true)` 后再 `NT.earn()`（票 1-3）

### 票 2-4 · MA-4 + MI-8（合并）

**文件**：`js/app-data.js` + `nantang-mobile.html`

**① MA-4**：storage 事件监听（深度合并）：
```javascript
window.addEventListener('storage', function(e) {
  if (e.key === 'nt_app_v2_shared' && e.newValue) {
    try {
      var d = JSON.parse(e.newValue);
      if (d) {
        // ponytail: 深度合并 — 避免浅复制覆盖嵌套对象的并发修改。天花板：两层深度，同级数组最后写入者胜出。
        Object.keys(d).forEach(function(k) {
          if (typeof d[k] === 'object' && !Array.isArray(d[k]) && typeof AppData._data[k] === 'object') {
            Object.assign(AppData._data[k], d[k]);
          } else {
            AppData._data[k] = d[k];
          }
        });
      }
    } catch(ex) {}
    if (typeof refreshUserUI === 'function') refreshUserUI();
  }
});
```

**② MI-8**：`logout()` 加 `safeStorage.removeItem('nt_remembered_user');`
**验证**：两标签页 → A 改 `map_locations.config` → B 收到后自身未保存的字段仍保留

> **注**：原票中的 MA-8（campRmb）和 MI-12（logout flush debounce）已由票 0-1 的 `immediate` 参数覆盖。原票中的 `pendingTransactions` 持久化已由票 0-1 的 `_saveShared` data 对象覆盖。
