# CSS/视觉一致性审查报告

## 涉及文件
- **A**: `nantang-mobile.html` (主应用，内联 `<style>`)
- **B**: `../实景地图移动端代码/theme.css` (地图 iframe 内的主题样式)

---

## 1. 颜色值不统一（高优先级）

| 语义 | nantang-mobile.html | theme.css | 问题 |
|---|---|---|---|
| 警告色 | `#c8892e` | `#c88740` | 两个不同的暖色 |
| 金色 | `#8a6a30` | `#8a6a20` | 两个不同的金色 |
| 卡片边框 | `#d0d9ce` | `#e0d8c8` | 冷灰绿 vs 暖奶油 |
| 页面底色 | `#1d2e24` | `#2a2520` | 墨绿 vs 深棕灰 |
| 卡片底色 | `#fff` | `#fffdf9` | 纯白 vs 暖白 |

**修复建议**: 统一到一套色板。以 theme.css 的 CSS 变量体系为准。

---

## 2. 字体大小缺乏层级体系（中优先级）

**theme.css**: `0.65 / 0.75 / 0.85 / 1.1 / 1.5rem`（5 级）

**nantang-mobile.html**: `.55 .58 .6 .62 .65 .68 .7 .72 .75 .78 .85rem`（11 级）

同一语义的文字有多种字号。

**修复建议**: 缩减到 5-6 个层级，与 theme.css 对齐。

---

## 3. 按钮样式不统一（中优先级）

- **圆角**: `8px` / `10px` / `12px` 三套
- **渐变 vs 纯色**: `.btn-pri` 用渐变，其他主要按钮用纯色
- **Padding**: `14px 24px` / `10px 24px` / `8px 18px` / `10px` / `4px 10px`

**修复建议**: 统一圆角为 `8px`，统一 3 种按钮尺寸。

---

## 4. 间距不一致（低优先级）

gap 值：`4/6/8/10/12px`，padding 从 `4px` 到 `32px` 都有。theme.css 有变量体系但 nantang 完全没有。

---

## 5. 未使用的 CSS 类（低优先级）

`.card-detail` 类定义了但从未使用——所有 card expand 通过内联 style 实现。

---

## 6. 地图 iframe 与主应用视觉风格冲突（高优先级）

场景切换时用户经历：墨绿底→纯白卡→打开地图→深棕灰底→暖白卡。明显的"换皮"感。

双色板对齐到同一色板是最大的视觉一致性需求。

---

## 7. z-index 层级（合理，低优先级）

分配合理但数字跳跃无规律（250/260/300/400/9999）。

---

## 优先级总结

| 优先级 | 问题 | 影响 |
|---|---|---|
| **高** | 地图 iframe 与主应用颜色不一致 | 用户感知到两个产品 |
| **高** | nantang 无 CSS 变量，全靠硬编码 | 维护噩梦 |
| **中** | 字体大小 11 级 vs 5 级 | 视觉节奏混乱 |
| **中** | 按钮圆角/渐变/尺寸不统一 | 操作可预见性差 |
| **低** | `.card-detail` 死类 | 代码清洁度 |
