## 🔴 P0 — 立即修复

### P0-1 ⚠️ NT任务生命周期断裂 — 所有任务奖励永久冻结

**Agent**：Economy Designer（审查点3）· **交叉验证**：Lead Programmer (DATA-1)

**根因**：`claimTask()` 和 `doSubmit()` 只更新UI任务状态（`TASKS[name].status`），从未调用NT核心API `NT.acceptTask()` / `NT.submitTask()`。

```
doPublish → NT.createTask() ✓ (NT冻结到TASK_ESCROW)
claimTask → 只改TASKS ✗ (NT状态仍为pending)
doSubmit  → 只改TASKS ✗ (NT状态仍为pending)
reviewTask(approve) → NT.verifyTask() → 检查status !== 'completed' → 返回null ✗
```

**结果**：发布者的NT已扣除，完成者从未收到NT，`TASK_ESCROW`持续膨胀。**这是资产永久损失。**

**修复**：`nantang-mobile.html`：
- L968 `claimTask()` 末尾加 `if(t._ntTaskId) NT.acceptTask(t._ntTaskId, CURRENT_USER);`
- L959 `doSubmit()` 末尾加 `if(t._ntTaskId) NT.submitTask(t._ntTaskId, {note: sub});`
- `nt-core.js` L189 `verifyTask` 状态检查从 `!== 'completed'` 放宽为允许 `active` 状态的任务直接验证

### P0-2 🔓 approveTx() 零权限检查 — 任意用户可无限铸币

**Agent**：Security Engineer (FINDING 1, CVSS 9.8)

**根因**：`nantang-mobile.html:2010-2028` — `approveTx()` 直接调用 `NT.topUp()` / `NT.cashOut()`，无任何角色检查。任何登录用户可通过浏览器Console直接调用该全局函数。

**修复**: 在函数开头加：
```javascript
var u = getUsers()[CURRENT_USER];
if (!u || u.role !== 'admin') { showToast('权限不足', 'error'); return; }
```

---

### P0-3 🌐 postMessage桥接Origin验证可绕过

**Agent**：Security Engineer (FINDING 3, CVSS 9.0)

**根因**：`nantang-mobile.html:2112` — `e.origin !== 'null'` 允许所有file://协议的本地HTML。任何本地恶意文件可发送`ntEarn`消息铸造NT（有10000/次上限但不限次数）。

**修复**：对file://场景，在app初始化时生成session nonce，通过URL hash传给地图iframe，消息处理中验证nonce。

---

### P0-4 💾 _saveState防抖300ms — 页面关闭前数据丢失

**Agent**：Lead Programmer (DATA-1)

**根因**：`nt-core.js:50-59` — `_saveState()` 使用 `setTimeout(300)` 防抖。关键操作（createTask/verifyTask/earn）后立即关闭标签页会导致NT状态丢失。

**修复**：关键操作路径调用 `_saveState` 时传入 `{immediate: true}` 跳过防抖；或监听 `beforeunload` 强制flush。

---

### P0-5 🔄 approveTx幂等缺失 — 管理员双击导致双重到账

**Agent**：Economy Designer（审查点4）

**根因**：`NT.topUp(userId, amount)` 不检查txId是否已处理。管理员双击审批按钮 → `NT.topUp()` 执行两次 → 用户双倍到账。

**修复**：`NT.topUp()` / `NT.cashOut()` 增加txId参数和去重集合。或在 `approveTx` 中先标记交易为 `approved` 再保存，NT操作前检查状态。

---

### P0-6 🧊 冻结NT计算逻辑双重错误 — 始终显示0

**Agent**：Economy Designer（审查点8）

**根因**：`nantang-mobile.html:1802-1807` — 检查 `tt.status !== 'pending'` 跳过所有active任务 + 检查 `tt.assignee === u`（pending任务无assignee，永远false）。冻结NT始终为0。

**修复**：改为检查 `tt.poster === userId` 且 status 非 verified/settled/cancelled。或在NT核心暴露 `getFrozenBalance(userId)` API。
