## 方案核心决策

| 决策 | 结论 |
|---|---|
| 营队任务 vs 个人委托 | 两套体系。营队任务发出者=营队名，类型=6种标签（主线M/支线S/共创C/隐藏H/身份I/彩蛋E）；个人委托发出者=个人，标签自定（日常/周期…） |
| 工作台区分 | [🏕️营队] Tab 显示营队任务，[📋委托] Tab 显示个人委托。物理隔离，不用筛选芯片 |
| 导航方式 | 7 步独立 overlay，底部分页按钮，← 可回退 |
| 人员选择 | 「虚位以待」空圆圈 → 认领后填入头像。选框弹出紧凑小列表（头像+名字+在线红/绿点，含搜索框） |
| 成为共建人 | 认领任务包 = 身份升级。不是简单分配任务 |
| 分包确认 | 管理员发送 → 共建人逐人逐页确认（防跳过）。分 3 步：预算→日程→任务 |
| 日程 | 日期滚轮（今天高亮，横向滑动）+ 完整时间线切换 |
| 底部栏 | 按角色切换。共建者：👤/📅/📬/＋。冒险者：👤/📅/🥬订餐/🏨住宿 |
| 概览 Tab | 个性化——根据领的任务包类别显示不同区块 |
| 任务厅 | 不在营地内重复。村口已有 overlayQuestHall |

3. **底部栏角色切换是否完整？** 共建者/冒险者/NPC
4. **个性化概览是否覆盖所有角色？** 管理员/后勤/宣传/冒险者
