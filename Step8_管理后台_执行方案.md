## 做什么

数据统计面板 + NT 流向图 + 物品消耗报表 + 打扫/委托统计 + 治理工具（提案/投票）。

## 关键设计决策

- 事件回放和治理系统延后——先做数据统计面板
- 存储分层在 Step 2 已做（照片→IndexedDB），此处不重复

## 面板清单

| 面板 | 核心数据 |
|------|---------|
| 💰 NT 总览 | 总池/已发行/流通中、本月流入/流出分类统计 |
| 📊 NT 流向图 | 桑基图：发行→劳动/记录→用户→消费→回收 |
| 👤 各人 NT 明细 | 当前余额/本月收入/本月支出/净变动，点击展开流水 |
| 📦 物品消耗报表 | 当前库存/本月入库/消耗/过期/浪费率、按分类消耗比例 |
| 🔍 物品生命周期 | 点击物品→放入→评论→消耗→从放入到消耗的天数 |
| 🧹 打扫统计 | 各空间完成率、各人打扫贡献排行 |
| 📋 委托统计 | 发布数/完成率/最活跃发布人/接单人 |
| 🗳️ 提案系统 | 发起提案、投票（赞成/反对/弃权）、通过阈值 |
| 🔥 空间热力图 | `ponytail:` MVP 3-5 人社区肉眼可判断活跃度，热力图属顺手优化。P2 延后或删除。 |

## 月底清算面板

月底最后一天管理员操作：
1. 系统汇总本月虚拟账本 → 计算每人净 NT 头寸
2. 生成 P2P 清算方案 → 公共池债权优先兑付 → 剩余按 P2P 分配
3. 管理员确认 → 导出清算单 → 各人钱包执行转账
4. 账本归档 → 新月份开始

## 公共贡献池面板

显示：贡献值余额 / 本月磨损收入 / 冻结回收 / 陪审团支出 / 新手支出 / 待清算债权
