## 问题诊断

当前 `nantang-mobile.html` 的「表面化残缺」根因：

| 症状 | 根因 |
|------|------|
| 田间管理不完整 | 只有数据展示（`renderFieldPlots`），没有记录农活→被发现→结算的流 |
| 住宿管理不完整 | 只有床位展示，入住/退房只是切换状态，没有费用自动计 |
| 大扫除不完整 | 有排班日期和脏污度，但打扫→拍照→校核→NT 闭环不存在 |
| 卫生管理不完整 | 脏污度自动增长有，但触发打扫→完成→确认的链是断的 |
| 新人引导不完整 | 任务清单存在，但每步完成后的即时反馈→自动下一步链条缺失 |
| 卡片室猜卡不完整 | **模块写了但没接入**；而且它依赖 `#overlayCardRoom` 这个 HTML 容器 |

**本质：当前代码是数据浏览器，不是工作流引擎。每块都只做了「读」和「浅层写」，没有把「动作→验证→奖励→记录」四个步骤串起来。**

---

## 架构方案

### 原则

1. **不动 `nantang-mobile.html`**——它继续跑生产，改坏了得不偿失
2. **在 `nantang-mobile/` 目录完成合并**——把 monolithic 版本缺的模块逐步加进来，直到 `index.html` 能完全替代 `nantang-mobile.html`
3. **每个模块独立可测**——加一个模块，`index.html` 就能多跑一个功能
4. **卡片室优先接入**——因为它是产品说明书的核心闭环，而且代码已经写好了

### 模块接口约定

每个模块：
```javascript
// 1. 初始化（可选）
window.ModuleName = {
  init: function() { /* 绑定事件、注入 DOM */ },
  // 2. 入口（对外暴露的函数，供其他模块或 HTML onclick 调用）
  open: function() { /* 打开此模块的主界面 */ },
  close: function() { /* 关闭 */ }
};
```

模块之间通过以下方式通信：
- **读/写数据**：`window.AppData._data` → `AppData._saveShared()`
- **写 NT**：`window.NT.earn()` / `NT.spend()`
- **事件通知**：`EventBus.emit('user:updated', data)` / `EventBus.on('user:updated', fn)`
- **UI 刷新**：`refreshUserUI()` （全局函数）

### 步骤 3：打通「打扫闭环」（2-3 小时）

这是第一个完整的工作流闭环。产品说明书的核心是：

```
行为发生 → 线索记录 → 观察者发现 → 猜测身份 → 做事者确认 → 双方得 NT
```

当前状态：
- 卡片室猜卡已有（步骤 2 接入后）
- 劳动上报已有（`openSelfReport` → 校核队列）
- **缺失**：从地图空间直接触发「我做了这个」

**要做的事**：

1. **地图房间详情页加「我要打扫」按钮**
   - 在 `buildRoomDetail()` 中，加一个按钮：「🧹 标记已打扫」
   - 点击 → 弹出拍照表单（前后对比照，先用 file input 模拟）
   - 提交 → 写 verifications + journal，脏污度重置为 0

2. **地图全貌页的「快捷入口」改接真实流程**
   - 「放/取物品」→ 调冰箱物品录入（已有部分逻辑）
   - 「记录农活」→ 调田间劳动记录 → 写入校核队列
   - 「快速打扫」→ 同 1

3. **校核闭环**
   - 校核通知已经在 `AppData._data.pendingVerifications` 中
   - 地图全貌页的校核入口 `_openVerificationPanel()` 需要工作
   - 校核通过 → NT 到账 + 记录公开

**验证**：从「我看到厨房脏了」到「我打扫了厨房」到「有人校核通过」到「NT 到账」全程跑通。

## 风险与验证策略

| 风险 | 缓解 |
|------|------|
| 拆分时破坏全局变量依赖 | 每个模块提取后立即在 `index.html` 中加载测试 |
| CSS 差异导致视觉不一致 | `main.css` 从 `nantang-mobile.html` 的 `<style>` 块直接复制 |
| `ui-cardroom.js` 有未发现的 bug | 步骤 0 先独立验证 |
| 两个 `app-data.js` 版本不同 | 比较差异，统一用 `nantang-mobile/js/app-data.js`（587 行，更新） |

## 关键差异对比

`nantang-mobile/js/app-data.js` (587 行) vs `js/app-data.js` (542 行)：
- `nantang-mobile/` 版本多了：`addVerification()`, `addJournal()`, `flipPresence()`, `checkPresenceReminder()`, `addAnnouncement()`, `_saveShared()` 含更多字段
- **必须用 `nantang-mobile/` 版本**作为统一数据层

类似地，`nantang-mobile/js/core.js` (1216 行) 包含了从 monolithic 提取但扩展过的版本。
