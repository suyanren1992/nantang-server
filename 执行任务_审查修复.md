# 执行窗口 · 审查修复任务

> 游戏设计师审查发现 4 个代码层问题。逐一修复。

---

## 任务 1：删除 mobile-game.css 中的重复 `:root` 块

**问题：** `mobile-game.css` 第 9-44 行有自己的 `:root{}`，重新定义了 `--g-bg`、`--g-gold` 等变量。和 `variables.css` 冲突。

**修法：**
1. 打开 `css/mobile-game.css`
2. 找到 `:root{` 开头的块（约第 9-44 行）
3. 删除整个 `:root{}` 块
4. 保存

**验证：** 打开 `nantang-mobile.html`，所有颜色正常。Console 无 CSS 变量未定义警告。

---

## 任务 2：删除重复的地图文件

**问题：** `js/map-app.js` 和 `css/map-theme.css` 是地图端的副本（1429 行 × 2）。改一端忘记同步另一端 = bug。

**修法：**
1. 确认 `nantang-mobile.html` 的 `<script>` 标签没有引用 `js/map-app.js`
2. 删除 `js/map-app.js`
3. 删除 `css/map-theme.css`

**如果 HTML 中确实引用了这两个文件：** 不删，改为在文件头部加注释 `/* ⚠️ 副本，修改请同步 ../实景地图移动端代码/ */`。

**验证：** 打开 `nantang-mobile.html`，无 404 报错。

---

## 任务 3：统一 window.Game 对象

**问题：** `nantang-mobile-ui.js:36` 定义了 `window.Game`，HTML 内联脚本可能也有 Game mock。两处定义逻辑上应该是一个对象，但现在是两个——改一处另一处不动。

**修法：**
1. 在 `js/nantang-mobile-ui.js` 中搜索 `window.Game = {`
2. 确认这是**唯一**的定义位置
3. 搜索 `nantang-mobile.html` 中是否有 `window.Game` 或 `Game = {`
4. 如果有 → 删掉 HTML 中的，只保留 JS 中的
5. 在 JS 的 `window.Game` 对象中补充缺失的方法（对照 BRIDGE.md 的接口契约）

**验证：** 全局搜索 `window.Game`，整个项目只有一处定义。

---

## 任务 4：清理 nantang-mobile-ui.js 中不再使用的旧函数

**问题：** 旧版 NM.renderWorkspace() 被替换后，以下函数可能成为死代码。确认后删除。

**检查清单：**
- `NM._wf` / `NM._wsf` / `NM._wst` / `NM._rwt` — 旧版工作台筛选函数
- `NM._settle` — 旧版结算函数（如已有 `NM._doSettle` 替代）

**修法：** 搜索每个函数名在 `nantang-mobile-ui.js` 和 `nantang-mobile.html` 中的引用次数。如果只在定义处出现一次 → 删除。

**验证：** 删除后打开页面，工作台/任务列表/结算功能正常。

---

## 执行顺序

```
1 → 2 → 3 → 4
```

每做完一项，刷新页面确认无报错，再做下一项。
