# 南塘云村 · 地图与功能系统深度审查报告

> **日期**：2026-07-20
> **审查范围**：地图模块（app.js 2081行）、住宿系统（双路径）、田间系统、信箱、设置
> **审查方式**：三个专项 Agent 全文扫描

---

## 一、根因总述

**地图模块 `app.js`（2081 行）是一个独立运行的硬编码演示系统。** 它有自己的数据（`HARDCODED_BUILDINGS`）、自己的配置（`_defaultConfig()` 60+ 个魔法数字）、自己的存储（`MGMT_DATA` 写入 localStorage 而非 AppData）。它和 AppData、API、NT 系统之间几乎完全没有数据连接。

你在页面上看到的一切——在线人数、整洁状态、入住信息、代收费用、土地作物——全部是**硬编码字符串**，从不更新。

---

## 二、发现了什么

### 2.1 硬编码数据（60+ 处）

**HARDCODED_BUILDINGS 中的全部状态字符串：**

| 显示内容 | 实际代码 | 应该从哪里读 |
|---------|---------|-------------|
| 在线人数 "👤0人在线" | `'👤无人'` 硬编码 | `AppData._data.map_locations.people_on_site` |
| 整洁程度 "🟢整洁" | `'🟢整洁'` 硬编码 | `AppData._data.cleaning.spaces` |
| 入住信息 "王五入住·5天·150NT待付" | 硬编码字符串 | `AppData._data.map_locations.accommodations` |
| 待收费用 "💰待收240NT" | 硬编码 | NT 账本 |
| 田地作物 "A区番茄" | `crops:[]` 空数组 | `AppData._data.map_locations.plots` |
| 房间数量 "3层·15间" | 硬编码 | 动态计算 |
| 空间数量 "10个空间" | 硬编码 | `getBuildings().length` |

**app.js stats 栏——页面加载后永远是 "—"：**

7 个统计芯片（`ubStatDate`、`ubStatDirt`、`ubStatPeople`、`ubStatStay`、`ubFarmStat`、`ubStatTasks`、`ubNewbieChip`）只被 `refreshUserUI()` 更新，但 `app.js` 从不调用 `refreshUserUI()`。地图打开后这些永远显示占位符。

### 2.2 住宿系统——四套数据，三套入口，选不了床位

**问题场景**：用户在手机上打开地图 → 点击"大地书房" → 上滑到 2F → 看到 dorm101-dorm106 → 点击 → 只显示一个空白详情面板，没有任何预订按钮。

**根因**：住宿数据有**四套互不同步的数据源**，住宿入口有**三个互相独立的界面**：

| 数据源 | 位置 | 房间ID |
|--------|------|--------|
| `HARDCODED_BUILDINGS` dorm | `app.js:21` | dorm101-106 |
| `map_locations.accommodations` | `app-data.js:242` | dorm101-106（字段不同） |
| `INN_ROOMS` | `ui-phase4.js:2` | zhu_a, mei_a, lan_a-d, ju_a（完全不同） |
| `map_locations.state.room_items` | `app-data.js:265` | 物品记录 |

| 入口 | 触发方式 | 能否选床位 |
|------|---------|:---:|
| 地图 → 书房 2F → 点击宿舍 | `selectRoom()` → `buildRoomDetail()` | ❌ 只读，无表单 |
| 营地 → 🏨 住宿按钮 | `openInn()` → `showInnDetail()` | ✅ 但用不同的房间系统 |
| 地图 → 管理面板 → 住宿 | `_showStaySheet()` | ✅ 但格式不兼容 |

**`buildRoomDetail()` 缺 return 语句（已在上轮审计发现）**：第 677-739 行构建了 60+ 行 HTML，但函数末尾没有 `return body;`，导致所有宿舍房间点击后显示空白。

### 2.3 田间管理系统——旧格式数据，空数组渲染

**问题场景**：地图点"田地A-E" → 5 块地全部显示空白，没有作物信息。

**根因**：代码有两条数据读取路径：

| 路径 | 函数 | 读取来源 | 结果 |
|------|------|---------|------|
| 地图渲染 | `curRooms()` | `HARDCODED_BUILDINGS[field].plots` | `crops:[]` 空 → 空白 |
| 管理面板 | `getPlots()` | `AppData._data.map_locations.plots` | ✅ 有种子数据 |

种子数据（`app-data.js:257`）使用旧格式（`crop:'番茄'` 直接挂在 plot 上），但新格式要求 `crops:[]` 数组。没有迁移代码。

### 2.4 地图底栏设置——按钮不存在

地图底栏（`bottom-carousel`）只包含建筑轮播和模糊遮罩。**没有设置按钮。** HTML 上没有，JS 里也没有。你看到可能是早先版本的设计稿或截图，当前代码里根本没有实现。

### 2.5 信箱——按钮存在，但不更新

地图顶栏右侧有 📬 按钮（HTML line 416），调用 `_toggleInbox()`。函数存在且能打开，但：
- 信箱角标（未读数）只被 `refreshUserUI()` 更新
- `app.js` 从不调用 `refreshUserUI()`
- 角标永远显示 0 或初始状态

### 2.6 管理员配置按钮——有按钮，从不显示

HTML line 406 有 ⚙️ 按钮（`ubAdminCfgBtn`），但 `display:none` 默认隐藏。只有 `refreshUserUI()` 会对管理员显示它，但 `app.js` 从不调用 `refreshUserUI()`。管理员也看不到。

### 2.7 社区动态——不在本次审计范围（非 app.js 问题）

社区动态（activity feed）在 `data.js` 的 `renderCommunityHub` 中渲染，读取 `AppData._data.activity_log`。如果它点不开，问题不在 app.js。需要单独排查该处的 onclick 绑定。

## 三、架构问题总结

```
┌─────────────────────────────────────────────────────────┐
│                    app.js（2081 行）                      │
│  ┌─────────────────────────────────────────────────┐    │
│  │  HARDCODED_BUILDINGS（10个建筑，全部硬编码）       │    │
│  │  _defaultConfig()（60+ 魔法数字）                 │    │
│  │  MGMT_DATA → localStorage（不经过 AppData）       │    │
│  └─────────────────────────────────────────────────┘    │
│                         │                               │
│                    ❌ 无数据连接                         │
│                         │                               │
│  ┌─────────────────────────────────────────────────┐    │
│  │  AppData / API / NT 系统（你之前建的所有后端）     │    │
│  │  map_locations（在线人数、住宿、整洁度、作物）      │    │
│  │  NT ledger（费用、待收）                           │    │
│  │  activity_log（社区动态）                          │    │
│  └─────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────┘
```

**app.js 和 AppData/API 是完全隔离的两个世界。** 地图上看到的一切都不会变，因为它不读 AppData。你在管理面板、AppData、API 里改的任何数据，地图上都看不到。

---

## 四、修复方案

不是"修几个 bug"——需要**把 app.js 的数据层接到 AppData 上**。具体：

1. **在线人数**：`renderInfoPage()` 从 `AppData._data.map_locations.people_on_site` 读取而非硬编码
2. **整洁状态**：渲染函数从 `AppData._data.cleaning.spaces[buildingId].dirtiness` 读取而非硬编码 summary
3. **住宿**：`buildRoomDetail()` 加上预订表单，数据统一读写 `map_locations.accommodations`
4. **田间**：`curRooms()` 改用 `getPlots()` 统一数据源
5. **stats 栏**：`app.js` 的 init/render 中调用 `refreshUserUI()` 或自行更新 stat 芯片
6. **信箱角标**：同上
7. **管理员按钮**：同上
