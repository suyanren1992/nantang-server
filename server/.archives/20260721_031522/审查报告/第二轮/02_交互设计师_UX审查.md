# 第二轮审查 · 交互设计师 · UX 审查

> 2026-07-21 · Agent: ux-designer
> 审查范围：卡片室 tab、任务大厅二级 chip、贡献路径选择器、导航流、管理后台入口

---

## 1. 卡片室「猜/确认」双 tab

**问题**：tab 标签用动作命名（「🃏 猜」「✓ 确认(3)」），用户看 tab 想知道「里面有什么内容」而非「做什么操作」。两个 tab 内容本质完全不同但共用"卡片室"标题。

**建议**：
- 改为内容描述型标签：「🃏 匿名好事」「✓ 待验证(3)」
- 加副标题：「谁做了这件事？猜对有奖」「确认社区成员的劳动」

## 2. 任务大厅二级 chip 行

**问题**：条件性出现的 UI 元素发现率极低。用户点击 chip 后视线已下移到卡片列表。

**建议**：二级 chip 始终存在但灰显（opacity:0.4+pointer-events:none），社区激活时恢复。非在地成员缺少的 chip 灰显而非消失。

## 3. 贡献路径选择器（报备 vs 委托）

**问题**：「报备」「委托」是正式用语，对普通玩家不友好。

**建议**：
- 「✅ 我做完了，求验证」「📝 我需要帮忙」
- 固定推荐「我做完了」，不根据上下文动态变化

## 4. 导航流

**问题**：轮播缺少小圆点指示器；卡片室 overlay 无来源痕迹，← 退出不知回哪。

**建议**：加 3 个小圆点；卡片室标题栏加来源痕迹（如「←（地图）🃏 卡片室」）

## 5. 管理后台入口

**问题**：在「我的」tab 内 chip，两层嵌套太深，无发现有审核任务的信号。

**建议**：底部导航「我的」tab 加红点徽章；管理后台 chip 放在「我的」页面顶部独立行。
