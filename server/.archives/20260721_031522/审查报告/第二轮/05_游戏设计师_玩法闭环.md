# 第二轮审查 · 游戏设计师 · 玩法闭环

> 2026-07-21 · Agent: game-designer
> 审查范围：三层空间玩家旅程、双路径动机、周期任务游戏性、管理员权力边界、整体正循环

---

## 1. 新玩家引导 — 未定义引导顺序

三层空间并列展示但无 onboarding funnel。建议：任务大厅→校核室→卡片室（按内容可得性排序）。

## 2. 卡片室冷启动 — 影响被低估

0 张卡片 = 信任侵蚀，不只是"低影响"。建议：占位卡 + 前3报备免多人确认 + 系统种植示范卡。

## 3. 校核室确认动机 — 陌生人愿意确认吗？

纯经济激励（1-2 NT）不覆盖社交成本。建议：「亲眼见证」vs「信任确认」两级 + 验证者徽章。

## 4. 委派 vs 报备 — 动机不对称

玩家总选"别人出钱"的报备。建议：委派=创造参与机会（赞助者框架），报备=主动劳动（贡献者框架）。

## 5. 72h 无人接 — 情感影响被忽视

"无人认领"是三重伤：需求不重要、NT不够、社区不活跃。建议：48h 提醒 + 过期后给出替代 CTA。

## 6. 周期任务重复性 — 第7天疲劳

每天相同模板 → 背景噪音。建议：子描述随机轮换 + 连续领取加成 + 卡片室金边联动。

## 7. 多槽位 — 与门还是或门？

当前有 doSubmit bug（第一人提交锁死）。建议：或门——每槽位独立结算。

## 8. 管理员审核延迟 — 无 SLA

建议：新人前 3 任务 4h 自动通过（渐进信任）+ 显示预估审核时间。

## 9. 管理员 self-claim — 未防护

建议：admin/builder 不能认领 poster='社区' 的任务 + reward 硬上限 20 NT。

## 10. 整体正循环 — 飞轮薄弱在确认者动机

SDT 分析：Autonomy ✅ / Competence ⚠️ / Relatedness ⚠️
建议：校核室验证者徽章 + 「我的贡献全景」面板 + 链式信任（A→B→C 而非 A↔B）
