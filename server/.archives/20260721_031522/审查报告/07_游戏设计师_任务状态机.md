## 发现清单（13 项）

### ISSUE 1 [CRITICAL] — 多槽任务 escrow 退款只退 reward × 1

`nt-core.js:124, 229, 247`

createTask 冻结 `reward × slots`。cancelTask 和 verifyTask rejected 只退 `t.reward`。3 槽任务冻结 300 NT，取消只退 100 NT，200 NT 永久锁在 TASK_ESCROW。

### ISSUE 2 [CRITICAL] — disputed 是死胡同

`nt-core.js:192-266`

一旦 disputed，不能重验、取消、重接。escrow 可能释放也可能没释放，无恢复路径。

### ISSUE 3 [CRITICAL] — disputeTask 无状态守卫、无权限

`nt-core.js:261-266`

无状态检查（settled 任务也能 disputed）。无权限检查（任何人都能 disputed 任何任务）。无 reason 参数。

### ISSUE 4 [HIGH] — R9-4 修复未执行

`nt-core.js:225`

verifyTask rejected 仍设 `disputed` 而非 `active`。

### ISSUE 5 [HIGH] — 三套不兼容状态词汇

| 层 | 状态集 |
|----|--------|
| NT 核 | pending/active/completed/verified/settled/cancelled/disputed (英文) |
| 服务端 | 进行中/待提交/待审核/退回修改/待结算/已结算/已取消/已争议 (中文A) |
| UI | 进行中/待提交/待审核/退回修改/待结算/已结算/已完成/draft (中文B) |

### ISSUE 6 [HIGH] — 已取消/已争议任务在 UI 消失

`core.js:270-272, 1026-1029`

filterQuests 和 renderMyTasks 不处理已取消/已争议，这两个状态的任务完全不可见。

### ISSUE 7 [HIGH] — toggleQuestCard 无已取消/已争议按钮

`core.js:374-396`

操作按钮 if/else 链不覆盖已取消/已争议，展开卡空白。

### ISSUE 8 [MEDIUM] — cancel_task 不阻止已争议

`nt.py:457`

已争议不在防护列表里，可取消——escrow 可能在争议中就被释放。
