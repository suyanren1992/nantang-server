# 游戏设计师审查 · 任务状态机

> 审查范围：任务生命周期、状态机、营地、多人任务
> 文件：server/routes/nt.py, server/routes/tasks.py, nantang-mobile/js/nt-core.js, nantang-mobile/js/data.js

## 发现 15 项

### 1. 🔴 unclaimTask 双花 — 退款给 poster 后任务可被重新接取
**文件：** nt-core.js:273-282, server/routes/nt.py（无服务端 unclaim）
`unclaimTask` 退还 `TASK_ESCROW -= t.reward` 给 poster 并将任务重置为 `pending`。重新 accept + verify 再次扣减 TASK_ESCROW → 凭空印钞。
**修复：** (a) 不从 escrow 退款，仅清空 assignee；或 (b) 退款后设标志阻止重接。需加服务端 unclaim 端点。

### 2. 🔴 update_task (PUT /api/tasks/{id}) 绕过所有状态守卫
**文件：** server/routes/tasks.py:95-126
可直接将 `task.status` 设为任意值，无状态转换验证。跳过 submit/verify → 破坏会计等式。
**修复：** 加状态转换白名单。禁止直接设状态。

### 3. 🔴 task.escrow_amount 从不递减 → 双重退款
**文件：** server/routes/nt.py:293, server/routes/nt.py:385-390, server/routes/tasks.py:143-151
创建时设一次，verify/cancel/delete 后从不更新 → 读陈旧值 → 双重退款 → pool.task_escrow 变负。
**修复：** 每次 escrow 释放后设 `task.escrow_amount = 0`。cancel/delete 中屏蔽已取消状态。

### 4. 🔴 多人任务仅 UI 假象 — DB/NT 层仅单 assignee
**文件：** server/routes/nt.py:313, nt-core.js, models.py:46
UI 管理 `claimants[]` 数组、检查 slots、等待所有人提交。但 NTTask.assignee 是单个 String。第二个人 UI 显示已领取但无 NT 状态。
**修复：** 全部实现多 assignee，或强制 slots=1 直到完成。

### 5. 🔴 拒绝后重提交断裂 — nt-core verifyTask(rejected) 设 disputed 阻止重提交
**文件：** nt-core.js:198-241, data.js:381-398
confirmReject 调 `NT.verifyTask(taskId, user, false, reason)` → 设 `t.status = 'disputed'`。resubmit 时 `NT.submitTask` 要求 `status === 'active'` → 失败。服务端拒绝直接设 `已取消`。
**修复：** nt-core 拒绝不设 disputed，设回可重提交状态。服务端加 `退回修改` 状态。

### 6. 🔴 server verify_task 允许在未提交状态下验证
**文件：** server/routes/nt.py:333
guard 允许 `"进行中"`。admin 可在无 evidence 情况下批付 assignee。
**修复：** 从允许列表移除 `"进行中"`。

### 7. 🟡 客户端/服务端状态词汇不兼容 → 三个独立状态机
**文件：** server/nt.py（中文：进行中/待审核/待结算/已取消/已争议），nt-core.js（英文：pending/active/completed/verified/settled/cancelled/disputed），data.js（另一套中文：进行中/待审核/待结算/已取消/退回修改）
三套并行。nt-core.js 无 `退回修改`。服务端无 `退回修改`。
**修复：** 统一为中文。nt-core.js 改用服务端词汇。

### 8. 🟡 cancel_task 无退款 ledger 记录
**文件：** server/routes/nt.py:375-393
退还 escrow 但从不调 `_add_ledger`。资金流动不可见。
**修复：** 退款后加 `_add_ledger`。

### 9. 🟡 无服务端 resolveDispute 端点 → 争议任务永久冻结
**文件：** server/routes/nt.py:410-420（仅有 dispute_task），nt-core.js:284-299（有 resolveDispute）
dispute_task 设状态为 `已争议` 但无解析端点。任务和 escrow 永久冻结。
**修复：** 加 `POST /api/nt/tasks/{id}/resolve`，admin 权限。

### 10. 🟡 delete_task 可对已验证任务双重退款
**文件：** server/routes/tasks.py:129-154
guard `task.escrow_amount > 0 and task.status != "已结算"` 允许 `待结算` → 读陈旧 escrow_amount → 双重退款。
**修复：** `delete_task` 屏蔽 `待结算` 和 `已取消`。

### 11. 🟡 nt.py 和 tasks.py 有两个 create-task 端点，字段集不同
**文件：** server/routes/nt.py:275-300, server/routes/tasks.py:63-92
nt.py 版本用 `CreateTaskRequest(title, reward, category)` — 无 slots、reviewer、deadline。tasks.py 用完整字段。两个都扣款。
**修复：** 废弃 `POST /api/nt/tasks` 重定向到 `POST /api/tasks`。

### 12. 🟡 getFrozenBalance 错误排除 disputed → 冻结余额低估
**文件：** nt-core.js:606-615
直接 disputed 的任务仍有 escrow 冻结，但被排除。冻结余额被低估。
**修复：** 移除 `'disputed'` 从排除列表，或检查 escrow 是否实际释放。

### 13. 🟢 nt.py earn 端有不可达的 camp pool 代码路径
**文件：** server/routes/nt.py:153
`hasattr(req, 'pool')` — `EarnSpendRequest` 无 `pool` 字段 → 始终 False。死代码。
**修复：** 删掉或补 `pool` 字段。

### 14. 🟢 cancel_task 非幂等 → 可取消已取消任务
**文件：** server/routes/nt.py:375-393
仅屏蔽 `已结算`/`待结算`。`已取消` 和 `已争议` 可通过 → 双重退款（配合 #3）。
**修复：** 屏蔽列表加 `已取消` 和 `已争议`。

### 15. 🟢 rejectCount ≥3 自动关闭 → 客户端/服务端不同步
**文件：** data.js:384-391
客户端调 `NT.verifyTask`（客户端） → 设 disputed。服务端任务从未更新。
**修复：** 自动关闭路径也调服务端 `POST /api/nt/tasks/{id}/verify?approved=false`。
