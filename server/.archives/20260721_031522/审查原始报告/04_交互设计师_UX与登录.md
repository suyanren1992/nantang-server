# 交互设计师审查 · UX 与登录

> 审查范围：登录/注册、错误反馈、空状态、导航
> 文件：nantang-mobile/index.html, nantang-mobile/js/core.js, nantang-mobile/js/api.js

## 发现 17 项

### 1. 🟡 注册错误 UI 元素是死的 — regError div 从未填充
**文件：** index.html:50, core.js:746
`#regError` div 存在于 HTML 但从未被写入。错误用 `showToast` 而非内联显示。
**修复：** 错误写入 `#regError` 并显示，输入聚焦时清除。

### 2. 🔴 邀请码输入是死代码 — 承诺"解锁身份"但从不生效
**文件：** index.html:49, core.js:742-768
`#regInviteCode` 输入框被渲染但 `enterVillage()` 从不读取或传递。
**修复：** 读取值并传给 API 和 registerUser。

### 3. 🟡 登录无名字空值校验 → 令人困惑的错误
**文件：** core.js:770-771
只检查密码空值。空名字发到服务端 → 不友好错误。
**修复：** 加 `if(!ln){showToast('请输入名字','error');return}`

### 4. 🟡 加载 toast 2.5 秒消失但超时 5 秒 → 中间空白无反馈
**文件：** index.html:1262, data.js:483-501
showToast 2.5 秒自消失，silentRefresh 超时 5 秒。3-5 秒间空白页无反馈。
**修复：** 加载状态用持久元素替代自消失 toast。

### 5. 🟡 autoLogin 超时后成功回调仍触发 → 超时警告后突然登录
**文件：** index.html:1263-1295
silentRefresh 在 5 秒后返回 → 超时已显示警告 → 回调仍调用 setCurrentUser。
**修复：** 加 `_refreshTimedOut` 标志。

### 6. 🟡 autoLogin no_token 和 server_error 均显示通用"未连接"错误
**文件：** index.html:1269-1271
`'no_token'`（首次访问）→ 误报"服务器未连接"。`'server_error'`（500）→ 同样误报。
**修复：** `'no_token'` 静默处理。`'server_error'` 显示不同提示。

### 7. 🟢 登录页 safe-area 正常（无 bug）
`.login-page { padding-bottom: env(safe-area-inset-bottom, 20px); }` 正确应用。

### 8. 🔴 inbox 内联 style `top:50px` 覆盖 CSS safe-area → iPhone 刘海遮挡
**文件：** index.html:692, css/theme.css:590
`_toggleInbox()` 设 `panel.style.top = '50px'` → 覆盖 CSS `calc(50px + env(safe-area-inset-top))`。
**修复：** JS inline style 改为含 safe-area 的值。

### 9. 🟡 收件箱面板 DOM 泄漏 — 打开/关闭循环产生孤立节点
**文件：** index.html:704-706
`closeInbox()` 设 `display:none` 但不移除。重复开/关产生孤立 `<div>`。
**修复：** 复用同一面板或关闭时 `remove()`。

### 10. 🟢 静态 `#myInboxPanel` 从未使用 — 死 HTML
**文件：** index.html:144
存在于 myPage 但从未被填充或显示。实际收件箱是动态创建的 `#inboxPanel`。
**修复：** 删掉或复用。

### 11. 🟢 `_computeInboxMessages()` 每次渲染调两次 —— 但在 R6 中已修复缓存

### 12. 🟡 空状态"暂无待领取任务"中的"发布委托"不可点击
**文件：** core.js:274
文本"「+ 发布委托」"无 onclick。真正按钮在页面底部，视觉上分离。
**修复：** 包裹在带 onclick 的 `<span>` 中。

### 13. 🟢 deleteDraft 确认弹窗正常（无 bug）

### 14. 🟡 closeMyPage 设 hash='' 产生历史条目 → 后退按钮破坏导航
**文件：** core.js:1229,1339-1344
`window.location.hash = ''` 推新历史条目。无 hashchange 监听器 → 后退键失效。
**修复：** 用 `history.replaceState` 替代 `location.hash`。

### 15. 🟢 touch-action:manipulation 仅匹配 [onclick] → 遗漏 addEventListener
**文件：** index.html:8, css/theme.css:592
`[onclick]` 选择器遗漏编程式事件监听器。
**修复：** 改为 `button, a, [role="button"], [onclick]`。

### 16. 🟢 头像选择器无边界夹持 → 小屏溢出
**文件：** core.js:681
`left: r.left - 120`，280px 宽，小屏左边溢出。
**修复：** `Math.max(8, Math.min(r.left - 120, window.innerWidth - 288))`

### 17. 🟢 注册 vs 个人资料头像选择器样式不一致
**文件：** core.js:681 vs core.js:1161
两个完全不同的视觉风格。可能有意为之（暗色 vs 亮色背景），但应加注释。

### 18. 🟢 硬编码中文字符串（观察 — 非 bug）
所有用户可见字符串均为内联中文文字。可运行但不利于 i18n。
