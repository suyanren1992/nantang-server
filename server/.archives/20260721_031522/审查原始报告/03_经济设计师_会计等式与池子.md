# 经济设计师审查 · 会计等式与池子

> 审查范围：会计等式、池子、托管、住宿扣费
> 文件：server/routes/nt.py, server/models.py, nantang-mobile/js/nt-core.js, nantang-mobile/js/app-data.js

## 发现 11 项

### 1. 🔴 earn camp-scope 向 ledger 写入错误池 → 会计等式破坏
**文件：** server/routes/nt.py:168
`scope == "camp"` 时从 `pool.camp_balance` 扣款，但 ledger 记录 `from_user=None`（非 `"camp_pool"`）。verify 从 ledger 推导 camp_balance → camp 减少不感知 → 会计等式破坏。
**修复：** `from_user = "camp_pool" if req.scope == "camp" else "community_pool"`

### 2. 🔴 住宿 overdueNT 几何级增长 → 第 N 天欠费 = 150 × 2^N
**文件：** app-data.js:432
`totalDue` 已含旧 `userOverdue`，`unpaid`（也含旧欠费）又被 `+=` 累加 → 指数增长。第3天应欠 450，实际欠 1050。
**修复：** `user.overdueNT = unpaid;`（非 `+=`）

### 3. 🟡 服务端 verify(approved) 不增加 contribution_value
**文件：** server/routes/nt.py:342-343
任务完成（最应得 CV 的行为）只增 XP 不增 CV。客户端已增 CV。不一致。
**修复：** 加 `assignee.contribution_value += task.reward`

### 4. 🟡 verify 函数从 ledger 推导 camp_balance 而非读 DB 列
**文件：** server/routes/nt.py:245-249
读 ledger 而非 `pool.camp_balance` → verify 非真正的复式记账校验。
**修复：** 直接读 `pool.camp_balance`，并与 ledger 推导值交叉校验。

### 5. 🟡 拒绝时 poster 存在路径缺 ledger 记录
**文件：** server/routes/nt.py:358-369
拒绝退 NT 给 poster → 无 ledger。仅 poster 不存在时有 `escrow_absorbed`。
**修复：** poster 存在时也加 ledger。

### 6. 🟡 cancel_task 完全无 ledger 记录
**文件：** server/routes/nt.py:375-393
退 escrow 但从不写 ledger。资金流动不可见。
**修复：** 退款后加 `_add_ledger`。

### 7. 🟡 client earn() 忽略 scope → 始终从 community pool 提取
**文件：** nt-core.js:388
`earn(userId, amount, reason, scope)` 硬编码 `'community'`。camp scope 调用者静默从错池提取。
**修复：** scope 为 camp 时路由到 camp 池。

### 8. 🟡 服务端 User 模型缺 overdueNT 列
**文件：** server/models.py:6-22
客户端住宿系统在 user 对象上存 `overdueNT`。服务端无此列。数据模型缺口。
**修复：** 加 `overdueNT = Column(Integer, default=0)` 或等到服务端住宿实现再说。

### 9. 🟡 spend 始终退还社区池 → camp pool 只出不进
**文件：** server/routes/nt.py:188, nt-core.js:393
camp-scoped spend → NT 仍回 community pool。camp pool 单向 faucet。
**修复：** scope 为 camp 时回 camp_balance。

### 10. 🟢 hasattr(req, 'pool') 始终 False → 死代码
**文件：** server/routes/nt.py:153
`EarnSpendRequest` 无 `pool` 字段。死代码。
**修复：** 删掉或补字段并加 TODO。

### 11. 🟢 amount 无整数检查 → 浮点数可污染余额
**文件：** nt-core.js:427,455
`earnFromPool`/`spendToPool` 检查 `<= 0` 但不检查 `Number.isInteger()`。
**修复：** 加 `!Number.isInteger(amount)` 检查。
