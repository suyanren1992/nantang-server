# UI 程序员审查 · 客户端与移动端

> 审查范围：渲染、移动端适配、收件箱、同步
> 文件：nantang-mobile/index.html, nantang-mobile/js/core.js, nantang-mobile/js/ui-cardroom.js, nantang-mobile/js/ui-wizard.js, nantang-mobile/css/theme.css

## 发现 10 项

### 1. 🔴 index.html:718 — `];` 语法错误 → 整个内联脚本崩溃
孤立 `];` 前无匹配 `[` → `SyntaxError` → 阻止整个 `<script>` 执行 → autoLogin、inbox、enterVillage、polling、hash 路由全部失效。页面完全不可用。
**修复：** 删除第 716-718 行（注释 + 悬空对象 + `];`）。

### 2. 🔴 ui-cardroom.js — UI 显示罚金 "-0.1 NT" 但代码扣 "-1 NT"
UI 文案说 -0.1 NT → 实际 `NT.spend(g.name, 1, ...)` 扣 1 NT。整数系统下文案与行为矛盾。
**修复：** 将 UI 文案更新为 "-1 NT" 与代码对齐。

### 3. 🔴 index.html:636,641 — _computeInboxMessages 仍被重复调用 + 随机 ID 使已读状态丢失
R6 中添加的 `_fullMsgs` 缓存只修复了分类计数调用。日志/NT 条目的随机 ID 在每次渲染时重新生成 → `_inboxReadIds` 永远无法匹配 → 所有日志/NT 消息永远显示为未读。
**修复：** 对日志/NT 条目使用确定性 ID。

### 4. 🟡 core.js:262 — filterQuests 中 `已完成` 任务在任何区域都不显示
三区划分（待领取/进行中/已完成）不含 `已完成` 状态 → 这些任务在任务大厅中不可见。虽然 R3 移除了"已完成"作为死状态，但旧数据或过渡状态仍可能出现。
**修复：** done 过滤器含 `已完成`，或确保所有遗留任务都迁移到其他状态。

### 5. 🟡 core.js:1190-1213 — logout() 不调 _stopPolling() → 登出后持续 401
**修复：** logout() 中在 setCurrentUser('') 前调 _stopPolling()。

### 6. 🟡 core.js:207-209 — 任务轮询永不更新现有任务数据
仅插入新任务 `if (!dup)`。服务器状态变更不反映到客户端。
**修复：** 对已有任务加合并/更新逻辑。

### 7. 🟡 theme.css:58 — #app 使用 `height:100vh` 无 safe-area → iOS 上内容被遮挡
**修复：** 加 `padding-top: env(safe-area-inset-top)` 和 `padding-bottom: env(safe-area-inset-bottom)`。

### 8. 🟢 core.js:904 — _calcCampTotalNT 仅含住宿/餐饮 NT，遗漏任务奖励
命名暗示含全部 NT 但实际只加 lodgingNT + mealNT。
**修复：** 加任务合计和激励金，或重命名为 `_calcCampBaseNT`。

### 9. 🟢 index.html:8 / theme.css:592 — touch-action:manipulation 仅匹配 [onclick]
选择器遗漏 addEventListener 绑定的元素。
**修复：** 改为更通用的 `button, a, [role="button"], [onclick]` 或根元素继承。

### 10. 🟢 index.html:686-702 — _toggleInbox 引用不存在的 `#inboxPanel`，创建新面板，静态 `#myInboxPanel` 从未使用
**修复：** 复用 `#myInboxPanel` 或删除之。
