## 1. 登录/注册流程

```
🔴 | index.html:532 | _inboxReadIds 从未初始化，所有 _isInboxUnread() 调用均抛 ReferenceError
用户影响：当 _toggleInbox() 调用 _renderInbox() 时，_isInboxUnread(m.id) 试图访问未定义的 _inboxReadIds，直接抛出 ReferenceError，导致整个 inbox 面板渲染崩溃。全局搜索仅有 _inboxReadIds[msgId] = true 和在 onclick 中的 _inboxReadIds={} 赋值，无 var _inboxReadIds = ... 声明。
建议修复：在文件顶部添加 `var _inboxReadIds = {}; try { _inboxReadIds = JSON.parse(localStorage.getItem('nt_inbox_read') || '{}'); } catch(e) {}`

🔴 | index.html:633 | _inboxCategory 未初始化，同抛 ReferenceError
用户影响：_renderInbox() 第 633 行 `if (_inboxCategory !== '全部')` 引用了未声明的变量。无 `var _inboxCategory` 任何声明。
建议修复：添加 `var _inboxCategory = '全部';`

🟡 | core.js:733 | 注册失败仅报告第一个校验错误，用户体验差
用户影响：enterVillage() 中，若用户同时未填名字且密码不足 6 位，仅显示 "请输入名字"，用户在第二个 field 上浪费尝试。
建议修复：一次性校验所有字段，汇总所有错误信息展示。

🟡 | api.js:23 | 网络错误 toast 使用通用消息，未区分 DNS 失败/超时/拒绝连接
建议修复：区分 TypeError(网络不通) 和 timeout，分别给提示。

🟡 | core.js:728 | 注册/登录 network error 与 server error 反馈相同
建议修复：在 asyncAuth 的 catch 中显式 callback({error:'network'})，enterVillage 据此区分。

🟢 | api.js:27-43 | asyncAuth 的网络失败传递了 null，enterVillage 正确区分处理
```

## 2. 会话恢复 (autoLogin)

```
🟡 | core.js:1259 (index.html:1259) | silentRefresh 失败时所有错误都降级为相同提示
用户影响：三种失败（网络不通 / refresh token 不存在 / refresh token 过期）统一显示 "服务器未连接，请手动登录"。refresh token 过期不等于服务器未连接。
建议修复：silentRefresh 在 .then 中区分 d._offline 和 {ok:false, error:'token invalid'}。

🟡 | core.js:1253 (index.html:1253) | autoLogin 期间页面完全隐藏无 loading 反馈
用户影响：autoLogin 开始时页面 visibility:hidden，直到 silentRefresh 成功才显示。若超过 3 秒，用户只看到空白页。
建议修复：在 autoLogin 开始时显示轻量 loading 遮罩（如渐入的 logo + "正在恢复会话..."）。

🟡 | core.js:1281 (index.html:1281) | 代码重复：file:// 的 HTTP sync 分支永远不会执行
建议修复：删除该死代码块。
```

## 3. 任务大厅 UX

```
🔴 | core.js:244-274 | filterQuests 搜索仅匹配 t.name，但服务端同步任务的 name 是 ID 不是标题
用户影响：filterQuests() 第 255 行 `t.name.toLowerCase().indexOf(keyword)!==-1` —— 当任务从服务端同步时，t.name 被赋值为 t.id（UUID），用户搜索 "筹备会" 不会命中任何任务。
建议修复：搜索条件改为 `(t.name||'').toLowerCase().indexOf(keyword)!==-1 || (t.title||'').toLowerCase().indexOf(keyword)!==-1`

🟡 | core.js:324-325 | toggleQuestCard 残留 expand 清理使用 is-visible 语义而非 DOM 结构
🟡 | core.js:238 | openQuestHallPage 筛选后未立即执行 filterQuests 当 API 不可用时
🟡 | core.js:274 | 任务大厅空状态缺少 CTA
🟢 | core.js:269-274 | 分段渲染（待领取/进行中/完成）合理，折叠状态可记忆
```

## 4. 错误反馈

```
🟡 | core.js:554-559 | doPublish 校验顺序问题：先 addTask 再 check NT
用户影响：doPublish() 先调用 AppData.addTask(data) 写入本地数据，再检查 NT.createTask() 是否成功。若 NT 余额不足，addTask 已成功写入但发布流程中断，导致本地出现孤儿任务。
建议修复：先调用 NT.createTask() 检查余额，成功后再 AppData.addTask()。

🔴 | data.js:333-346 | claimTask / unclaimTask 没有网络操作失败回退
用户影响：claimTask() 和 unclaimTask() 仅操作本地 TASKS 数据，不调用 API 同步。若服务端同步失败，本地和远端状态不一致。
建议修复：操作成功后调用 API.syncTaskUpdate() 并在失败时 showToast + 回滚。

🟡 | core.js:581 | deleteDraft 无确认对话框，可一键误删
🟡 | core.js:786 | resetAllData 的 confirm 对话框使用原生 confirm()
🟡 | data.js:470-489 | showToast 在 anchor 已被移除时静默失效
🟢 | data.js:470-489 | showToast 四种类型颜色语义清晰
```
