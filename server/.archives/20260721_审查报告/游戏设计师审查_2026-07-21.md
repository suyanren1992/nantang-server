> 2026-07-21 · 审查范围：nt-core.js, data.js, core.js, nt.py, tasks.py

## 🔴 阻断

### Bug 1. verifyTask 参数传递方式错误 — 驳回功能完全失效
服务端 `verify_task` 的 `approved` 和 `reject_reason` 声明为查询参数，客户端发送 JSON body。FastAPI 不读取 JSON body 中的非 Body() 参数。
- 通过: `approved` 默认为 `True`，碰巧正确
- 驳回: `approved` 仍然是 `True`（默认值），`reject_reason` 始终为空字符串

### Bug 2. `canUserClaim` 字段名与状态值完全不匹配
| 字段 | canUserClaim (auth.js) | TASKS 实际 (data.js) |
|------|----------------------|---------------------|
| 状态检查 | `task.status !== 'active'` | 实际为 `'进行中'` |
| 发布者 | `task.poster` | 实际为 `task.publisher` |
| 名额 | `task.max_slots` | 实际为 `task.slots` |

由于 `task.status !== 'active'` 永远为 true，`canUserClaim` 对任何任务都返回 `{ok: false}`。

### Bug 3. HTTP 模式 `_ntTaskId` 未设置 — accept/submit 从不调用服务端
