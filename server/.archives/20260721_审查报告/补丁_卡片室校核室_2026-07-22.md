## 1. 卡片室 confirmDiscovery 走 API（~5 行）

**位置**：[ui-cardroom.js:1122-1136](nantang-mobile/js/ui-cardroom.js#L1122)
**问题**：`NT.earn()` / `NT.spend()` 纯客户端，HTTP 模式下服务端不可见。
**改**：在 `NT.earn()` 调用之后追加 API 同步：

```javascript
// 在 NT.earn(d.guessedPerson, d.ntDoer, ...) 之后加：
if (typeof API !== 'undefined' && API.token) {
  API.request('POST', '/api/nt/transfer', {
    to_user: d.guessedPerson, amount: d.ntDoer, reason: '卡片室发现: ' + (d.actionLabel||''), scope: 'community'
  }).catch(function(){});
}
```

ponytail：只同步 doer 奖励。guesser ±1NT 小额的暂不同步（每人 1 NT，总量 <5 NT/卡片，等校核制服务端化后统一走 verification 端点）。

## 2. `_openVerificationPanel` 加 isMember 门（~3 行）

**位置**：[app.js:1908](nantang-mobile/js/app.js#L1908)
**问题**：独立校核弹窗无角色检查，visitor 可看到并点击校核按钮。
**改**：函数开头加：

```javascript
// 在 var me = _me(); 之后
if (typeof userCan === 'function' && !userCan({role: (AppData.me()||{}).role||'visitor'}, 'isMember')) {
  showToast('入住后可用', 'warn'); return;
}
```

已有先例：`_submitCleanEntry` (app.js:2105) 用了完全相同的守卫模式。
