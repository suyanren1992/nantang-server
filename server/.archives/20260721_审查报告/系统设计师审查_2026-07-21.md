# 系统设计师审查 — 数据层 / API 通信 / Hash 路由 / 每日 Tick

> 2026-07-21 · 审查范围：index.html, core.js, api.js, nt.py, accommodation.py, models.py, main.py

## 🔴 阻断

### 1. core.js:806 语法错误
`}).catch(function(){});` 是孤立的闭合——额外的 `}` 和 `.catch(...)` 没有匹配的开括号。导致整个 core.js 文件加载失败。

### 2. Daily tick 非并发安全
`_get_pool(db)` 未传 `lock=True`。两个同时的 POST 可同时通过 `last_tick_date != today` 检查，导致双重扣费和双重池补填。

### 3. Daily tick 不写 NT 账本记录
住宿费扣款和社区池补填静默执行，NTLedger 无记录。

### 4. Accommodation checkin 入住数检查不是原子的
`SELECT COUNT` 和 `INSERT` 之间无行锁。

## 🟡 功能异常

### 5. hashchange 单布尔 flag 可能丢失导航
`_hashFromCode` 只能记一道门，两次连续程序化 hash 变更可能丢失第一个。

### 6. GET /api/nt/sync 的 accommodation 硬编码为 None
C2.5 端点存在但从未被轮询 sync 调用，跨设备不一致。

### 7. 住宿欠费无上限，退房后不追缴

### 8. Tenancy 模型 bed_num 未约束，checkout_date 缺失

### 9. 轮询退避仅在下次成功 poll 时复位
