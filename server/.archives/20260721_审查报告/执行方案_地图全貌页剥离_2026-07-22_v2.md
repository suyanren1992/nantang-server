## 设计参考：厨房原型

[prototype/shared-kitchen.html](prototype/shared-kitchen.html) 中可复用的 UI 模式：

| 原型组件 | 模式 | 地图中对应 |
|---------|------|-----------|
| `.zone-card` 手风琴 | 点击展开/折叠，`open` class 切换，CSS transition | **建筑楼层**：每层一个 zone-card |
| `.fridge-doors` 双门 | 左右两扇冰箱门，各自展开独立内容 | **建筑房间**：一层内多房间的卡片网格 |
| `.item-row` 列表 | icon + name + meta + status tag + action btn | **房间物品**：已有 `itemsOverlay`，保留 |
| `.cook-banner` | 模式切换时的上下文横幅 | **不采用**，建筑内不需要模式横幅 |
| `.bottombar` 底部固定 | 主操作按钮固定在底部 | **建筑 FAB**：打扫/物品入口固定在底部 |
| `.picker-overlay` 底部弹出 | 从底部滑出的选择面板 | **已有 `_openQuickSheet`**，不动 |
| `.tabbar` 标签栏 | 水平 tab 切换内容区 | **已有 `floorTabs`**，搬到 building overlay |

**冰箱模式（核心参考）**：`.fridge-doors` 是双栏手风琴——两个 door-card 并列，点击一个展开，另一个自动收起。这个模式适合「一层楼 2-3 个房间」的场景。

### 步骤 4 · 新建 `openBuilding(idx)`（25 行）

```javascript
function openBuilding(idx) {
  var b = getBuildings()[idx];
  if (!b || b.id === 'info') return;

  // F4: checkin_5 quest hook（从 goTo() 迁移）
  if (b.id && window.AppData) {
    if (!AppData._data.visitedSpaces) AppData._data.visitedSpaces = [];

  overlay.classList.add('open');
  // 确保 overlayMap 也打开（从全貌进来的路径 overlayMap 本就是 open）
  if (!document.getElementById('overlayMap').classList.contains('open'))

    '<div class="ph-fallback"><div class="ph-emoji">'+b.icon+'</div></div>' +
    // F1: 左右箭头 — 保留建筑间连续切换
    '<button class="ph-arrow left" onclick="openBuilding('+(idx-1)+')">‹</button>' +
    '<button class="ph-arrow right" onclick="openBuilding('+(idx+1)+')">›</button>';
```

## 设计参考的采用

从 kitchen 原型借鉴，地图建筑详情中**不**采用：
- `.cook-banner` 模式横幅 → 建筑内不需要
- `.picker-overlay` 底部弹出 → 已有 `_openQuickSheet`
- `.tabbar` → 已有 `floorTabs`，搬到 building overlay 即可

**采用**：
- `.zone-card` 手风琴 → 建筑楼层展开模式（如果一层房间多，用 zone-card 分组）
- `.fridge-doors` 双栏 → 一层 2-3 个房间时，用 room-card 网格（当前 `roomsGrid` 的 `.room-card` 就是这种模式）
- `.item-row` 列表 → 已有 `buildRoomDetail` 函数渲染，不动
- `.bottombar` → FAB 固定在 building overlay 底部
