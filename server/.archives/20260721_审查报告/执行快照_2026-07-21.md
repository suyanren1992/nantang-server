## 🔴 立即修：6 项阻断（~1h，先做完再动别的）

### CR1 — canUserClaim 字段名不匹配
**位置**：[auth.js:200-241](nantang-mobile/js/auth.js#L200)
**问题**：`task.status !== 'active'`（应是`'进行中'`）/ `task.poster`（应是`publisher`）/ `task.max_slots`（应是`slots`）
**改**：修正三处字段名，或直接删函数（`claimTask` 已有内联检查，不走这个函数）

### CR2 — daily_tick 与 sync 并发竞态
**位置**：[core.js:801-805](nantang-mobile/js/core.js#L801)
**问题**：`GET /sync` 和 `POST /daily-tick` 并发发出，tick 的余额变更可能被 sync 的旧数据覆盖
**改**：把 `POST /api/system/daily-tick` 移到 `GET /api/nt/sync` 的 `.then()` 回调**之后**

### CR3 — 公约 quest 在 confirm 之前完成
**位置**：[app.js:1170→1185](nantang-mobile/js/app.js#L1170)
**问题**：`_completeNewbieQuest('sign_covenant')` 在 `confirm()` 对话框**前面**执行，用户取消后 quest 仍标记完成
**改**：`_completeNewbieQuest` 移到 `confirm(true)` 之后 + 补 `changeUserRole('npc')`（当前缺失）

### CR4 — newbie_quest_steps 死代码未真删除
**位置**：[app.js:68-69](nantang-mobile/js/app.js#L68) + [app.js:238-240](nantang-mobile/js/app.js#L238) + [app.js:1913-1914](nantang-mobile/js/app.js#L1913)
**问题**：设为 `[]` 而非删除，死代码骨架仍在（238 行仍在读取）
**改**：删属性定义 + 删读取逻辑 + 确认 data.js:59 的 5 quest 数组不受影响

### CR5 — E3.7 三个权限门未接线
**位置**：物品存取 [app.js:1937→1987](nantang-mobile/js/app.js#L1937) / addVerification 入口 / joinDeepClean+markAreaDone 入口
**问题**：visitor 可以存取物品赚 NT、发起校核请求、参与大扫除——完全没有 `isMember` 检查
**改**：三个入口各加 `if (!userCan(currentUser, 'isMember')) { showToast('入住后可用'); return; }`

### CR6 — 客户端住宿扣费加 HTTP 模式跳过
**位置**：[app-data.js:413](nantang-mobile/js/app-data.js#L413)
**问题**：客户端 `_deductAccommodation`（费率 perBed 30-150）与服务端 daily_tick（费率 10）独立运行，可能双重扣费+金额不一致
**改**：函数开头加 `if (typeof API !== 'undefined' && API.token) return;`
