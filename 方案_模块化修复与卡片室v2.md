## 一、诊断：模块化为什么失败了

### 现状：双轨腐烂

```
巨石版 nantang-mobile.html (6,872 行)
  ├── 内联 CSS 522 行
  ├── 内联 HTML ~440 行
  ├── 外部 JS 11 个（js/*.js）
  └── 内联 JS ~5,600 行 ← 核心问题

模块版 nantang-mobile/ (理论上的未来)
  ├── index.html (482 行) ← 不完整，不能独立运行
  ├── 外部 JS 20 个（已按模块拆分）
  └── 缺：登录/村口/我的/任务大厅/地图的 HTML 和 JS
```

### 差距清单

**9 个模块只在模块版存在，巨石版仍以内联 JS 重复实现：**

| 模块文件 | 行数 | 对应巨石版内联 JS 段 |
|---------|------|-------------------|
| `core.js` | 1,216 | 工具函数 + UI 辅助，散落在内联 JS 中 |
| `data.js` | 611 | CURRENT_USER / setCurrentUser / getUsers 等 |
| `ui-village.js` | 113 | 营地入口、邀请码 |
| `ui-camp.js` | 1,569 | 营地全部功能（概览/日程/资金/成员/管理/结算） |
| `ui-wizard.js` | 977 | 7 步创营向导 |
| `ui-phase4.js` | 532 | 客栈 + 食堂 |
| `ui-social.js` | 443 | 团队动态 feed |
| `ui-archive.js` | 418 | 档案室 + 鼓励 |
| `api.js` | 192 | API 适配层 |

**2 个文件已分歧：**

| 文件 | 巨石版 | 模块版 | 差异 |
|------|--------|--------|------|
| `app-data.js` | 543 行 | 587 行 | 模块版多了 addVerification/addJournal/flipPresence/cardDiscoveries |
| `nt-core.js` | 691 行 | 697 行 | 微小差异 |

**ui-cardroom.js 手动同步中：**
- 两版各有一份（`js/ui-cardroom.js` 和 `nantang-mobile/js/ui-cardroom.js`）
- 每次改动后手动 `cp` 同步
- 已同步但脆弱——下次忘 cp 就分裂

### 根因

1. **模块版 index.html 不能独立运行**——缺登录页/村口页/我的页/任务大厅/地图 overlay 的 HTML 和 JS，从未完工
2. **巨石版一直在演进**——所有新功能（卡片室、信箱、公约定价）都在巨石版上开发
3. **没人切过去**——因为模块版跑不起来，所以巨石版继续膨胀
4. **同步靠手工**——每次改 ui-cardroom.js 后手动 cp，其他 9 个模块早已不同步

## 二、修复策略：不追求完美，追求可用

### 原则

- **不动巨石版**——它继续跑，不做大手术
- **不全量迁移**——不把 5,600 行内联 JS 全部拆出去
- **只做增量**——新功能写在模块版，巨石版逐步缩减

### 路线

```
现在                      目标
巨石版 ← 生产              模块版 ← 生产
模块版 ← 残废              巨石版 ← 冻结（仅修 bug）
                          增量开发在模块版
```

### 怎么做到

**不追求「把巨石拆成模块」——这需要 2-3 天全職工作。而是：**

1. 补全模块版 index.html 使其能独立运行（2-3h）
2. 在模块版上开发卡片室 v2 + 世界喇叭（2-3h）
3. 巨石版加载模块版的输出（过渡期）
4. 新功能全部在模块版写，巨石版只做 bugfix

**A2. 提取剩余 JS（1h）**

**A3. 统一分歧文件（30min）**

### 阶段 B：卡片室 v2（2-3h）

**B1. 每人一次猜测机会（30min）**

```javascript
// cardDiscoveries 中每张牌加 guesses 数组
d.guesses = d.guesses || []; // [{name:'砚仁', guessedAt:'...'}]

// 猜测时检查
if (d.guesses.some(function(g){ return g.name === CURRENT_USER; })) {
  showToast('你已经猜过这张牌了', 'warn'); return;
}

// 确认后锁定
if (d.status === 'confirmed') {
  // 牌翻开，不再接受新猜测
}
```

**B2. 世界喇叭（20min）**

```javascript
// 全局函数，放在 core.js 或独立 trumpet.js
function _trumpet(msg, type) {
  // type: 'discovery' | 'golden' | 'streak' | 'system'
  var el = document.createElement('div');
  el.className = 'trumpet-bar';
  el.innerHTML = '🎺 ' + msg;
  el.style.cssText = 'position:fixed;bottom:60px;left:8px;right:8px;z-index:400;'+
    'background:linear-gradient(135deg,#5a4a20,#8a6a30);color:#ffd700;'+
    'padding:10px 16px;border-radius:12px;font-size:.72rem;font-weight:600;'+
    'text-align:center;box-shadow:0 4px 20px rgba(0,0,0,.3);'+
    'animation:trumpetIn .3s ease-out;pointer-events:none';
  document.body.appendChild(el);
  // 4 秒后滑出
  setTimeout(function(){ el.style.transition='opacity .3s,transform .3s'; el.style.opacity='0'; el.style.transform='translateY(20px)'; }, 3500);
  setTimeout(function(){ if(el.parentNode) el.remove(); }, 4000);
}
```

触发点：
- `confirmDiscovery()` → `_trumpet(guesser + '猜对了！'+ guessedPerson + ' 在 ' + spaceName + ' ' + actionLabel)`
- 金边牌（NT≥15）→ 加 🌟 前缀
- 连续猜中 3 次 → `_trumpet(guesser + ' 手气正旺！连续猜对 3 张牌！', 'streak')`

**B3. 扑克牌网格（1h）**

改 `renderCardRoom()`：
- 4 列 CSS Grid 替换横滑列表
- 每张牌：固定宽度 `calc(25% - 6px)`，统一高度
- 花色角标（左上）+ 劳动 emoji（中央）+ 空间（底部）+ NT（右下）
- 金边牌（NT≥15）金色边框
- 待揭晓（米白底）/ 已揭晓（微绿底）/ 已过期（灰色底）

**B4. 全貌页入口精简（10min）**
