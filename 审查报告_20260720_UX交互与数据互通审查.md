## 〇、关键发现：index.html 存在大量内联覆盖

`index.html` 的 `<script>` 块（第 490–6362 行）重定义了 `ui-camp.js` 和 `ui-phase4.js` 中的函数。**内联脚本后加载，完全覆盖外部文件。** 对外部 .js 文件的修改无效——实际运行代码在 index.html 里。

受影响的函数包括：`openCampHome`、`switchCampTab`、`_openQuickMenu`、`openCanteen`、`closeCanteen`、`openInn`、`closeInn`、`renderCanteen`、`renderInn` 等数十个。

**修复建议**：将内联覆盖代码提取到独立的 .js 文件中，或者删除重复定义。

## 一、营地 UI 导航问题

### 1.1 订餐/住宿/成员 → 退出到村口

**根因**：`closeOverlay()` 无条件执行 `villagePage.classList.remove('hidden')`，没有导航栈。

```
营地主页 → 点 X 关闭 → closeOverlay('overlayCampHome')
    → overlayCampHome 关闭 ✓
    → villagePage 显示 ✗（不应该）
```

**位置**：`index.html:1234` / `data.js:500`

食堂和客栈的关闭按钮**是正确的**——`closeCanteen()` 和 `closeInn()` 会重新打开 `overlayCampHome`。问题出在下一步：关闭营地主页时就回到村口了。

**修复**：`closeOverlay` 不要无条件显示 villagePage；改由调用方决定是否回到村口。

### 1.2 底栏头像点不动

**根因**：`showMy()` 显示工作台但**没有关闭 `overlayCampHome`**，营地全屏遮罩盖住了工作台。

```html
<div class="cb-btn" onclick="showMy()">  <!-- index.html:168 -->
```
→ `showMy()` 显示 `myPage` ✓
→ 但 `overlayCampHome` 的 `z-index` 更高，盖住了 ✗

**对比**：营地概览页的"我的任务包"卡片做了正确的事：
```javascript
onclick="closeOverlay('overlayCampHome');showMy({presetChip:'营队'})"
```
头像按钮少了 `closeOverlay('overlayCampHome')`。

### 1.3 按钮标签"发布任务"应为"营队管理"

| 位置 | 当前标签 | 实际行为 | 应改为 |
|------|---------|---------|--------|
| `index.html:3467` | "📋 发布任务" | `switchCampTab('manage')` 打开完整管理面板 | "📋 营队管理" |
| `index.html:1718` | "📋 发布任务" | 同上 | "📋 营队管理" |

## 二、快捷菜单问题

### 2.1 11/16 个菜单项是占位符

16 个 `ALL_MENUS` 项目中有 **11 个调用 `_wipToast()`**——显示"即将上线"提示，无实际功能：

- 成员签到 → toast
- 添加采购项 → toast
- 提交报销 → toast
- 今日课程 → toast
- 材料准备 → toast
- 上传照片 → toast
- 征集文稿 → toast
- 写简报 → toast
- 结项进度 → toast
- 展览准备 → toast
- **问题反馈 → toast**（用户直接报告的问题）

只有 6 个有实际功能：发布营队任务、查看待审核、待结算列表、NT池统计、资金流水、卡片室。

### 2.2 菜单过滤逻辑是"按任务类别"而非"按角色"

```javascript
// 用户被分配了哪些任务 → 任务的 category 字段 → 匹配 ALL_MENUS 的 key
```

如果用户被分配的任务全属于"管理"类别，菜单只显示管理相关项。管理员没有"显示全部"的绕过机制。

### 2.3 冒险者永远看不到快捷菜单

冒险者的底栏被替换为订餐+住宿按钮（`ui-camp.js:27-30`），没有"+"按钮。

## 三、信箱——三套独立系统

| 系统 | 数据源 | 触发方式 | 问题 |
|------|--------|---------|------|
| 阶段1信箱 `_toggleInbox` | 待校核、发现、公告 | 地图顶栏📬按钮 | 数据不全，未读数不共享 |
| 营地信箱 `openCampInbox` | 待审核、待提交、待结算 | 营地底栏📬按钮 | 仅在营地内可用 |
| 香草信箱 `renderInbox` | TASKS + finance | 工作台收件箱面板 | **死代码**——`openInboxPanel()` 实际调用的是阶段1信箱 |

三个系统的不读标记（`nt_inbox_read` vs `inbox_seen_ids`）也互不通信。一个信箱里标为已读，另一个信箱里还是未读。

## 四、档案室——内容为空

**根因**：档案室的三个 tab 依赖的数据只在特定操作中写入：

| Tab | 数据源 | 何时写入 |
|-----|--------|---------|
| 📂 成员 | `AppData._data.users` | 注册时 |
| 📜 日志 | `activity_log` | `logActivity()` 每次调用 |
| 📦 往期营期 | `archived_periods` | 营地完结（close ceremony）时 |

**用户报告"档案室没有内容"的原因**：
- 如果从未完结过营地 → 📦 往期营期为空
- 如果 activity_log 为空（新服务器）→ 📜 日志为空
- 如果用户数据从 API 加载而非本地 → 📂 成员依赖 `getUsers()`（localStorage），在 HTTP 模式可能为空

**额外发现**：`index.html` 内联脚本中有 `ui-archive.js` 的一份副本，其中包含了外部文件里没有的 `cards` tab 处理器。两边代码不一致。

## 五、信息互通——根因总结

所有"A 改了什么 B 看不到"的问题，根因都一样：

```
每个模块读自己的数据源，没有共享层。

地图     → HARDCODED_BUILDINGS（永远不变）
信箱     → 各自的过滤逻辑（三套互不通）
档案室   → activity_log（只在特定操作写入）
在线人数 → 硬编码3人
整洁状态 → 硬编码字符串
```

**需要建一个统一的数据层**——每个模块从同一个 AppData/API 读取，而非各自硬编码或独立存储。
