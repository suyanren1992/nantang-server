# 世界终端 · 设计定稿

> 2026-07-22 · 定稿 · 源自砚仁口述设计
> 现状：社区副本页内的 admin「创建营队」入口 + 7 步创营向导 + 营地 7 Tab

---

## 0. 一句话定位

**世界终端 = 线下活动的全流程操作系统 + 团队的仪式感发生器。**

它不只是一个"创建营队"的按钮。它是一个辅助线下的人把一场活动/一期营队**办成、管好、记下来、展示出来**的工具。没有这个平台，这些信息都会消失。

---

## 1. 覆盖的生命周期

```
① 筹备          ② 开营          ③ 活动中            ④ 结算          ⑤ 沉淀
预算/人员    →   仪式感启动   →   日程/住宿/吃饭   →   NT 结算      →   档案室长期展示
天数/日程        （点火仪式）     桌游室/八卦室        （结营报告）     （成为社区记忆）
```

- **筹备**：预算、人员、天数、日程编排
- **开营**：仪式感（现有的点燃火种动画保留并强化）
- **活动中管理**：住宿、吃饭已接入（客栈/食堂）；扩展桌游室、八卦室/茶馆
- **结算**：营内 NT 收支清算、结营仪式、结营报告
- **沉淀**：全部记录进档案室，成为**长期的展示空间**——往期营期不只一行卡片，而是可以点进去回看的一场完整活动

---

## 2. 特色场景：八卦室 / 茶馆

线下最有意思的部分之一，**情报交换的经济闭环**：

- **发八卦要花 NT**——说话有成本，八卦有分量
- **打听别人的八卦也要花 NT**——情报有价格
- 八卦记录进档案室（社区记忆的一部分）

定价建议（可公约调整）：发一条 2 NT，打听一条 1 NT，费用进社区池。

---

## 3. 与世界终端相关的既有资产

| 已有 | 状态 |
|---|---|
| 7 步创营向导（含 sessionStorage 草稿、点火仪式动画） | ✅ 可作筹备+开营的底子 |
| 营地主页 7 Tab（概览/日程/资金/成员/排行/结算等） | ✅ 活动中管理的底子 |
| 结营仪式 + 结营报告 + 往期营期归档 | ✅ 结算+沉淀的底子 |
| 客栈住宿 / 食堂订餐 | ✅ 已接入营队 |
| 桌游室 / 八卦室 | 🔴 未做 |

**所以世界终端不是从零做，而是把已有的营地系统"升维"：从"管理一期营队"扩展为"运营一切线下活动"。**

---

## 4. 设计原则

1. **记录即价值**：每个环节产生的信息都自动沉淀，不需要谁专门写总结
2. **仪式感不可省**：开营点火、结营报告——凝聚团队靠的就是这些时刻
3. **让积分流转起来**：活动的每个环节都是 NT 流动的机会（做饭、住宿、八卦、悬赏）
4. **让人更容易做事**：从"我有个想法"到"活动办成"的路径越短越好——这也是整个平台的初衷
