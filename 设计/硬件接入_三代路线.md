# 硬件接入 · 三代路线

> 2026-07-22 · 定稿 · 源自 v11 说明书 L1/L2 层 + 2026-07-22 可行性讨论
> 核心结论：**远期完全可行，钱约等于没有，升级是"平滑接入"不是"推倒重来"。**

---

## 0. 关键设计：卡片室不变，只换线索来源

```
第一代（现在）：线索来自人 → 有人发现变化，手动发起发现
     ↓ 卡片数据结构、猜人玩法、NT 结算 全部不变
第二代（硬件试点）：线索来自摄像头 → 自动生成匿名线索卡
     ↓ 还是同一个卡片室
第三代（多源融合）：摄像头 + 手环 + GPS 交叉验证，置信度分级
```

软件先行是正确顺序：等硬件到位，游戏闭环已被真人验证，硬件只是接进来一个新的"线索生产者"。

---

## 1. H1 · 试点（增量成本≈0，到货后 1-2 周）

- 现有 3-4 个 RTSP 摄像头中选 1 个（厨房或画室）
- 旧电脑跑 **Frigate NVR**（开源，无需 GPU；可选 Coral TPU ¥400-800 加速）
- Frigate 事件 → webhook → FastAPI 后端 → 生成匿名线索卡：
  「画室，下午 2:00-2:20，有人持续活动」
- **线索只说"有人活动"，不声称知道在干什么**——判断交给人（猜），这是设计哲学不是技术妥协

## 2. H2 · 布点与双源印证

- ESP32-CAM ×3（¥35/个）：厨房、工作台、工具柜，定时拍照 HTTP 上传
- 手机 GPS（HTML5 Geolocation）：打卡与区域停留印证
- 树莓派 5 + CSI（¥450）：楼顶每日 8:00 俯拍菜地 → 作物阶段分析

## 3. H3 · 多源融合

- 小米手环 ×3-5（¥250/个）：体力消耗数据（**集成最麻烦**：小米无官方 API，需第三方方案，所以放最后）
- 人脸识别（CompreFace/Double Take）：住宿自动登记——**必须先签知情同意书**
- 置信度体系：单源 < 双源 < 多源，线索标注置信度

## 4. 隐私三区策略（不可妥协）

| 区域 | 例子 | 策略 |
|---|---|---|
| 公共区域 | 厨房、画室、菜地 | 摄像头覆盖，AI 识别 |
| 半公共区域 | 走廊、楼梯间 | 红外感应器，只感知移动 |
| 私密区域 | 宿舍内、卫生间 | **零设备** |

视频流本地存储不上云（7 天）；人脸仅存特征向量；GPS/手环随时可关可取；隐身模式随时可开。

## 6. 诚实的难度分级

1. "有没有人"——简单，Frigate 开箱即用 ✅
2. "在干什么"——难，但设计已绕开（线索不声称动作）✅
3. "这个人是谁"——最难，多源交叉 + 知情同意，放 H3
