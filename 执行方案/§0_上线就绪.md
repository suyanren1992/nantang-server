### R4 P0 — 结算双写 LEDGER + NT

> **🔍 审查：两处语义问题需要明确**
> 
> **1. NT 给谁？当前写法 `NT.earn(CURRENT_USER, ...)` 把 NT 给了结算人（settler），但结算人是付款方，不是收款方。** 收款方应该是任务完成者（`t.claimants` 中 `status==='completed'` 的人）。在现有测试数据中，所有任务的结算人恰好也是完成人（如"营地布置"：砚仁发布→砚仁完成→砚仁结算），所以 MVP 阶段不暴露问题。但 §1 引入多用户测试后会出错（如"宣传工作"：砚仁结算→15NT 应该给小云，但当前写法给了砚仁）。
> 
> **修正建议（§0 可选，§1 必做）：**
> ```javascript
> // 给所有完成者发 NT
> var completed = (t.claimants||[]).filter(function(c){return c.status==='completed'});
> completed.forEach(function(c) {
>   LEDGER.personal.balance += t.nt;  // 应改为按人数均分
>   LEDGER.personal.income += t.nt;
>   LEDGER.personal.records.unshift({d: today().slice(5), note: t.name+' 结算', amt: t.nt, in: true});
>   if (window.NT) NT.earn(c.name, t.nt, t.name+' 结算', 'personal');
> });
> ```
> 
> **2. 总是写 `LEDGER.personal`，不区分 scope。** 如果任务 scope 是 `'camp'` 或 `'community'`，应该写入对应账本。当前测试数据中结算的任务都是 `scope:'营队'` 或 `scope:'个人'`，但逻辑上写死 `personal` 是错误的。
> 
> **结论：§0 可接受当前写法（所有测试数据 settler=doer），但 §1 执行前必须修正。**

**影响：** 任务结算后 NT 余额不变——核心数据 bug

```javascript
// 在 t.status='已结算' 之后加：

  d: today().slice(5), note: t.name + ' 结算', amt: t.nt, in: true
```

## 🔍 测试场景说明（审查补充）

**§0 的"多人测试"实际上是同一浏览器 Tab 内切换用户身份。** 因为 `TASKS`、`LEDGER`、`NT` 数据都在内存中（无 localStorage、无后端），每个浏览器 Tab 拥有独立的 JS 运行环境。测试方式是：

1. 登录砚仁 → 发布任务 → 登出
2. 切换小云 → 接任务 → 提交 → 登出  
3. 切换砚仁 → 审核 → 结算

三个人可以各自在自己的设备上独立测试相同的流程，但不能跨设备协作同一个任务。**这在 §0 是预期行为**——真正的多人协作需要后端或 localStorage 同步，属于 §3 F6-F7 的范畴。

另外，**刷新页面会丢失所有数据**（TASKS、LEDGER、NT 全部重置为初始值）。测试前提醒测试者：测试期间不要刷新页面。这个限制在 §0 部署说明中应当明确写出。
