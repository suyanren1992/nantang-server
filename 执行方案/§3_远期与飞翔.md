## F1：卡片室 — 「猜谁在干活」

**这是产品说明书 v11 的核心创新。** 不是"发任务→接任务"，而是"系统静默记录→有人发现变化→猜是谁做的→猜对双方都得 NT"。

### 核心流程

```
摄像头/Frigate 感知 → 生成匿名线索 → 存入线索池
     ↓
社区成员路过 → 发现变化 → 在卡片室发起"谁做的？"
     ↓
系统推送相关线索 → 成员猜测 → 系统比对确认
     ↓
猜对 → 发现者 +5 NT（立刻到账，不等确认）
     ↓
做事的人收到通知 → 确认公开(+10 NT) 或 保持匿名(0 NT)
     ↓
公开记录写入个人面板 + 活动记录
```

### 需要的数据结构

```javascript
// 线索池
var CLUES = [{
  id: 'C001',
  locationId: 'studio',
  description: '画室下午2:00-2:20有人持续活动',
  detectedAt: '2026-07-18T14:20:00Z',
  confidence: 0.85,
  sources: ['camera','wristband'],
  possiblePersons: ['小红'],
  status: 'pending'  // pending → guessed → revealed
}];

// 发现卡片
var DISCOVERIES = [{
  id: 'D001',
  clueId: 'C001',
  initiator: '朝林',      // 发起发现的人
  guess: '小红',           // 猜的人
  actualPerson: '小红',    // 系统确认的人（比对后填充）
  guessCorrect: true,
  finderReward: 5,         // 发现者奖励
  doerReward: 10,          // 做事的人奖励（确认后发放）
  doerConfirmed: false,    // 做事的人是否已确认公开
  createdAt: '...'
}];
```

### 过渡方案（硬件未接入时）

硬件未部署前，可以用**手动创建线索**的方式让卡片室先跑起来：
- 任何人可以手动输入一条线索：「厨房变干净了」
- 其他人来猜
- 做事的人可以自己站出来确认
- 猜对的人 +5 NT，做事的人 +10 NT

**功能等同于完整版，只是线索来源从硬件变成了人工。** 这样 L5 卡片室不需要等硬件。

## F2：称号系统

### 称号树

```
主干：🌱 新芽(0) → 🌿 小树(100XP) → 🌳 大树(500XP) → 🏔️ 山峰(2000XP) → ⭐ 星辰(5000XP)

分支称号：
  🧹 清洁工    打扫 10 次
  🧑‍🌾 园丁      农活 10 次
  🎨 画师      画室使用 50 小时
  👀 发现者    发起发现 10 次
  🗺️ 初级测绘员 完成新手引导
  🔧 维修工    维修 5 次
```

### 实现

- 主称号 = f(总 XP)，自动升级
- 分支称号 = f(特定行为计数)，达到阈值解锁
- 称号解锁权限（如 🌳 大树才能参加陪审团）
- 个人面板展示当前称号 + 进度条

## F3：赏金池

- 任何人发布悬赏（猎杀苍蝇 ×3、喂猫、遛狗…）
- 其他人认领 → 完成 → 确认 → 领赏金
- 资金来源：成员充值 + 标签费 + 社区活动收入抽成
- 池余额实时可见

---

## F4：标记系统

- 任何人在任何时间可在地图上添加标记
- 系统自动用摄像头校验
- 推送核验任务给附近的人
- 到场拍照确认 → 标记永久生效
- 创建人得信誉分，核验人得 NT

## F5：新手引导

7 步任务链（对应产品说明书 §2.2）：
1. 找到地图上 3 个标记 +5 NT
2. 核验 1 个待确认标记 +5 NT
3. 添加 1 个新标记 +5 NT
4. 参加 1 次大扫除 +10 NT
5. 参加 1 次饭局 +10 NT
6. 完成 1 次晨练 +5 NT
7. 全部完成：🗺️ 初级测绘员称号 + 额外 +10 NT
