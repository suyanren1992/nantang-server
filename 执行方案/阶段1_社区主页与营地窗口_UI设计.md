## 三、邀请码弹窗

### 设计

```
┌──────────────────────────────────────┐
│                                      │
│          ⚔️                          │  ← 32px
│                                      │
│       输入冒险者邀请码                 │  ← .85rem bold
│   加入冒险者行列，进入共创营           │  ← .62rem #8a8a8a
│                                      │
│ ┌──────────────────────────────┐   │
│ │     NT-XXXXXXXX              │   │  ← center, letter-spacing 2px
│ └──────────────────────────────┘   │     2px solid #d0d9ce, r=8px
│                                      │     placeholder="NT-XXXXXXXX"
│ 格式：NT- 开头 + 4-12位字母数字       │  ← .58rem #8a8a8a
│                                      │
│ ┌──────────────────────────────┐   │
│ │          确认提交             │   │  ← bg=#3d6b52 white
│ └──────────────────────────────┘   │
│ ┌──────────────────────────────┐   │
│ │          取消                 │   │  ← bg=transparent #8a8a8a
│ └──────────────────────────────┘   │
│                                      │
│ 没有邀请码？请联系管理员获取          │  ← .58rem #8a8a8a
└──────────────────────────────────────┘

弹窗: 居中 modal, bg=white, r=16px, max-width=300px
遮罩: rgba(0,0,0,0.4), 点击遮罩=取消
input: 字号 1rem, 自动转大写, 提交时 trim
```

### 交互

```
输入邀请码 → 点确认
  ├── 空     → input border 变红, 显示 "请输入邀请码"
  ├── 格式错 → input border 变红, 显示 "格式：NT- 开头 + 4-12位字母数字"
  ├── 不存在 → input border 变红, 显示 "该邀请码不存在"
  ├── 已被用 → input border 变红, 显示 "该邀请码已被使用"
  ├── 不匹配 → input border 变红, 显示 "邀请码不匹配当前营队"
  └── 正确   → 关闭弹窗 → toast "身份升级成功！⚔️" → 进入营地

点取消 / 点遮罩 → 关闭弹窗
```

### 设计原理

- `prompt()` 只有一个空白输入框，用户不知道要输入什么
- 图标+标题+副标题回答了「是什么→为什么→怎么做」
- 错误在弹窗内显示（不在 toast），用户不需要转移注意力
- 自动转大写，减少格式错误

## 四、营地窗口「送鼓励」

### 参考来源

PC 端 `社区窗口方案.md` §四「打赏选人」— 已实现的完整系统：
- `💬 送个鼓励` 面板
- `送给：[选择成员 ▾]` — 复用 `showReviewerPicker`
- `[👏太棒了] [💪加油]` — 免费鼓励
- `[🌹1] [🎁5]` — 付费打赏，走 `recordTransaction({type:'tip'})`
- 显示发送者 NT 余额
- 没有池子——打赏全从个人账户出

### 移动端适配

```
── 💬 送个鼓励 ──────────────────────
                                      │
送给：[小云 ▾]                        │  ← 点击弹出选人 Sheet
                                      │
[👏] [💪] [🌹1] [🎁5]               │  ← 免费 + 付费
                                      │
你的余额：💎 125 NT                   │  ← 仅登录用户可见
```

### 成员选择 Sheet

```
从屏幕底部滑入，高度约 60vh，圆角顶边

┌──────────────────────────────────────┐
│ 选择送给谁                    ✕ 关闭 │
├──────────────────────────────────────┤
│ 🔍 搜索…                            │
│                                      │
│ 👤 小云   🧱 共建者                  │  ← 点击选中，关闭 Sheet
│ 👤 砚仁   🧙 管理员                  │
│ 👤 朝林   ⚔️ 冒险者                  │
│ 👤 淑惠   👥 NPC                    │
└──────────────────────────────────────┘
```

数据源：`AppData.shared('camps')` 中当前营队的成员列表，或 `getUsers()` 全部注册用户。

### 鼓励类型

| 按钮 | 类型 | NT 消耗 | 效果 |
|------|------|---------|------|
| 👏 | `encourage_applause` | 0 | 写 journal："xxx 给 yyy 送了 👏太棒了" |
| 💪 | `encourage_support` | 0 | 写 journal："xxx 给 yyy 送了 💪加油" |
| 🌹1 | `tip` | 1 NT | `recordTransaction({type:'tip', from, to, amount:1})` |
| 🎁5 | `tip` | 5 NT | `recordTransaction({type:'tip', from, to, amount:5})` |

### 交互规则

| 情况 | 行为 |
|------|------|
| 未选人点鼓励 | toast「请先选择送给谁」 |
| 免费鼓励（👏/💪） | 直接发送，写 journal |
| 付费鼓励（🌹/🎁）未登录 | toast「请先登录」 |
| 付费鼓励余额不足 | toast「余额不足」 |
| 付费鼓励成功 | `recordTransaction()` + toast「已送 🌹 给 yyy！」 |
| 打赏自己 | toast「不能送给自己」 |
| 选过一次人 | 变量记住，下次默认选中 |

### 设计原理

- **免费鼓励是社交信号，付费打赏是经济行为。** 分开按钮，视觉上免费（👏💪）和付费（🌹🎁）有明确边界
- **选人先于操作。** 先定目标，再选行为。不是先点了按钮再弹选人——减少一步
- **记住上次选择。** 同一个人可能被多次鼓励，不用每次都选
- **打赏全从个人账户出。** PC 端已验证：池子从未生效，删掉了
