# 阶段 1 前置B — 档案室 · UI 设计

> 2026-07-19 · 前置：阶段 0（AppData + 身份系统）+ 阶段 1 前置A（信箱 + activity_log 写入）已完成
> 定位：社区公共信息存档空间。地图上有建筑入口，社区活动里有链接入口。

## 〇、定位

**档案室不是个人工作台面板。** 它是社区公共基础设施——和画室、厨房、客栈同级的地图建筑。任何社区成员都可以进入查看。

```
入口 1：实景地图 → 📚 档案室（一个建筑，和画室/厨房并列）
入口 2：社区活动主页 → 📚 档案室（底部链接，和世界终端并列）
```

两个入口打开同一个档案室 overlay。

## 一、信息架构

### 1.1 社区存档体系全景（参考 PC 端 `云村档案室统合方案.md`）

PC 端有五个存档入口，分属三个体系。移动端对应关系：

| PC 端入口 | 体系 | 做什么 | 移动端对应 |
|-----------|------|--------|-----------|
| Hub「📚 档案室」 | **成员档案** | 成员目录 + 个人案卷（打赏/财务） | **阶段 1 前置B（本阶段）** |
| 工作台「📦 存档」 | **任务归档** | 已完成且超过 7 天的任务 | 阶段 3（数据层 `getTaskDoneDate()` + `ARCHIVE_DAYS`） |
| 任务大厅「📦 已归档」 | **任务归档** | 同上，全村视角 | 阶段 3 |
| 村庄「📦 往期」 | **营期快照** | `data.archived_periods` 完结快照 | 阶段 6（结营报告） |
| NT管理「📦 完结本期」 | **管理员操作** | `closePeriod()` 批量转角色 | 阶段 5（营地管理面板） |

**本次（阶段 1 前置B）只做 Hub 档案室（成员档案体系）。** 任务归档和营期快照是独立系统，留给对应阶段。

### 1.2 Hub 档案室 Tab 结构

```
📚 档案室（社区公共空间）
  ├─ 📂 成员      ← MVP：按角色分组 + 搜索 + 折叠（数据源：getUsers()）
  ├─ 📜 日志      ← MVP：activity_log 时间倒序列表（数据源：data.activity_log）
  ├─ 📋 个人案卷  ← 阶段 5：点头像进入，打赏/财务/审核记录（数据源：data.finance + data.tips）
  └─ 📦 往期营期  ← 阶段 6：历期共创营回顾（数据源：data.archived_periods）
```

**本阶段做前 2 个 Tab。** 后 2 个依赖后续阶段的数据积累。

## 二、地图入口

### 2.1 位置：大地书房

档案室不新建建筑——放在已有的大地书房内。书房天然是社区文字记录和档案的存放处。在大地书房的房间列表中加一个「📚 档案室」房间：

```javascript
// 大地书房（已有建筑）加一个房间：
{
  id: 'archive',
  name: '档案室',
  icon: '📚',
  sub: '社区记录 · 成员目录 · 运行日志'
}
```

点击大地书房 → 看到现有房间 + 新增的「📚 档案室」→ 点击 → 打开档案室 overlay。

### 2.2 如果大地书房在地图上尚不存在

则在 `data.map_locations.buildings` 中新增大地书房建筑，内含档案室房间：

```javascript
{
  id: 'dadi_study',
  name: '大地书房',
  icon: '📖',
  floors: {
    '1F': [
      { id: 'archive', name: '档案室', icon: '📚', sub: '社区记录 · 成员目录 · 运行日志' }
    ]
  }
}
```

## 四、UI 设计

### 4.1 档案室 overlay（`#overlayArchive`）

```
┌──────────────────────────────────────┐
│ ← 返回    📚 档案室                   │  ← header：和社区主页/营地窗口一致
├──────────────────────────────────────┤
│ [📂 成员]  [📜 日志]                  │  ← Tab 栏（本阶段 2 个 Tab）
├──────────────────────────────────────┤
│                                      │
│         （当前 Tab 内容）              │
│                                      │
└──────────────────────────────────────┘
```

### 4.2 Tab 1：📂 成员

```
│ 🔍 搜索姓名…                         │  ← sticky 搜索框
│                                      │
│ ▸ 🧙 管理员 · 1人                    │  ← 默认折叠
│ ▾ 🧱 共建者 · 3人                    │  ← 展开
│   ┌──────────────────────────────┐   │
│   │ 👤 小云   💎 125 NT   🟢     │   │  ← 成员行
│   │ 👤 朝林   💎 80 NT    🔴     │   │
│   │ 👤 淑惠   💎 45 NT    🔴     │   │
│   └──────────────────────────────┘   │
│ ▸ ⚔️ 冒险者 · 5人                    │  ← 折叠
│ ▸ 👥 NPC · 2人                       │
│ ▸ 🏕️ 云村民 · 3人                    │
```

**交互：**
- 搜索框输入 → 实时过滤，跨分组显示匹配结果
- 分组标题点击 → 折叠/展开。默认展开第一个有人的分组
- 点击成员行 → 阶段 3 做详情卡，本阶段 toast 显示名字+角色+NT

### 4.3 Tab 2：📜 日志

```
│ ── 最近 50 条 ──────────────────   │
│                                      │
│ 7/19 14:30  📋 砚仁 发布了「厨房打扫」· 5 NT  │
│ 7/19 14:25  🔄 朝林 从 云村民 升级为 冒险者   │
│ 7/19 14:20  💬 砚仁 → 小云 🌹 1 NT          │
│ 7/19 10:00  ✅ 砚仁 审核通过 小云 →「厨房打扫」│
│ ...                                  │
│                                      │
│ 显示 50/128 条 · [加载更多]           │
│                                      │
│ [📥 导出日志]                        │  ← 阶段 5 做
```

**日志类型图标：**

| type | 图标 | 格式 |
|------|------|------|
| `task_post` | 📋 | `{user} 发布了「{taskName}」· {amount} NT` |
| `task_claim` | ✋ | `{user} 领取了「{taskName}」` |
| `review_approve` | ✅ | `{reviewer} 审核通过 {user} →「{taskName}」` |
| `review_reject` | ❌ | `{reviewer} 退回 {user} →「{taskName}」` |
| `tip` | 💬 | `{from} → {to} 🌹 {amount} NT` |
| `encourage` | 💬 | `{from} → {to} 👏太棒了` |
| `role_change` | 🔄 | `{user} 从 {oldRole} 升级为 {newRole}` |
| 其他 | 📝 | 直接显示 `text` |

## 五、与信箱/工作台的关系

```
工作台 📬 信箱           地图 大地书房→📚 档案室    工作台 📚 归档文件
─────────────────      ──────────────────────    ─────────────────
通知：现在有什么事       归档：过去发生了什么       快捷统计入口
面向个人                面向全社区                面向个人
实时计算，按人聚合      永久记录，时间倒序         统计数字 + 跳转链接
点击 → 行动             浏览 → 查阅              点击 → 打开档案室 overlay
顶部 📬 图标            地图建筑 / 社区链接        任务 Tab 底部
```

**三层各司其职，不互相替代。** 工作台的 📚 归档文件改为跳转链接——点击后打开 `#overlayArchive`，不再内嵌渲染。
