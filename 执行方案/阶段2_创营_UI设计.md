## 一、7 步总览与翻页机制

### 1.1 流程

```
① 创营仪式 → ② 时间参数 → ③ 财务参数 → ④ 任务参数 → ⑤ 人员选择 → ⑥ 分包确认 → ⑦ 启营仪式
```

每步一个独立 overlay（复用同一个 overlay 容器，切换内容）。底部固定翻页按钮。← 可回退，已填内容保留在内存中（一个 `_campDraft` 对象）。

### 1.2 翻页按钮规则

| 步 | 左侧按钮 | 右侧按钮 |
|----|---------|---------|
| ① | ✕ 关闭（放弃创建） | `[✨ 点燃火种 →]`（确认后触发动画 → 自动进入②） |
| ②-⑤ | `[← 上一步]` | `[下一步 →]` |
| ⑥ | `[← 上一步]` | `[📤 发送确认请求]`（全部确认后变为 `[🎊 启营仪式 →]`） |
| ⑦ | 无（仪式进行中） | `[🏁 正式启动]`（动画完成后出现） |

### 1.3 草稿保留

```javascript
var _campDraft = {
  step1: { name, season, type, theme, desc, testMode },
  step2: { startDate, days, template, schedule: [], milestones: [] },
  step3: { adventurers, builders, earlyBirdPrice, earlyBirdPct, fullPrice,
          住宿单价, 住宿NT, 吃饭单价, 吃饭NT, extraIncome: [], extraExpense: [] },
  step4: { tasks: [] },
  step5: { builders: [] },
  step6: { sentAt: null, confirmations: {} },
  step7: { launched: false }
};
```

← 回退时从 `_campDraft` 恢复对应步骤的表单值。✕ 关闭时 `_campDraft = null` 丢弃。

### 3.2 交互细节

**编辑时段格：** 点击任意格 → 底部 Sheet 滑入，包含一个 textarea + 时间选择 +「确定」按钮。点击确定 → Sheet 关闭，格子内容更新。

**加天/减天：** 在日历末尾追加/删除一天。减天时有内容的天提示「该天有日程安排，确认删除？」。

**加时段：** 在时段列表末尾追加一行（默认时间 = 上一时段 + 4 小时）。最多 6 个时段。

**复制前一天：** 将前一天的全部格子内容复制到当前选中的天。按钮旁有一个小的日期选择器。

**存为模板：** 将当前日历存入 `localStorage` 的 `schedule_templates` 键。下次创营时「模板」下拉可选。

**生成里程碑：** 根据开营日期 + 天数，自动计算 4 个里程碑日期。里程碑日期可手动覆盖（点🔒解锁）。

### 4.2 实时计算逻辑

```javascript
function calcStep3() {
  var s = _campDraft.step3;
  var adventurers = s.adventurers || 0;
  var builders = s.builders || 0;
  var totalPeople = adventurers + builders;
  var days = _campDraft.step2.days || 1;

  // 收入
  var earlyBirdIncome = adventurers * s.earlyBirdPrice * (s.earlyBirdPct / 100);
  var fullPriceIncome = adventurers * s.fullPrice * ((100 - s.earlyBirdPct) / 100);
  var extraIncome = (s.extraIncome || []).reduce(function(sum, item) { return sum + item.amount; }, 0);
  var totalRmbIncome = earlyBirdIncome + fullPriceIncome + extraIncome;

  // 支出
  var lodgingRmb = s.lodgingRmb * totalPeople * days;
  var lodgingNT = s.lodgingNT * totalPeople * days;
  var mealRmb = s.mealRmb * totalPeople * days;
  var mealNT = s.mealNT * totalPeople * days;
  var extraExpenseRmb = (s.extraExpense || []).reduce(function(sum, item) { return sum + item.amount; }, 0);
  var totalRmbExpense = lodgingRmb + mealRmb + extraExpenseRmb;
  var totalNTExpense = lodgingNT + mealNT;

  return {
    incomeRmb: totalRmbIncome, incomeNT: 0,
    expenseRmb: totalRmbExpense, expenseNT: totalNTExpense,
    balanceRmb: totalRmbIncome - totalRmbExpense,
    balanceNT: 0 - totalNTExpense
  };
}
```

每次 input 变更时调用，更新底部预测损益卡片的数字和颜色。

## 八、⑦ 启营仪式

### 8.1 仪式动画

全部共建人确认后，管理员点「🎊 启营仪式」：

```
阶段 1（0.6s）：全屏变暗，⚔️ Logo 从顶部坠入中心（ease: Bounce.easeOut）
阶段 2（0.5s）：光芒从 Logo 扩散，金色粒子向外飞出
阶段 3（0.8s）：营队名 + 日期 + 主题 依次浮现（stagger 0.1s）
阶段 4（1.2s）：每位共建者头像卡片从右侧依次滑入
               （stagger 0.15s，每张卡片：头像+名字+任务数+NT）
阶段 5（0.6s）：「正式开营」按钮从底部弹入（弹性动画）
               停留 0.5s → 所有元素定格
```

总时长约 4 秒。动画期间不可交互。

### 8.2 启动确认

动画完成后，显示确认卡片：

```
┌──────────────────────────────────────┐
│          🏁 营队正式启动              │

│   ⚠️ 启动后原档锁定，不可修改         │

│   2位共建者 · 8个任务                 │
│                                      │
│   [🏁 正式启动]    [← 回去修改]       │
```

点击「正式启动」→ `AppData.sharedSet('camps', ...)` 写入 → 营队状态 `active` → `logActivity('camp_launch', ...)` → toast「🔥 营队正式启动！」→ 关闭所有创营 overlay → 回到社区主页（主页自动刷新，新营队出现在「进行中」）。
