```
Step 0：地图桥接补全      ← 合并原 S5
Step 1：S1 数据清理
Step 2：空间语义 + 物品     ← 合并原 任务05+10
```

## 移动端代码模块化策略

执行后期（Step 5 开始），代码量将超过 3000 行，单文件维护成本非线性增长。参照 [App 架构方案](../南塘云村App/架构方案_导航与模块骨架.md) 的 Hub-and-Spoke 模型，按以下模块拆分：

```
nantang-mobile/
├── index.html              ← 入口，加载所有模块
├── core/
│   ├── data.js             ← 共享数据层（nt_users/nt_spaces/nt_ledger/…）
│   ├── eventbus.js         ← 模块间事件通信
│   └── storage.js          ← localStorage/IndexedDB 封装
├── modules/
│   ├── village/            ← 村口（Hub 根节点，三张轮播卡）
│   ├── map/                ← 实景地图（空间交互 + 物品 CRUD + 打扫状态灯）
│   ├── tasks/              ← 任务大厅（发布/认领/审核/结算）
│   ├── events/             ← 社区副本（活动列表 + 报名）
│   ├── profile/            ← 我的工作台（四 Tab：任务/账本/物品/设置）
│   ├── economy/            ← NT/CV/XP 三层引擎 + 虚拟账本
│   └── admin/              ← 管理后台（Step 8）
└── ui/
    ├── components/         ← 可复用 UI 组件（玩家卡/Toast/Modal/状态灯）
    └── theme.css           ← 全局视觉 token
```

- `ponytail:` 单文件超过 3000 行后拆分为 `<script src="modules/...">` 加载。当前阶段不建模块框架——只在实际需要时拆分。
- 拆分时从 App 架构的模块边界接口（§4.1-4.3）直接复用——不重新设计。

## 各阶段概要

### Step 0：地图桥接补全
- **文档**：[Step0_地图桥接_执行方案.md](Step0_地图桥接_执行方案.md)
- **来源**：`地图融合方案.md`
- **做什么**：iframe 孤岛 → 主应用和地图互通。补全 `window.Game` 桥接对象。
- **验**：地图按钮能打开主应用页面，用户名显示当前用户，NT 操作生效。

### Step 1：数据清理
- **文档**：[Step1_数据清理_执行方案.md](Step1_数据清理_执行方案.md)
- **来源**：`硬编码清理_完整方案.md`
- **做什么**：清掉所有硬编码假数据。App 从零状态启动。
- **验**：首次打开无预设数据，注册第一个用户后正常使用。

### Step 2：空间语义 + 物品标记
- **文档**：[Step2_空间与物品_执行方案.md](Step2_空间与物品_执行方案.md)
- **来源**：`02_空间地图与物品标记.md`（主）、`07_管理后台深化.md`（存储分层）
- **做什么**：地图可交互。空间定义→子区域→物品CRUD→过期追踪。
- **验**：点地图厨房→看到冰箱上层有白菜×2→可消耗/查看详情。

### Step 3：打扫系统
- **文档**：[Step3_打扫系统_执行方案.md](Step3_打扫系统_执行方案.md)
- **来源**：`03_卫生脏污度与打扫审核.md`
- **做什么**：脏污度自动增长 + 前后拍照 + 复议制审核 + 奖励发放。
- **验**：空间变脏→拍照打扫→复议通过→脏污度归零+发奖励。

### Step 4：NT/CV/XP 三层扩展
- **文档**：[Step4_NT三层扩展_执行方案.md](Step4_NT三层扩展_执行方案.md)
- **来源**：`南塘NT经济系统_完整设计.md`（主）、`01_身份住宿与NT基础.md`（用户字段）
- **做什么**：NT 一层 → NT+CV+XP 三层 + 虚拟账本 + 公共贡献池。
- **验**：`verifyLedgerBalance()` 三维全过。打扫 +40 CV +40 XP。购买 CV 75/25 分流。

### Step 5：社区活动
- **文档**：[Step5_社区活动_执行方案.md](Step5_社区活动_执行方案.md)
- **来源**：`南塘社区空间_设计方案.md`（主）、`01_身份住宿与NT基础.md`（新手任务/CV抵押）
- **做什么**：6 空间模块 + 新手任务 + 邀请码。P0→P1→P2 分批。
- **验**：P0 可交易+订餐。P1 可住宿+组局+新手引导。P2 有提醒+HUB。

### Step 6：统一时间线
- **文档**：[Step6_时间线_执行方案.md](Step6_时间线_执行方案.md)
- **来源**：`05_在地记录与NT激励.md`（主）、`06_云村民互动层.md`（点赞/留言/打赏）
- **做什么**：所有事件汇入一条时间线。发记录+点赞+留言+打赏+名片弹窗。
- **验**：发记录→出现在时间线→+CV+XP→可按空间/按人过滤→可点赞/留言/打赏。

### Step 7：服务器 + 多用户
- **文档**：[Step7_服务器部署_执行方案.md](Step7_服务器部署_执行方案.md)
- **来源**：`多人部署方案.md`
- **做什么**：localStorage → SQLite + Express + SSE 实时推送 + 乐观更新。
- **验**：5 人同时操作→实时同步→冲突自动重试→NT/CV/XP 分毫不差。

### Step 8：管理后台
- **文档**：[Step8_管理后台_执行方案.md](Step8_管理后台_执行方案.md)
- **来源**：`07_管理后台深化.md`
- **做什么**：NT 总览/流向图/物品报表/打扫统计/提案投票/数据导出。
- **验**：所有统计面板数据正确，提案可发起和投票。

## 明确排除

| 内容 | 原因 |
|------|------|
| 日限 30 NT | 熟人社区不存在灌水土壤，过度设计 |
| 排班系统 | MVP 3-5 人口头协调更灵活 |
| 弹幕 | 实现成本高、与留言重叠、边际价值低 |
| 互评点数分配法 | 治理哲学，不是软件功能 |
| NT 信息市场 | 付费查档案太重 |
| 注册赠送 100 NT | 已改为新手任务赚取 |
| 累进税（CV>2000 征 5%） | P2 延后 |
| NFT 铸造 | 依赖 web3 基础设施，远期 |
| 事件回放 | 远期，不急 |
