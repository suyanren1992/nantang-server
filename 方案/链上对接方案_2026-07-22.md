## 二、四层资金模型

```
┌─────────────────────────────────────────────┐
│            OP Chain（链上）                   │
│  ┌───────────────────────────────────────┐  │
│  │  PLATFORM_WALLET（多签 3/5）           │  │
│  │  持有全部社区 NT 代币                   │  │
│  │  审计等式：chain_balance ≥ total_issued │  │
│  └──────────┬────────────────────────────┘  │
│             │ deposit / withdraw             │
│  ┌──────────┴────────────────────────────┐  │
│  │  用户个人钱包                           │  │
│  │  User.wallet_address                   │  │
│  └───────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
                    │
                    │ ChainScanner 同步
                    ▼
┌─────────────────────────────────────────────┐
│          CommunityPool（链下账本）             │
│  ┌──────────┬──────────┬──────────┐         │
│  │ balance  │ reserve  │ task_escrow│         │
│  │ (运营池) │ (储备池)  │ (托管冻结) │         │
│  ├──────────┼──────────┼──────────┤         │
│  │ frozen   │ camp_balance│ total_issued│     │
│  │ (争议冻结)│ (营队资金) │ (发行总量) │         │
│  └──────────┴──────────┴──────────┘         │
│  等式：∑user.balance + ∑pool.* = total_issued│
└─────────────────────────────────────────────┘
                    │
                    │ _add_ledger / NTLedger
                    ▼
┌─────────────────────────────────────────────┐
│              User（用户）                     │
│  nt_balance = 链下可用余额                    │
│  wallet_address = 提现目标地址                 │
└─────────────────────────────────────────────┘
```

**核心约束**：`chain_balance >= pool.total_issued`（多签钱包余额 ≥ 链下发行总量）。如果这个等式破裂，说明链下记账出了多签钱包实际余额无法兑付的 NT——这就是"资不抵债"。

## 三、充值流程（已有 ✅，加 reserve 分流）

### 当前流程
```
用户 → POST /api/nt/deposit-intent (amount: 100)
     → 返回 intent_id + to_address (PLATFORM_WALLET)
     → 用户用个人钱包向 PLATFORM_WALLET 转 100 NT
     → ChainScanner 检测到交易
     → 调 topup(user, 100)
     → user.nt_balance += 100
     → pool.total_issued += 100
```

### 改进：充值分流到 reserve
```python
# chain_scanner.py _process_log() 中：
if intent and intent.status == "pending":
    intent.status = "detected"
    user.nt_balance += amount
    pool = await _get_pool(db, lock=True)
    pool.total_issued += amount
    pool.reserve += amount    # ← 新增：充值直接进储备池
    # 不再进 balance——充值不是日常收入，是长期储备
```

用户提现时从 reserve 扣（而非 balance），保证运营池不受充值/提现波动影响。

## 四、提现流程（新增 · 两阶段冻结）

> **6 Agent 审查修正**：原方案在链上转账前直接扣款销毁 total_issued，存在资金消失风险。
> 改为两阶段：先冻入 frozen → admin 多签确认链上转账完成 → 再从 frozen 扣减。

**第一阶段：用户发起提现（冻结）**

```python
@router.post("/withdraw")
async def withdraw(req: EarnSpendRequest, user: User = Depends(get_current_user),
                    db: AsyncSession = Depends(get_db)):
    """用户提现 NT——两阶段：先冻结到 frozen，管理员确认后释放"""
    from web3 import Web3
    
    # 输入校验
    if not user.wallet_address or not Web3.is_address(user.wallet_address):
        raise HTTPException(400, "请先设置有效的钱包地址")
    if req.amount < 50 or req.amount > 200:
        raise HTTPException(400, "单次提现 50-200 NT")  # per-day上限200
    
    # 冷却检查（7天内最多1次）
    week_ago = (datetime.utcnow() - timedelta(days=7)).isoformat()
    recent = (await db.execute(
        select(func.count(NTLedger.id)).where(
            NTLedger.from_user == user.id, NTLedger.type == "withdraw",
            NTLedger.created_at >= week_ago
        )
    )).scalar()
    if recent > 0:
        raise HTTPException(429, "7天内已提现过，请等待冷却期结束")
    
    if user.nt_balance < req.amount:
        raise HTTPException(400, f"余额不足（当前 {user.nt_balance} NT）")
    if user.trust_score < 60:
        raise HTTPException(400, "信誉不足，需可信等级(≥60)方可提现")
    
    pool = await _get_pool(db, lock=True)
    if pool.reserve < req.amount:
        raise HTTPException(400, f"储备池余额不足（当前 {pool.reserve} NT），请联系管理员")
    
    # 第一阶段：冻结（不销毁 total_issued）
    user.nt_balance -= req.amount
    pool.reserve -= req.amount
    pool.frozen += req.amount
    
    lid = _ledger_id()
    await _add_ledger(db, lid, user.id, None, req.amount, "withdraw",
                      f"提现至 {user.wallet_address[:10]}... 等待管理员签名", status="pending")
    await _adjust_trust(user, -10)  # 提现降低信任
    await db.commit()
    return {"ok": True, "entry_id": lid, "balance": user.nt_balance,
            "status": "pending", "expected_time": "24小时内"}
```

**第二阶段：管理员确认（释放冻结）**

```python
@router.post("/admin/withdraw/confirm")
async def confirm_withdraw(entry_id: str, admin: User = Depends(require_admin),
                            db: AsyncSession = Depends(get_db)):
    entry = (await db.execute(
        select(NTLedger).where(NTLedger.entry_id == entry_id, NTLedger.status == "pending")
    )).scalar_one_or_none()
    if not entry: raise HTTPException(404, "提现记录不存在或已处理")
    
    pool = await _get_pool(db, lock=True)
    pool.frozen -= entry.amount
    pool.total_issued -= entry.amount  # 链上确认后才减少发行量
    entry.status = "settled"
    entry.settled_at = datetime.utcnow().isoformat()
    await db.commit()
    return {"ok": True}

@router.post("/admin/withdraw/reject")
async def reject_withdraw(entry_id: str, admin: User = Depends(require_admin),
                           db: AsyncSession = Depends(get_db)):
    entry = (await db.execute(
        select(NTLedger).where(NTLedger.entry_id == entry_id, NTLedger.status == "pending")
    )).scalar_one_or_none()
    if not entry: raise HTTPException(404)
    
    pool = await _get_pool(db, lock=True)
    pool.frozen -= entry.amount
    pool.reserve += entry.amount  # 退回储备池
    # 退回用户余额
    user = (await db.execute(select(User).where(User.id == entry.from_user))).scalar_one_or_none()
    if user: user.nt_balance += entry.amount
    entry.status = "cancelled"
    await db.commit()
    return {"ok": True}

# 72h 超时自动回滚（在 daily_tick 中）
pending_withdraws = await db.execute(
    select(NTLedger).where(NTLedger.type == "withdraw", NTLedger.status == "pending",
                           NTLedger.created_at < three_days_ago)
)
for entry in pending_withdraws.scalars():
    pool.frozen -= entry.amount
    pool.reserve += entry.amount
    entry.status = "cancelled"
```

**批量结算**：
- `GET /api/admin/withdraws/pending` — 列出所有 pending 提现，按金额排序
- `POST /api/admin/withdraws/batch-settle` — 将选中的 pending 条目批量设为 `submitted`（"已提交多签"）
- `POST /api/admin/withdraws/batch-confirm` — 批量 settled（接收 `{ledger_ids: [...], tx_hash: "0x..."}` ）
- `GET /api/nt/withdraws` — 当前用户自己的提现历史（含状态：pending/submitted/settled/cancelled）
- 日终对账：`sum(pending + submitted)` = "应付未付链上负债"，与 `chain_balance` 对比

**链上充值确认时间**：ChainScanner 轮询间隔 30-60 秒（`SCAN_INTERVAL` 环境变量）。用户创建 deposit intent 后预期 1-2 分钟内到账。

## 五、链上校核机制

### 每日对账（在 daily_tick 中）

```python
# 5. 链上对账：多签余额 vs 发行总量
if PLATFORM_WALLET:
    try:
        from web3 import Web3
        rpc = os.environ.get("OP_RPC_URL", "https://mainnet.optimism.io")
        token_addr = os.environ.get("NT_TOKEN_CONTRACT", "")
        if token_addr:
            w3 = Web3(Web3.HTTPProvider(rpc))
            abi = '[{"constant":true,"inputs":[{"name":"_owner","type":"address"}],"name":"balanceOf","outputs":[{"name":"","type":"uint256"}],"type":"function"},{"constant":true,"inputs":[],"name":"decimals","outputs":[{"name":"","type":"uint8"}],"type":"function"}]'
            c = w3.eth.contract(address=w3.to_checksum_address(token_addr), abi=abi)
            decimals = c.functions.decimals().call()
            raw = c.functions.balanceOf(w3.to_checksum_address(PLATFORM_WALLET)).call()
            chain_balance = raw // 10**decimals
            # 告警：链上余额 < 发行总量（多签钱包不够兑付）
            if chain_balance < pool.total_issued:
                shortage = pool.total_issued - chain_balance
                logger.error(f"CHAIN SHORTFALL: 链上 {chain_balance} < 发行 {pool.total_issued}，缺口 {shortage} NT")
                results["chain_shortfall"] = shortage
            results["chain_balance"] = chain_balance
    except Exception as e:
        logger.warning(f"chain audit skipped: {e}")
```

**告警不等式中断时**：说明有人取得了链下 NT 但链上多签钱包从未收到对应充值——要么充值未到账，要么 ledger 有误。

## 六、个人钱包 + 校核联动

### 用户 profile 中的钱包
```
User.wallet_address = "0x..."
```
- 注册时可填（可选）
- 个人资料页可修改
- 提现时必须有

### 校核制 + 钱包绑定
```
劳动 → addVerification → approve → doer.nt_balance += nt_amount
                                         │
                                    doer 随时可提现到个人钱包
                                    POST /api/nt/withdraw
```

**设计意图**：日常劳动赚 NT 在链下累积，用户觉得够多了就提现到个人钱包。提现触发多签钱包转账——每笔提现都是链上可见的。
