# 重置数据库 — 安全执行方案

> 2026-07-22 · 目标：初始金库 500 NT 生效 · 零误操作

---

## 为什么不直接删

直接删 `nantang_fresh.db` 会丢失所有注册用户/任务/流水。方法 A 不删库，更安全。

## 方法 A（推荐）：不改库，只更新值

```bash
# 停服务器
Ctrl+C

# 进入 server 目录
cd C:\Users\苏砚仁\thinknote\项目\实景游戏移动端代码\server

# 一行命令更新金库
python -c "
import sqlite3
conn = sqlite3.connect('nantang_fresh.db')
conn.execute('UPDATE community_pool SET balance=500, total_issued=500')
conn.commit()
conn.close()
print('OK: 金库已设为 500 NT')
"

# 重启服务器
python main.py
```

**不会丢任何数据。** 只是把池子余额从 0 改成 500。

## 方法 B：彻底重置（仅当不需要旧数据时）

```bash
# 1. 先备份（防误触）
copy server\nantang_fresh.db server\nantang_fresh_backup.db

# 2. 停服务器
Ctrl+C

# 3. 删库
del server\nantang_fresh.db

# 4. 重启（自动创建新库 + 初始金库 500）
python server\main.py
```

备份文件 `nantang_fresh_backup.db` 保留，万一需要恢复。
