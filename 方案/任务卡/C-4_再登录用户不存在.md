# 复修卡 C-4：退出后再次登录提示"用户不存在"（严重）

> 施工方：Kimi Code · 先查后改 · 单独 commit，不 push

## 现象
注册成功后退出，再次登录时提示**用户不存在**，登不上。

## 排查方向（先只读，结论写 BUG_TRACKER.md）
1. 注册是否真的写进了服务器库：注册后查服务器 users 表有没有这个人（POST /api/auth/register 的落库链路）
2. 登录走的是服务器校验还是本地 localStorage 缓存：退出时是否清了不该清的本地数据，或登录请求参数（名字编码）与注册存的不一致
3. 服务器是否在用临时/内存数据库：Render 重启后数据是否丢失（若是 SQLite，数据库文件路径是否在持久磁盘上——Render 免费版磁盘重启即清）
4. 特别注意第 3 条：如果是"每次部署/重启后全库清空"，这是架构级问题，先报告不要瞎修

## 修法
按断点修；如果是 SQLite 持久化问题，给出方案（Render Disk 挂载 或 迁 Postgres）在 BUG_TRACKER.md 里写清楚再动手。

## 验收
注册→退出→再登录成功；服务器重启后账号仍在。
commit：`fix(C-4): 登录态用户持久化修复（断点:___）`
