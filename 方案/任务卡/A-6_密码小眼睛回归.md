# 任务卡 A-6：注册/登录密码框补回"小眼睛"查看（回归修复）

> 施工方：Claude Code · 验收方：Kimi Work · 单独 commit，不 push

## 问题现象
以前注册页和登录页的密码输入框有个小眼睛图标可以点击查看明文，现在没了。

## 位置
`nantang-mobile/index.html`
- 第 55 行：`<input type="password" id="regPwd" class="login-input" placeholder="设置密码（至少8位）">`
- 第 66 行：`<input type="password" id="loginPwd" class="login-input" placeholder="密码">`

## 改法（只改 index.html + 挂一个全局函数到 core.js）

1. 把两个 input 各包一层相对定位容器，右侧放眼睛按钮（以内联样式为主，与项目现有风格一致）：

```html
<div style="position:relative">
  <input type="password" id="regPwd" class="login-input" placeholder="设置密码（至少8位）" style="padding-right:38px">
  <span onclick="togglePwdEye('regPwd',this)" style="position:absolute;right:10px;top:50%;transform:translateY(-50%);cursor:pointer;font-size:.9rem;opacity:.7;user-select:none">👁</span>
</div>
```

loginPwd（第 66 行）同样处理。

2. 在 `nantang-mobile/js/core.js` 任意工具函数区（如 `initCarousel` 之前）加：

```javascript
function togglePwdEye(id, el){
  var i=document.getElementById(id);if(!i)return;
  i.type = i.type==='password' ? 'text' : 'password';
  if(el) el.textContent = i.type==='password' ? '👁' : '🙈';
}
```

3. 登录/注册逻辑读的是 `input.value`，与 type 无关，确认不需要改 enterVillage 等任何逻辑。

## 验收
- `node --check nantang-mobile/js/core.js` 通过
- 注册页、登录页密码框右侧出现 👁；点一次变明文（🙈），再点变回密码
- 注册、登录流程不受影响（能正常提交）
- `git status` 只有 index.html 和 core.js 两个文件
