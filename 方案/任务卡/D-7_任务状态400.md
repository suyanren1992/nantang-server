# 任务卡 D-7：任务状态变更每次触发 400 静默失败 🟠

> 施工方：Claude Code（一营） · 监察：Kimi Code · 验收：Kimi Work · 单独 commit，不 push
> 来源：《全面权限扫描_B方案_2026-07-24》H-7

## 位置
- `nantang-mobile/js/app-data.js:138` `updateTask` 无条件把 `updates`（含 status）发给 `API.syncTaskUpdate`
- `server/routes/tasks.py:142` `TaskUpdate` 显式拒绝 status 字段（400「状态请通过专用端点变更」）——服务端行为正确，**不动**

## 问题
每次任务状态流转都额外产生一次必然 400 的请求，浪费带宽、污染日志，且让"同步失败"类监控失真。

## 修法
- `syncTaskUpdate` 调用前：若 `updates` 含 `status` 键，剔出 status 后再发；剔除后无任何剩余字段则跳过请求直接 resolve
- 检查其他调用 `syncTaskUpdate` 的路径，确保状态字段一律走专用端点（cancel/submit/verify），不走通用 update
- 遵守铁律 6：升 app-data.js 的 ?v=

## 验收
- 认领/提交/取消任务时，DevTools Network 不再出现 400 的 `PUT /api/nt/tasks/{id}`
- 非状态字段（note/evidence 等）的 update 仍正常同步
- commit message：`fix(D-7): updateTask 同步前剔除 status 字段（H-7 每次状态流转必 400）`
