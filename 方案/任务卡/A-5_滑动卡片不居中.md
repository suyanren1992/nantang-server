# 任务卡 A-5：全貌页滑动卡片不居中（绿框偏左、半边出屏）

> 施工方：Claude Code · 验收方：Kimi Work · 单独 commit，不 push

## 问题现象
村落地页下方滑动模块（villageCarousel）的卡片偏在左边，一半滑出手机屏幕外。

## 根因（已定位）
- 卡片宽 260px（`nantang-mobile/css/main.css:104`，`.vp-card{flex:0 0 260px;scroll-snap-align:center}`）
- `nantang-mobile/js/core.js:1544` `initCarousel()` 里居中算法**没有补偿容器与卡片的宽度差**：
  - 第 1549 行：`c.scrollLeft=pw;`（只滚了一个卡宽，没减偏移）
  - 第 1566 行：圆点点击 `c.scrollTo({left:pw1*i,...})` 同样缺偏移

## 改法（只改 `nantang-mobile/js/core.js` 一个文件）
在 `initCarousel()` 内定义统一公式并替换两处：

```javascript
// 卡片 i 居中所需的 scrollLeft（260px 卡 + scroll-snap center）
function _cardLeft(i){ return pw*i - Math.max(0,(c.offsetWidth-pw)/2); }
```

1. 第 1547-1550 行 requestAnimationFrame 里：`c.scrollLeft = _cardLeft(1);`（初始居中第 2 张卡，保持原意图）
2. 第 1566 行圆点点击：`c.scrollTo({left:_cardLeft(i), behavior:'smooth'})`
3. 第 1556-1557 行 scroll 事件里算 idx 也要同步：`var idx = Math.round((c.scrollLeft + Math.max(0,(c.offsetWidth-pw,1)/2... 保守做法：`Math.round((c.scrollLeft + Math.max(0,(c.offsetWidth-pw)/2)) / Math.max(pw,1))`，仍 clamp 0-2。

注意 pw 在每处都是现取的（已有 `var pw=...260` 兜底），沿用即可。
