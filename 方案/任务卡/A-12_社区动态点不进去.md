# 任务卡 A-12：全貌页"社区动态"点不进去 + 明晰用途

> 施工方：Claude Code · 验收方：Kimi Work · 单独 commit，不 push

## 问题现象
全貌页"👥 社区动态"板块：用户反馈点击没反应、也看不懂这三个折叠区是干什么的。

## 位置
- `_renderCardRoomSection()`：`nantang-mobile/js/app.js:401-410`（三个折叠区：在地人员/活跃营地/时间线）
- `_collapsibleSection()`：`nantang-mobile/js/app.js:411-417`（手风琴，header 行内 onclick 切换 body 显隐）
- 营地行"进入 ▸"：`_renderCampSection()`（app.js 约 479 行起）

## 施工步骤

### 第 1 步：先诊断"点不进去"的真因（在浏览器验证，别猜）
代码层面 header 的行内 onclick 逻辑看起来是通的。重点怀疑三处，逐一排查：
1. 该板块是否被某个透明浮层盖住（检查 z-index / pointer-events）
2. 点击是否被父容器（`#cardRoomSection` 或 `.info-wrapper` 内其他带 onclick 的元素）拦截/冒泡冲突
3. `cr-header` 的 `lastElementChild` 取箭头 span 是否在某些渲染下错位

把诊断结论写在 commit message 或 BUG_TRACKER.md 里，再对症修。

### 第 2 步：明晰用途（无论真因是什么都做）
给 `_renderCardRoomSection()` 的三个 section 各加一行灰色小字说明，放在 title 下方（可给 sections 数组每项加 `desc` 字段，在 `_collapsibleSection` 里渲染）：

- 👤 在地人员 · `谁现在在村里，翻牌亮个相`
- 🏕️ 活跃营地 · `正在进行的营队活动，点"进入"看详情`
- 📜 时间线 · `村里最近发生的大事小情`

### 第 3 步：营地"进入 ▸"必须真的能点
检查 `_renderCampSection()` 里每行的"进入 ▸"是否绑了跳转；没绑就补上（跳对应营地页/弹窗，沿用项目现有的 openCamp 类函数；若没有现成函数，至少 toast"营地页施工中"并在 BUG_TRACKER.md 记录）。

### 第 4 步：可点感
`cr-header` 加 `min-height:44px`（触控热区），整行 hover/active 有反馈色。

## 验收
- 三个折叠区点开/收起正常，箭头 ▸/▾ 跟随切换
- 每个区有用途说明小字
- 营地"进入 ▸"有响应
- `node --check nantang-mobile/js/app.js` 通过，`git status` 只有 app.js（如需 CSS 微调可含 main.css）
- commit message：`fix(A-12): 社区动态点击修复（真因:___）+ 三区用途说明 + 营地进入接线`
