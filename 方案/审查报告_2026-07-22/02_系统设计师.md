## 🔴 严重（2 条）

### R12: access token 过期后无自动刷新，silentRefresh 从未被调用
{file: `nantang-mobile/js/api.js`, line: 54-71}

`silentRefresh` 定义了完整的 httpOnly cookie 刷新逻辑，但**整个代码库中无调用方**。token 60 分钟过期后所有 API 请求返回 401，用户必须手动刷新页面。长时间营地活动场景影响严重。

**建议**: (a) `_pollCycle` catch 中轮询调用；(b) `API.request` 收到 401 时自动试一次 refresh 后重试；(c) 进入村庄时启动定时器每 50 分钟刷新。

### R18: 无 token 过期前的预刷新机制（与 R12 同根）
{file: `server/auth_utils.py`, line: 25-28; `nantang-mobile/js/api.js`, line: 54-71}

服务端 access token 60 分钟过期，客户端无预刷新。用户 UI 正常（单页应用），但 NT 操作全部失败。

**建议**: poll cycle 30s 循环中 decode JWT exp，剩余 <5 分钟则调 `silentRefresh`。

## 🟡 重要（9 条）

### R1: /api/data/sync_all journal entries missing discovery_id
{file: `server/routes/data.py`, line: 329}

`sync_all` journal 返回 `type/content/time/space_id`，缺少 `discovery_id`。`/api/data/journal` 端点包含该字段。

### R2: /api/nt/sync 返回 verifier_id 但不返回 settler_id / settled_at
{file: `server/routes/nt.py`, line: 134}

任务序列化含 `verifier_id`/`verified_at`，缺 `settler_id`/`settled_at`。客户端 merge 也不映射这些字段——结算 UI 无法显示结算人。

### R3: /api/data/sync_all tasks 缺少 escrow_amount
{file: `server/routes/data.py`, line: 320-326}

`sync_all` 返回的 task 不含 `escrow_amount`（`/api/nt/sync` 包含）。争议仲裁需要此字段。

### R4: /api/data/sync_all 不返回 map_locations / canteen_menu 等共享数据
{file: `server/routes/data.py`, line: 356-359}

名为"全量同步"但只返回个人范围数据。共享数据需单独请求。

### R5: _mergeSyncData 和 _mergeNTSyncData 字段映射不一致
{file: `nantang-mobile/js/core.js`, line: 795-861}

两个 merge 函数处理的字段集不同。`_mergeNTSyncData` 丢弃 `settler_id`/`settled_at`/`verified_at`/`verifier_id`/`escrow_amount`。应用注释说明差异。

### R6: _mergeNTSyncData 在 NT 用户未注册时静默跳过余额同步
{file: `nantang-mobile/js/core.js`, line: 799-805}

`NT.getUser(me)` 返回 null 时整个 balance/CV/XP 同步块被跳。若注册因异常失败，数据静默丢失。

### R8: /api/system/daily-tick 住宿费每租户独立幂等，但整体无事务保证
{file: `server/routes/nt.py`, line: 842-884}

双重保护（`last_deducted` + `last_tick_date`）。tick 中途失败场景分析——已扣费的因 `last_deducted==today` 跳过，后半部分重扣——**实际正确**。

### R9: cron tick_daily 与 endpoint daily_tick 共享 CommunityPool 但无互斥
{file: `server/cron.py`, line: 55-119; `server/routes/nt.py`, line: 842-884}

cron 和 endpoint 都操作 `CommunityPool.balance`。endpoint 有 `lock=True`，cron 无 `with_for_update()`。00:05 恰好用户登录的极端概率场景。

**建议**: cron 的 `_get_pool` 也加 `lock=True`。

### R13: 401 响应未正确触发 poll 停止
{file: `nantang-mobile/js/core.js`, line: 269-270}

`_pollCycle` 检查 `srv.detail === 'unauthorized'` 来停轮询，但 `API.request` 401 返回 `error: '登录过期'`。条件永远不匹配——轮询持续向服务器发 401。

### R16: offline 路径下 enterVillage() 分支结构复杂
{file: `nantang-mobile/js/core.js`, line: 867-968}

`isReg`/`isLogin` 分支中 HTTP 和 file:// 路径交织，`_finishEnter` 的 fall-through 模式跟踪困难。
