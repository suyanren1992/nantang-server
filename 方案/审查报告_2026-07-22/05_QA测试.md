## 🔴 严重（6 条）

### B1: create_camp 接受裸 dict，无 Pydantic 模型
{file: `server/routes/camps.py`, line: 39}

`async def create_camp(req: dict, ...)` 所有字段走 `dict.get()`。无类型/长度/范围校验。负数 `lodgingNT` × `people × days` → 负 `camp_total` → 污染 `pool.camp_balance`。

### B2: reward 无上限
{file: `server/routes/tasks.py`, line: 20}

`reward: int = 5` 无 `ge`/`le` 约束。仅检查 `<= 0`。`reward=999_999_999` 通过，社区任务耗尽 `pool.balance`。

### B3: bio 和 location 无长度限制
{file: `server/routes/auth.py`, line: 160}

`UpdateProfileRequest.bio` 和 `location` 为 `str | None = None` 无 `max_length`。可 POST MB 级字符串到 SQLite TEXT 列。

### B11: chain_scanner 无 with_for_update() 在用户 NT 余额更新
{file: `server/chain_scanner.py`, line: 163}

`user.nt_balance += amount` 在 `async with db:` 内但无行锁。两笔链上 Transfer 事件同一区块命中同一用户时丢失更新。

### B12: transfer 无行锁
{file: `server/routes/nt.py`, line: 200-201}

`from_user_obj.nt_balance -= req.amount` 无 `with_for_update()`。两笔并发转账可双双通过余额检查导致透支。

### B13: CommunityPool 初始化竞态
{file: `server/database.py`, line: 36-38}

```python
r = await session.execute(select(CommunityPool).limit(1))
if not r.scalar_one_or_none():
    session.add(CommunityPool(...))
```

无唯一约束（除 autoincrement PK）。并发 `init_db()` 或 `init_db()` 与首次请求的 `_get_pool` 竞态可创建两个池行。后续 `.limit(1)` 只读第一个。

## 🟡 主要（5 条）

### B4: 无最大密码长度
{file: `server/routes/auth.py`, line: 62}

bcrypt 在 72 字节处静默截断。10KB 密码哈希与 72 字节前缀相同但用户不知。

### B5: spend/topup/cashout/deposit-intent 无上限
{file: `server/routes/nt.py`, line: 229,256,288,347}

四个端点仅检查 `<= 0` 无 `> MAX`。`transfer` 则有 10000 上限——不一致。

### B14: spend 无行锁
{file: `server/routes/nt.py`, line: 236-244}

`user.nt_balance -= req.amount` 无 `with_for_update()`。

### B15: settle_task 无 with_for_update()
{file: `server/routes/nt.py`, line: 604}

`settle_task` 用 `select(NTTask).where(...)` 无锁。`cancel_task` 正确使用了 `.with_for_update()`。

### B34/B35: fetch 无 AbortController timeout + JSON 解析错误被网络错误掩盖
{file: `nantang-mobile/js/api.js`, line: 21, 25}

`fetch` 无 `signal`——网络挂起无限等待。服务端返回非 JSON（如 HTML 502）时 `resp.json()` 抛 `SyntaxError`，被 catch 报告"网络异常"，真实原因被掩盖。

## 🟢 建议（1 条）

### B36: 客户端 encodePassword file:// 模式 only
{file: `nantang-mobile/js/auth.js`, line: 393}

`v2$` 盐化迭代哈希仅在 file:// 协议，HTTP 模式服务端处理 bcrypt。安全。
