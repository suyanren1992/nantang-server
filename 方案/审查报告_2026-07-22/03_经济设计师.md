## 🔴 严重（5 条）

### 1. chain_scanner 静默跳过 total_issued
{file: `server/chain_scanner.py`, line: 167-169}

`_process_log()` 用 `select(CommunityPool).limit(1).scalar_one_or_none()` 查池，返回 None 时静默跳过 `pool.total_issued += amount`。但 `user.nt_balance += amount` 已执行——等式破裂。

**修复**: 改用 `from nt_helpers import _get_pool`（所有其他模块均已使用）。

### 2. earn-sync 端点无服务端验真
{file: `server/routes/nt.py`, line: 815-835}

`POST /api/nt/earn-sync` 仅校验 `doer == user.id` 和 `amount <= 1000`。认证用户可发送任意 items 反复调用直到 `pool.balance` 耗尽。唯一刹车是池余额。（F16 引入）

### 3. 社区任务创建——流水来源错标
{file: `server/routes/tasks.py`, line: 119}

社区任务（poster="社区"）的冻结流水 `_add_ledger(db, lid, user.id, "escrow", ...)`——`from=user.id`。但 NT 来自 `pool.balance`（line 93）。应为 `from="community_pool"`。

对比: `cron.py:110` 正确写了 `from="community_pool"`。

### 4. 个人任务取消——流水来源错标
{file: `server/routes/nt.py`, line: 564}

`cancel_task` 退款流水 `_add_ledger(db, lid, "community_pool", refund_target, ...)`——`from=community_pool`。但 NT 来自 `pool.task_escrow`。应为 `from="escrow"`。

对比: 社区任务取消（line 549）正确写了 `from="escrow"`；`delete_task`（tasks.py:172）也正确写了 `from="escrow"`。

### 5. 转账 CV 通胀
{file: `server/routes/nt.py`, line: 205-208}

```python
from_user_obj.contribution_value = max(0, from_user_obj.contribution_value - cv_amount)
to_user.contribution_value = (to_user.contribution_value or 0) + int(cv_amount * 0.75)
```

发送方 CV 不足时 `max(0, ...)` 截断，但接收方仍获全额 75%。发送方 10 CV 转账 100 NT → 发送方被截为 0（实际只扣 10），接收方获 75 CV，贡献池获 25 CV。净创造 90 CV。

客户端 `nt-core.js:478-480` 有相同问题。

## 🟡 重要（4 条）

### 6. 验证驳回（超3次）——流水来源错标
{file: `server/routes/nt.py`, line: 523-524}

`pool.task_escrow` 减少，流水写 `from=community_pool`。应为 `from=escrow`。

### 7. 社区池日补填——无流水记录
{file: `server/routes/nt.py`, line: 877-880}

`pool.balance += 50; pool.total_issued += 50` 但无 `_add_ledger`。等式成立但不可审计。累计: 若池持续 <500，每年发行 ~18,250 NT 无流水。

### 8. 退房欠费——不可回收
{file: `server/routes/accommodation.py`, line: 74-81}

`if t.debt > 0 and user.nt_balance >= t.debt`——余额不足时不清零 `t.debt`，但 `t.status = "checked_out"` 已标记退房。无追讨机制。

**建议**: 至少写一条 `debt_unpaid` 流水。

### 9. 营队创建注资——池无行锁
{file: `server/routes/camps.py`, line: 85-87}

`_get_pool(db)` 未传 `lock=True`。两个管理员同时创建营队可能导致 `total_issued` 竞争。

### 10. 客户端 cancelTask 无 escrow_amount 对齐
{file: `nantang-mobile/js/nt-core.js`, line: 269}

客户端始终按 `reward * slots` 退款，不读 `escrow_amount`。file:// 离线模式可能重复退款。

### 11. 客户端 earn_sync 无日上限
{file: `nantang-mobile/js/nt-core.js`（不在客户端实现）}

服务端 `earn-sync` 无对应的客户端日上限保护。
