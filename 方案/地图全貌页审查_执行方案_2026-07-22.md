## 第一批：阻断（12 项 · ~2.5h）

### F1 — camps.py `claimants` 列不存在导致营地创建 500
**文件**：[camps.py:96](server/routes/camps.py#L96)
**问题**：T7 迁移后 NTTask 已无 `claimants` 列（改为 `assignees`），`create_camp` 仍传 `claimants=json.dumps(...)`，`NTTask(...)` 构造直接 TypeError。
**改**：`claimants=...` → `assignees=...`（claimaints JSON 已存 name 数组，直接复用）
**验证**：POST /api/camps 含 tasks → 200

### F2 — verify_task + settle_camp 双重支付营地任务
**文件**：[nt.py:549](server/routes/nt.py#L549) + [camps.py:149](server/routes/camps.py#L149)
**问题**：verify 时 `camp_balance -= total` + `assignee += reward`；settle_camp 再次 `camp_balance -= total` + `assignee += share`。assignee 得 2x，camp_balance 扣 2x。
**改**：`settle_camp` 去掉扣款和支付逻辑（L149-170），只保留状态更新（`t.status="已结算"` + `t.settler_id` + `t.settled_at`）。verify_task 是唯一支付点。
**验证**：创建营地任务→完成→verify→settle_camp→camp_balance 只减一次→assignee 只收一次

### F3 — NT verify 可重复支付
**文件**：[nt-core.js:206-207](nantang-mobile/js/nt-core.js#L206)
**问题**：`verifyTask()` 守卫允许 `verified → verified`，已结算任务可再次 verify 拿钱。`settled` 状态存在于 schema 但无代码可达。
**改**：守卫加 `verified` 和 `settled`。新增 `settleTask(taskId)` 函数：设 status="settled"。
**验证**：verify 同一任务两次 → 第二次返回 false

### F4 — 双消息处理器导致 NT 操作执行两次
**文件**：[core.js:1549](nantang-mobile/js/core.js#L1549) + [index.html:1218](nantang-mobile/index.html#L1218)
**问题**：两个文件各注册 `window.addEventListener('message', ...)`，每个 iframe postMessage 触发两次。NT earn/spend 执行两遍。
**改**：删除 `index.html:1218-1241` 中的重复处理器（保留 `core.js` 版本，功能更完整）。
**验证**：地图内 earn → NT 余额只增一次

### F5 — 双轮询定时器
**文件**：[core.js:293-298](nantang-mobile/js/core.js#L293)
**问题**：`_pollCycle` 内 `clearInterval` + 新 `setInterval`，但外层 298 行还有一个 `setInterval`。第一个周期后两个定时器并行。
**改**：删除外层 `setInterval`（298 行），只保留 `_pollCycle` 内部的递归 `setInterval`。
**验证**：登录后 30s → Network 面板只发 1 个 /api/nt/sync 请求

### F6 — 双脏污系统
**文件**：[app.js:1942](nantang-mobile/js/app.js#L1942) + [app-data.js:581](nantang-mobile/js/app-data.js#L581)
**问题**：`_growDirtiness`（app.js）写 `AppData._data.cleaning.spaces[buildingId]`（私有），`_tickDirtiness`（app-data.js）写 `map_locations.state.room_items`（共享）。不同粒度、不同增长速率、不同定时器。
**改**：废弃 `_tickDirtiness`（app-data.js 版本），统一用 `_growDirtiness`（app.js 版本）。dirtiness 数据从 cleaning.spaces 读取。
**验证**：两个来源的 dirtiness 值相同

### F7 — `_saveShared` 只同步 4/18 字段
**文件**：[app-data.js:222-229](nantang-mobile/js/app-data.js#L222)
**问题**：localStorage 存 18 字段，服务端同步只发 4 字段（camps/map_locations/inventory/canteenMenu）。tasks/users/verifications 变更永不服务端同步。
**改**：`_saveShared` 服务端分支补发 `tasks` 和 `users`（核心字段）。其余字段（announcements/config 等）低频，暂不补。
**验证**：创建任务→刷新→另一设备 syncAll→能看到任务

### F8 — `linkedTasks` 全死数据
**文件**：[app.js:15-26](nantang-mobile/js/app.js#L15) + [core.js:110](nantang-mobile/js/core.js#L110)
**问题**：11 个建筑的 `linkedTasks:[]` 永远为空。UI 渲染了链接任务区但 `openTask(taskId)` 丢弃 taskId 直接开任务大厅。
**改**：删除建筑数据中的 `linkedTasks` 字段和 UI 渲染代码（app.js:734-738）。保留 `openTask` 函数签名不变（兼容其他调用方）。
**验证**：建筑详情页不再显示空的"关联任务"区域

### F9 — 无 Hash 路由
**文件**：[core.js:1503](nantang-mobile/js/core.js#L1503) + [data.js:537](nantang-mobile/js/data.js#L537)
**问题**：`closeMyPage()` 写 `location.hash = ''` 但无 hashchange 监听。浏览器返回无效。
**改**：`closeMyPage()` 改用 `history.back()` 或 `history.pushState`。`closeOverlay` 调 `history.pushState({overlay: id}, '')`。加 `popstate` 监听处理浏览器返回。
**改量**：~20行。⚠️ 改动涉及导航栈，需测试所有 overlay 路径。
**验证**：开地图→开任务大厅→浏览器返回→回到地图（非村口）

### F10 — MGMT_DATA 与 AppData 双状态
**文件**：[app.js:856-908](nantang-mobile/js/app.js#L856)
**问题**：`MGMT_DATA` 写 `nt_mgmt_data` key，AppData 写 `nt_app_v2_shared`。大扫除/住宿/田地/厨房历史永不合并。
**改**：MGMT_DATA 的保存改为调用 `AppData._saveShared()`，不再用独立 key。MGMT_DATA._load 改为从 AppData._data 读取。
**验证**：大扫除选位→刷新→历史记录仍在

### F11 — 建筑数据三个初始化源竞争
**文件**：[app.js:29-43](nantang-mobile/js/app.js#L29) + [app-data.js:252](nantang-mobile/js/app-data.js#L252)
**问题**：`HARDCODED_BUILDINGS`、`_seedIfEmpty()`、`getBuildings()` 三个源互相覆盖。plots 数据来自两个不同源不一致。
**改**：`_seedIfEmpty` 补 `buildings` 初始化（从 `HARDCODED_BUILDINGS` 复制）。`getBuildings` 不再写入，只读取。
**验证**：首次加载→buildings 和 plots 数据一致

### F12 — transfer CV 整数截断蒸发
**文件**：[nt.py:222-226](server/routes/nt.py#L222)
**问题**：`int(X*0.75) + int(X*0.25)` 当 X%4≠0 时损失 1 CV。
**改**：`receiver = int(X * 0.75); pool = X - receiver`（确保总和 = X）
**验证**：转账 30 CV → 接收者 22 + pool 8 = 30 ✓

## 第二批：高危（10 项 · ~3h）

### F13 — 角色权限纯客户端
**文件**：[core.js:124](nantang-mobile/js/core.js#L124) + [app.js:724](nantang-mobile/js/app.js#L724)
**问题**：admin 检查全在客户端改 localStorage 即可绕过。
**改**：服务端 `/api/auth/login` 返回 `role`，客户端所有权限检查从 `API.user.role` 读取（服务端 sync 后设置）。admin-only 操作在服务端加 `require_admin`。
**验证**：改 localStorage role→刷新→权限恢复为服务端值

### F14 — camp scope 任务 escrow 路径不一致
**文件**：[tasks.py:93-94](server/routes/tasks.py#L93) + [nt.py:549](server/routes/nt.py#L549)
**问题**：创建 camp 任务时 escrow 进 `task_escrow`，verify 时从 `camp_balance` 扣，task_escrow 永不清零。
**改**：创建 camp 任务时不设 escrow（`escrow_amount=0`），预算从 `camp_balance` 扣（已在 create_camp 注资）。
**验证**：创建营地任务→escrow_amount=0→verify 后 task_escrow 不变

### F15 — 离线 earn 队列断网恢复不排空
**文件**：[app-data.js:411-428](nantang-mobile/js/app-data.js#L411)
**问题**：`_drainPendingEarns` 只在 `_finishEnter` 调（登录时）。在线恢复后 stale。
**改**：`_startPolling` 每次成功 sync 后调 `_drainPendingEarns()`。
**验证**：离线 earn→联网→30s 内自动 sync+drain→余额更新

### F16 — `verifyAction` 返回 ok 不等异步
**文件**：[app-data.js:399](nantang-mobile/js/app-data.js#L399)
**问题**：在线路径立即 return `{ok:true}`，API 调异步进行。调用方以为成功但 API 可能已失败。
**改**：函数签名改为接收 callback，或直接由调用方处理异步（不返回同步 ok）。
**验证**：断网 verify→调 API→失败→toast 提示→状态不更新

### F17 — `_applyStay` TOCTOU 竞态
**文件**：[app.js:1393-1398](nantang-mobile/js/app.js#L1393)（已重构为 `_confirmCheckin`）
**问题**：读 tenants→检查→push，中间无锁。两人同时选同一床位。
**改**：客户端先调 API checkin，服务端返回冲突时 toast "已被占用"。服务端 `checkin` 已有子查询防超额。
**验证**：两个浏览器同时选同一床位→一个成功一个 toast 提示

### F18 — 离线 admin 发布跳过角色检查
**文件**：[core.js:177-182](nantang-mobile/js/core.js#L177)
**问题**：`_adminPublishTask` 离线时不检查角色直接写 AppData。
**改**：加 `var role = (getUsers()[CURRENT_USER]||{}).role; if (role !== 'admin') { showToast('仅管理员可发布社区任务'); return; }`
**验证**：访客离线→点发布社区任务→toast 拒绝

### F19 — MGMT_DATA 独立 key 不跨设备同步
**合并入 F10**。

### F20 — `HARDCODED_BUILDINGS` 数据应外置
**文件**：[app.js:15-27](nantang-mobile/js/app.js#L15)
**问题**：200+ 行建筑/房间/物品数据写在 JS 中，改数据需改代码。
**改**：`_seedIfEmpty` 补 buildings（合并入 F11）。`HARDCODED_BUILDINGS` 保留为 fallback 但标记 deprecated。
**验证**：清空 localStorage→刷新→数据正常加载

### F21 — Script 加载顺序脆弱
**文件**：[index.html:494-511](nantang-mobile/index.html#L494)
**问题**：18 个 script 固定顺序，无依赖声明，加一个就全崩。
**改**：核心模块（app-data.js/nt-core.js/core.js）加 `if (!window.AppData) console.error('AppData not loaded')` 防护。暂不引入模块系统（改动太大）。
**验证**：调换 script 顺序→console 报警→功能降级但不白屏

### F22 — 全页无加载骨架
**文件**：[app.js:114-197](nantang-mobile/js/app.js#L114)
**问题**：`render()` 前 DOM 空白，用户看到白屏或闪烁。
**改**：`scrollArea` 内预置 3 个骨架卡片（灰色脉冲动画），`getBuildings()` 完成后替换。
**验证**：慢网速→看到骨架卡片→数据加载后替换为真实内容
