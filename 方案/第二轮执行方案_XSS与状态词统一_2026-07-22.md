## W1 — XSS 修复：`esc()` 无效 + 全局 onclick → addEventListener

### 问题
`esc()` 只转义 HTML 实体（`&<>'"`），但在 `onclick="...' + esc(name) + '..."` 中无效——浏览器先 HTML 解码再给 JS，`&#39;` 被还原为 `'`。用户名设为 `'); alert(1)//` 即可 XSS。

### 改法

**阶段 A：快速止血 — `encodeURIComponent` 替代 `esc()`（15min）**
**原理**：`esc()` 输出 HTML 实体（`'` → `&#39;`），浏览器 HTML 解码属性值后又还原为 `'`，注入成功。`encodeURIComponent` 输出 `%27`，HTML 解码不会触动 `%` 编码。
**文件**：全部 `nantang-mobile/js/*.js` + `index.html`
全局搜索 `onclick="...esc(name)..."` → 替换为 `encodeURIComponent(name)`。
接收端（函数参数）加 `decodeURIComponent` 还原。
```bash
grep -rn "esc(" nantang-mobile/js/*.js nantang-mobile/index.html | grep onclick
```
**示例**：
```javascript
// 改前（不安全）：esc() 的 &#39; 被 HTML 解码还原
onclick="openTask(\'' + esc(name) + '\')"

// 改后（安全）：%27 不会被 HTML 解码
onclick="openTask(\'' + encodeURIComponent(name) + '\')"
// 接收端：
function openTask(name) { name = decodeURIComponent(name); ... }
```
**防御原理**：`');alert(1)//` → `encodeURIComponent` → `%27);alert(1)//%27` → 浏览器不解码 `%` → JS 字符串安全。

**阶段 B：长期方案 — data 属性 + 事件委托（2h）**（阶段 A 之后执行）
全局搜索 onclick 注入点，逐一改写为 `data-action` 属性 + 统一 `click` 事件委托。

示例——改前：
```html
<div onclick="openTask(' + esc(name) + ')">任务</div>
```
改后：
```html
<div data-action="openTask" data-name="${encodeURIComponent(name)}">任务</div>
```
```javascript
document.body.addEventListener('click', function(e) {
  var btn = e.target.closest('[data-action]');
  if (!btn) return;
  var action = btn.dataset.action;
  var name = decodeURIComponent(btn.dataset.name || '');
  if (action === 'openTask') openTask(name);
  // ... 其他 action
});
```

**影响面预估**：~20 处 onclick 需改写，涉及 core.js/data.js/app.js/index.html。

### 验证
- 注册用户名为 `<img src=x onerror=alert(1)>` → 不弹窗
- 注册用户名为 `'); alert(1);//` → 不弹窗
- 所有任务卡片/用户列表/成员选择器正常点击

## W4 — 全貌页刷新跳转修复

> 2026-07-22 已修复 ✅

### 问题
刷新页面后底部滑动块停留在第一个建筑（A门），而不是默认的全貌页。`currentIdx = 4`（全貌页），但 init 调 `render()` 只建 DOM 不滚动轮播。

### 根因
[app.js:2036](nantang-mobile/js/app.js#L2036) — `render()` 只设 innerHTML，不调 `scrollIntoView`。轮播 scroll 事件触发时检测 scrollLeft=0 → 错误地切到 index 0。

### 改法
**文件**：[app.js:2036](nantang-mobile/js/app.js#L2036)
```diff
- render();
+ goTo(currentIdx);
```
`goTo()` = `render()` + `scrollIntoView(activeCard)`，一并完成渲染+滚动定位。

### 验证
刷新 → 底部滑动块居中全貌页 → 图标为📍。

## W5 — 用户系统完整性审计（2026-07-22 诊断）

### 问题

用户头像修改后不持久化、资料刷新丢失、资料页显示"ID: 未分配"、钱包地址不跟随用户。

### 全链路审计

**数据存储三层：**

| 层 | 载体 | 可写入？ | 跨设备？ |
|----|------|---------|---------|
| 服务端 | User model (SQLite) | ✅ 注册 + update_profile | ✅ |
| localStorage NT_USERS_KEY | `saveUsers()` → `nt_users` | ✅ `saveUsers()` | ❌ |
| localStorage nt_local_users | `_saveLocalUser()` → `{name: seed}` | ✅ | ❌ |

**登录后数据流：**

```
API.user (全字段) → _finishEnter → _saveLocalUser(name, seed)
                                    ↓
                              只存了 name→seed！
                              wallet/bio/uid/created 全部丢失
                                    ↓
getUsers(): NT_USERS_KEY 为空 → 回退 nt_local_users → {name, avatar_seed, role}
                                                                     ↓
                                                            uid → "未分配"
                                                            wallet → "未填写"
```

**根因**：`_finishEnter` 在 HTTP 模式下从未调 `saveUsers()`，只写了 `nt_local_users`。`getUsers()` 从 `nt_local_users` 回退只能拿到 3 个字段。

### 具体缺陷

| # | 问题 | 根因 | 严重度 |
|---|------|------|--------|
| 5.1 | 头像修改不跨设备 | `UpdateProfileRequest` 无 `avatar_seed` | 🔴 |
| 5.2 | 登录后资料显示"未分配" | `_finishEnter` 不同步 NT_USERS_KEY | 🔴 |
| 5.3 | refreshUserUI 头像随机 | `seed=null` 时从 `_avatarSeedPool[random]` | 🟡 |
| 5.4 | 不同浏览器看不到资料 | localStorage 不同步 | 🟡 |

### 建议改法

**5.1 — 服务端**：`UpdateProfileRequest` 加 `avatar_seed` 字段
```python
class UpdateProfileRequest(BaseModel):
    wallet_address: str | None = None
    bio: str | None = None
    location: str | None = None
    avatar_seed: str | None = None  # ← 新增
```

**5.2 — 客户端**：`_finishEnter` 加 `saveUsers()` 同步
```javascript
// 登录后把 API.user 全字段写入 NT_USERS_KEY
var apiU = API.user;
if (apiU && !isHTTP_file) {
  var users = getUsers();
  users[name] = {name: name, uid: apiU.uid, role: apiU.role,
    avatar_seed: apiU.avatar_seed, wallet: apiU.wallet_address,
    bio: apiU.bio, location: apiU.location, created: apiU.created_at};
  saveUsers(users);
}
```

**5.3 — 客户端**：`refreshUserUI` 头像优先级
```javascript
// 当前：seed=null时随机 → 改：洗牌稳定
if (seed == null) seed = u || _avatarSeedPool[0];  // 不随机
```

**5.4 — 客户端**：`saveProfileEdits` 同步 `avatar_seed` 到服务端
```javascript
API.updateProfile({wallet_address:wallet, bio:bio, location:loc, avatar_seed:u.avatar_seed})
```
