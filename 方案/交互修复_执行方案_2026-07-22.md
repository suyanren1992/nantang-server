## P1 — 地图配置被地图遮住

**文件**：[core.js:196](nantang-mobile/js/core.js#L196)
**根因**：`openAdminPanel()` 直接 `classList.add('open')` 打开 `overlayAdminConfig`，该元素嵌在 `overlayMap` 内部，z-index 低于地图全貌页。用户必须退出地图才能看到配置面板。
**改**：`openAdminPanel()` 改为创建独立居中弹窗（复制 `ci-overlay` 模式），不依赖 `overlayAdminConfig` DOM 节点。
**验证**：地图全貌页 → 点齿轮 → 配置面板浮在地图上方

## P2 — 个人资料保存不了（= D2）

**文件**：[auth.js:24](nantang-mobile/js/auth.js#L24)
**根因**：`saveUsers()` 在 HTTP 模式下是空函数——`if (window.location.protocol === 'file:')` 守卫拦截了所有非 file:// 协议的写入。线上环境 `saveProfileEdits()` 调 `saveUsers(users)` → 什么都不做 → 刷新丢失。
**改**：`saveUsers()` 去掉 `file:` 判断，任何协议都写 localStorage。服务端 API 已有（`API.updateProfile`），本地持久化补齐即可。
```javascript
// 旧：
function saveUsers(users) { if (window.location.protocol === 'file:') safeStorage.setItem(NT_USERS_KEY, JSON.stringify(users)); }
// 新：
function saveUsers(users) { safeStorage.setItem(NT_USERS_KEY, JSON.stringify(users)); }
```
**验证**：编辑资料 → 保存 → 刷新 → 资料仍存在

## P3 — 入住后不能添加房间物品

**文件**：[app.js](nantang-mobile/js/app.js) `_showStaySheet`
**根因**：物品功能只绑定了 A室（`if (rid==='dorm101')`）。且物品行在床位面板头部渲染，入住后展开床位的逻辑遮盖了物品区。
**改**：
1. 物品功能扩展到所有房间（去掉 `rid==='dorm101'` 限制）
2. 物品行放在床位面板顶部，在床位列表之前
3. 已入住用户也能看到 +添加按钮
**验证**：入住 B室 → 打开住宿面板 → 看到物品行 → 点 +添加 → 输入 → 物品显示

## P4 — 新人引导流程修复

**文件**：[core.js:978](nantang-mobile/js/core.js#L978) + [data.js](nantang-mobile/js/data.js#L49-L55)

### 当前（坏的）
```
登录 → 600ms 弹窗 showNewbieOnEntry()  ← 每次登录都弹，访客也弹
  → 5个任务里3个无法完成（无 hook）
  → 信箱不通知
```

### 目标
```
登录 → 静默初始化新手任务（后台）
  → 入住签公约 → 信箱通知 📬 "新手引导已开启"
  → 用户自己决定做不做
  → 完成 → 校核室 → CV解冻
```

### 改动

**4a — 删弹窗，改静默**：[core.js](nantang-mobile/js/core.js#L978)
```javascript
// 旧：setTimeout(function(){ showNewbieOnEntry(); },600);
// 新：setTimeout(function(){ _initNewbieQuests(CURRENT_USER); },600);
```

**4b — 签约后信箱通知**：[app.js](nantang-mobile/js/app.js) `_confirmCheckin` 的 `doCheckin` 里加：
```javascript
// 签约后发信箱通知
if (typeof logActivity === 'function') {
  logActivity('system', '🌱 新手引导已开启，完成5项任务解锁75贡献值');
}
```

**4c — 补 3 个缺失的 completion hook**：[data.js](nantang-mobile/js/data.js)

| 任务 | 触发点 | 当前状态 |
|------|--------|---------|
| `sign_covenant` | `_confirmCheckin` | ✅ 已有（但每次入住都触发，需加 `!q.done` 幂等） |
| `meet_3` | 营地成员互动 | ❌ 无 hook → 在 `ui-camp.js` 成员互动处加 `_completeNewbieQuest(name, 'meet_3')` |
| `checkin_5` | `goTo()` 累计 | ✅ 已有 |
| `join_meal` | 订餐 | ❌ 无 hook → 在 `ui-phase4.js` 订餐确认处加 |
| `join_cleaning` | 打扫完成 | ❌ 无 hook → 已有但触发条件不明确，在 `_doCleaning` 处加 |

**验证**：注册 → 登录 → 不弹窗 → 入住 → 信箱有新手通知 → 完成任务 → 信箱有校核通知

## P5 — 校核卡点不进去

**文件**：[app.js:2252](nantang-mobile/js/app.js#L2252) + [app.js:2197](nantang-mobile/js/app.js#L2197)

**根因**：`verifyAction` 同步返回 `{ok:true}`（不等 API 结果）。校核卡点"通过"后 UI 立即更新，但服务端可能失败——用户看到"校核完成"但实际没反应。另外 `_openVerificationPanel` 有 `isMember` 门禁，访客完全进不去。

**改**：
1. `_openVerificationPanel` 去 `isMember` 门禁——访客也能进（他的任务在排队等校核）
2. 校核按钮回调加异步处理：等服务端返回后再更新 UI，失败时 toast

**验证**：访客签到 → 打开校核室 → 看到待校核项 → 点通过 → 等服务器响应 → 成功/失败 toast

## P6 — 新版信箱通知类型

**文件**：[index.html](nantang-mobile/index.html) `_computeInboxMessages`
**问题**：信箱没有"新手引导"和"校核结果"类型的消息
**改**：`_computeInboxMessages` 加两种新消息：
- 系统通知（新手引导开启、角色升级）
- 校核结果（你的劳动被校核通过/退回）
**验证**：完成新手任务 → 校核通过 → 信箱收到通知
