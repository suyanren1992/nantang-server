  │ 等待管理员    │ ───→ │ 管理员看到待处理列表 │
  │ status=pending│     │ 3/5 多签 → 链上转账 │
  └──────┬──────┘      └────────┬────────┘
         │                      │
         │     ┌────────────────┘

    ┌────────────┐    ┌────────────┐
    │ 成功        │    │ 失败/超时   │

**人话版**：提现不是立即到账。系统先把你的 NT 冻住，等管理员用多签钱包在链上给你转账。转完确认后，系统把钱从冻结池销毁。如果 3 天没人处理，自动退给你。

### ⚠️ 关键注意事项

1. **执行顺序不可变**：第1步 DB 迁移必须先完成。第2-4步任何代码在 reserve/frozen 列不存在时运行都会 AttributeError。
2. **首次部署必须手动注资 reserve**：`POST /api/nt/topup {"user":"community_pool","amount":500}`。否则 reserve=0，用户提现永远失败。
3. **盈余划拨和自动调水同时触发**：balance>1000 时先执行盈余划拨（balance 降到 500），再检查是否需要调水（500>150 不触发）。不会出现先划出再调回的浪费。
4. **frozen 清理**：72h 超时回滚仅对 withdraw pending 生效。争议仲裁的 frozen 资金不会自动清理——需 admin 定期手动处理。
5. **BED_RATES 硬编码**：如果 dorm 房间增减，需改代码重新部署。中期方案是从客户端配置同步。
