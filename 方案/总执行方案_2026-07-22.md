## 第一轮：数据层（3 项 · 20min）

### S1 — 三个端点补多 assignee 支持
**文件**：[nt.py:99](server/routes/nt.py#L99) + [tasks.py:64](server/routes/tasks.py#L64) + [data.py:395](server/routes/data.py#L395)
**问题**：三处 `assignee == user.id` 只用单值列，多人任务的其他 assignee 看不到自己的任务。`tasks.py` 已有 `_safe_assignees` 后过滤但 SQL 层仍漏。
**改**：三处均在 `.where()` 后加 `| (NTTask.assignees.like(f'%"{user.id}"%'))`（SQLite Text 列用 LIKE 匹配 JSON 数组）。或改为后过滤 `_safe_assignees(t)`。
**验证**：用户 B 作为多人任务第 2 个 assignee → sync/任务列表/syncAll 都能看到

### S2 — fetchTasks 改调市场接口
**文件**：[api.js:141](nantang-mobile/js/api.js#L141)
**问题**：`fetchTasks` 调 `GET /api/tasks`（只返回自己相关的），任务大厅看不到别人发的委托。
**改**：URL 改为 `/api/tasks?mode=hall`
**验证**：两个用户分别登录 → 各自发任务 → 任务大厅互相能看到

### S3 — saveUsers 去 file: 守卫（= P2）
**文件**：[auth.js:24](nantang-mobile/js/auth.js#L24)
**问题**：HTTP 模式下 `saveUsers()` 不写 localStorage，资料编辑刷新丢失。不同浏览器也看不到同一用户的头像/资料。
**改**：去掉 `if (window.location.protocol === 'file:')` 守卫，任何协议都写。
**验证**：编辑资料 → 保存 → F5 → 资料仍在

## 第二轮：经济安全（3 项 · 30min）

### E1 — verify_task escrow 统一释放
**文件**：[nt.py:553-558](server/routes/nt.py#L553) + [tasks.py:117](server/routes/tasks.py#L117)
**问题**：非 camp 任务 verify 时只扣 `reward × len` 而非 `escrow_amount`，task_escrow 膨胀。camp 任务 escrow 永不释放。
**改**：
- 非 camp：`pool.task_escrow -= task.escrow_amount`（替代 `total_payout`）
- camp：加 `pool.task_escrow -= task.escrow_amount`
- 创建 camp 任务时 `escrow_amount = 0`（预算从 camp_balance 走）
**验证**：创建 3 槽位任务（1 人认领）→ verify → task_escrow 归零

### E2 — 营地创建不凭空印钞
**文件**：[camps.py:108-109](server/routes/camps.py#L108)
**问题**：`pool.total_issued += camp_total` 无人扣款，NT 凭空产生。
**改**：去掉 `total_issued += camp_total`（营地预算不是新发行，是社区池拨款）。营地预算从 `pool.balance` 扣除，或改为管理员手动注资。
**验证**：创建营地 → pool.total_issued 不变

### E3 — withdraw NameError
**文件**：[nt.py:347](server/routes/nt.py#L347)
**问题**：`func.count` 未导入。
**改**：文件顶部 `from sqlalchemy import select, func`
**验证**：调 withdraw → 不 500

## 第三轮：交互修复（5 项 · 1h）

### U0 — 多用户登录后点不进主页
**文件**：[core.js:971-973](nantang-mobile/js/core.js#L971)
**根因**：`_finishEnter` 只关 `overlayCommunity`/`overlayQuestHall`/`overlayMap` 三个 overlay。前一个用户打开的营地/档案室/配置面板等残留遮住 villagePage。`profileCard`、`mgmt-sheet`、`ci-overlay` 也不会被清理。
**改**：`_finishEnter` 里加通用清理：
```javascript
document.querySelectorAll('.overlay.open').forEach(function(o){o.classList.remove('open')});
document.querySelectorAll('.mgmt-sheet,.ci-overlay,.confirm-card,.profile-card').forEach(function(e){e.remove()});
```
替换硬编码的三个关闭行。
**验证**：用户 A 开营地页 → 退出 → 用户 B 登录 → 村口正常可见

### U1 — 地图配置遮罩
**文件**：[core.js:196](nantang-mobile/js/core.js#L196)
**改**：`openAdminPanel()` 改为创建独立居中弹窗（`ci-overlay` 模式），不依赖 overlayMap 内的 DOM
**验证**：地图全貌页 → 点齿轮 → 配置面板浮在地图上方

### U2 — 全房间物品
**文件**：[app.js](nantang-mobile/js/app.js) `_showStaySheet`
**改**：去掉 `if (rid==='dorm101')` 限制，所有房间显示物品行
**验证**：入住 B室 → 住宿面板 → 加物品

### U3 — 新人引导静默
**文件**：[core.js:978](nantang-mobile/js/core.js#L978) + [data.js](nantang-mobile/js/data.js) + [app.js](nantang-mobile/js/app.js)
**改**：
- 两处 `showNewbieOnEntry()` → `_initNewbieQuests(CURRENT_USER)`
- 签约后 `logActivity('system', '🌱 新手引导已开启')`
- 补 `meet_3`/`join_meal`/`join_cleaning` 的 completion hook
**验证**：登录 → 不弹窗 → 入住 → 信箱有通知

### U4 — 校核室去门禁
**文件**：[app.js:2197](nantang-mobile/js/app.js#L2197) + [app.js:2252](nantang-mobile/js/app.js#L2252)
**改**：去 `isMember` 门禁；校核按钮等服务端响应后再更新 UI
**验证**：访客 → 打开校核室 → 看到待校核 → 点通过 → 等响应

### U5 — 信箱新类型
**文件**：[index.html](nantang-mobile/index.html) `_computeInboxMessages`
**改**：加系统通知（新手引导）+ 校核结果（通过/退回）
