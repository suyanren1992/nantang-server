## 第一步：修复 NT 系统 CRITICAL Bug

### 1.1 服务端 spend 修复
**文件：** `server/routes/nt.py`  
**行号：** ~171  
**问题：** `spend()` 只有 `scope=="personal"` 时才把 NT 返回社区池，其他 scope 时 NT 消失。  
**修复：** 去掉 scope 判断，总是返回社区池。

```python
# 找到这段（约 170-173 行）：
    if req.scope == "personal":
        pool = await _get_pool(db)
        pool.balance += req.amount

# 改为：
    pool = await _get_pool(db)
    pool.balance += req.amount
```

### 1.2 服务端 verify_task 修复
**文件：** `server/routes/nt.py`  
**行号：** ~306-312（approve），~326-332（reject）  
**问题：** 如果 assignee 或 poster 用户被删除，NT 销毁。  
**修复：** 添加用户不存在的保护。

```python
# approve 路径（约 306-312 行）：
# 找到：
        if assignee:
            assignee.nt_balance += task.reward
            ...
        pool.task_escrow -= task.reward

# 改为：
        if not assignee:
            raise HTTPException(status_code=400, detail="任务执行者不存在")
        assignee.nt_balance += task.reward
        ...
        pool.task_escrow -= task.reward

# reject 路径（约 326-332 行）：
# 找到：
        if poster:
            poster.nt_balance += task.reward
            pool.task_escrow -= task.reward

# 改为：
        if poster:
            poster.nt_balance += task.reward
        else:
            pool.balance += task.reward  # 孤儿托管退还给社区池
        pool.task_escrow -= task.reward
```

### 1.3 服务端 tasks.py create_task 修复
**文件：** `server/routes/tasks.py`  
**行号：** ~80-85  
**问题：** 如果 CommunityPool 不存在，create_task 的 escrow 不会增加，NT 消失。  
**修复：** 使用 `_get_pool()` 代替直接查询。

```python
# 文件顶部加 import（约第 10 行，已有 from routes.nt import _ledger_id, _add_ledger, _adjust_trust）：
from routes.nt import _ledger_id, _add_ledger, _adjust_trust, _get_pool

# 找到（约 80-85 行）：
    pool_result = await db.execute(select(CommunityPool).limit(1))
    pool = pool_result.scalar_one_or_none()
    if pool:
        pool.task_escrow += req.reward

# 改为：
    pool = await _get_pool(db)
    pool.task_escrow += req.reward
```

### 1.4 服务端 tasks.py delete_task 修复
**文件：** `server/routes/tasks.py`  
**行号：** ~145  
**问题：** `max(0, ...)` 静默产生 NT；poster 可能不存在。  
**修复：** 断言一致性 + poster 保护。

```python
# 找到（约 137-148 行）：
    if task.escrow_amount > 0 and task.status != "已结算":
        poster_result = await db.execute(select(User).where(User.id == task.poster))
        poster = poster_result.scalar_one_or_none()
        if poster:
            poster.nt_balance += task.escrow_amount
        pool_result = await db.execute(select(CommunityPool).limit(1))
        pool = pool_result.scalar_one_or_none()
        if pool:
            pool.task_escrow = max(0, pool.task_escrow - task.escrow_amount)

# 改为：
    if task.escrow_amount > 0 and task.status != "已结算":
        poster_result = await db.execute(select(User).where(User.id == task.poster))
        poster = poster_result.scalar_one_or_none()
        pool = await _get_pool(db)
        if poster:
            poster.nt_balance += task.escrow_amount
        else:
            pool.balance += task.escrow_amount  # 孤儿托管退还给社区池
        if pool.task_escrow < task.escrow_amount:
            raise HTTPException(status_code=500, detail="托管金额不一致")
        pool.task_escrow -= task.escrow_amount
```

### 2.4 更新 server/main.py — 加缓存策略

**文件：** `server/main.py`

JS/CSS 文件加长期缓存（版本号控制更新），HTML 不缓存：

```python
# 在 StaticFiles mount 之前加：
from starlette.staticfiles import StaticFiles as _StaticFiles

@app.middleware("http")
async def add_cache_headers(request, call_next):
    response = await call_next(request)
    if request.url.path.endswith('.js') or request.url.path.endswith('.css'):
        response.headers['Cache-Control'] = 'public, max-age=86400'
    elif request.url.path.endswith('.png') or request.url.path.endswith('.jpg'):
        response.headers['Cache-Control'] = 'public, max-age=604800'
    return response
```

```javascript
// 在 AppData 对象中添加：
syncCamp: function(camp) {
  if (typeof API === 'undefined' || !API.token) return;
  var xhr = new XMLHttpRequest();
  xhr.open('POST', '/api/camps', true);
  xhr.setRequestHeader('Content-Type', 'application/json');
  xhr.setRequestHeader('Authorization', 'Bearer ' + API.token);
  try { xhr.send(JSON.stringify(camp)); } catch(e) {}
}
```

